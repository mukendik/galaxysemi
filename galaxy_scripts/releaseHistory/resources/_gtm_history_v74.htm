GCORE-4158 Some JaveScript API calls do not work properly via HTTP requests The following JavaScript API calls accept file paths as method argument but fail when they are called from an HTTP request:
* GSEngine.RemoveFileFromDisk(<filepath>)
* GSFile.Remove(<filepath>)
* GSFile.Copy(<filesrc>,<filedest>)

Bug fixes
Case Number Subject Description
GCORE-3784 Undefined pin / test combinations in final test PAT logs When the PAT recipe changes between runs but the lot ID is the same, an init error occurs because the tests in the last PAT recipe do not match the current PAT recipe.

V7.4.E (10 May 2015) (Top)
Enhancements
Case Number Subject Description
GCORE-3629 Allow final test PAT to use test name as key instead of test number Not all testers support both test numbers and test names. For testers that support test names only, PAT recipes must be able to use test names as keys instead of test numbers.

V7.4.D (3 April 2015) (Top)
Enhancements
Case Number Subject Description
GCORE-815 Update GTL process to use new SQLite database schema The test data generated from Quantix Tester Library calls are collected in an SQLite file. V7.4 supports the insertion of the SQLite data into a Test Data Repository. This software enhancement enables the GTL process to use the new SQLite database schema.
Bug fixes
Case Number Subject Description
GCORE-3432 An exception was discovered in a PAT traceability file For PAT at final test, the baseline distribution shape used to identify outliers is normally correctly written into a traceability file and displayed in a PAT report. A customer discovered an abnormal traceability file from which the wrong baseline distribution shape was reported.

In some cases, distribution shapes written into a PAT traceability file and displayed in a final test PAT report are wrongly reported.
GCORE-2641 RESTful API is not functioning properly with the GTM When using RESTful commands with the GTM to stop and start the tool, the GTM does not respond as expected.

V7.4.G (27 February 2015) (Top)
Bug fixes
Case Number Subject Description
GCORE-1482 Cannot stop the GTM (Quantix Tester Monitor) daemon from a command line Once the GTM daemon is installed, there is a convention to type "sudo service gs-gtmd start|stop|status" at a command line. The "sudo service gs-gtmd stop" command does not work.

V7.4.A (20 November 2014) (Top)
Enhancements
Case Number Subject Description
GCORE-1302 Additional resume/retest matching GTL (Quantix Tester Library) parameters Common GTL resume/retest matching parameters are lot_id, tester_id, and program_name. Extend the matching parameters to include all available metadata to specify how to match the current lot to previously run lots in memory (temp file history).
Bug fixes
Case Number Subject Description
GCORE-1674 GTM daemon is attempting to get an SYA, TDR, and gene license, which it does not need During start-up, the Quantix Tester Monitor (GTM) is getting an gex-init license, which it needs, but is also attempting to get an SYA, TDR, and gene license, which it does not need.
GCORE-495 When running GTL, a SELECT query will trigger an error When running GTL, a SELECT query will trigger an error while a select trigger will not trigger an error. Fix this case sensitivity issue.
