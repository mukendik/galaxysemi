gexOptions(
  const section,      // section entry name
  const field,        // field name
  const value)        // field value

Sets the specified option entry field to a given value. Used to configure all the gex report generation
options and environment . Fields are not case sensitive except when specifying a file path.

For a complete example, take a look at the ready-to-use template options.csl

Example:

// Sets report output format to HTML
gexOptions('output','format','html');
// Because parameters are not case sensitive, it is the same as: 
gexOptions('Output','Format','HTML');
// or even the same as...
gexOptions('Output','FORMAT','HTML');

Here is the list of sections you can configure using this script function:

Section Description
reset Reset all options at once to default values, or to all fields disabled
databases lets you define the path to the databases: Examinator Pro only
output Set the report output format (HTML, Word, PDF,...) , paper size and location

duplicate_test Define how to handle identical test numbers with different test names (merge / no merge)
part_id Show/Hide PartID on trend chart & tables.
sorting Define the data sorting policy (none, or by date)
fail_count Define how test failures must be counted
multi_parametric Define how to handle multi-results parametric tests (merge or No merge mode)
stdf_compliancy Define how to handle STDF data files not compliant
outlier Define the outlier policy
statistics Define the test statistics table options
histograms Define the standard histogram options (3D prebuilt .gif images)
wafer Define the wafer map report options
pareto Define the pareto reports options
binning Define the binning summary report options
adv_histogram Define the advanced histogram report options
adv_trend Define the advanced trend report options
adv_correlation Define the advanced correlation, XY scatter, BiVariate report options
adv_productionyied Define the production yield/volume report options
adv_boxplot Define the advanced Gage R&R - box-plot report options
adv_datalog Define the advanced datalog report options
adv_what_if Define the options for the what-if wizard (which bin# to use when passing a fail part, or when failing a good part).
adv_pearson Define the options for the pearson's correlation report (tests that highly correlate). Report typically used for optimizing test time by removing some tests that strongly correlate with other.
toolbox Define the options for the toobox functions.

Section: reset

Field Value / Description
all Resets all options of all sections to either one of the following stage:
default resets all options to default value (as when first installing GEX). This mode is convenient if you do not want to specify each section options individually.
clear resets all options to a cleared value. This option must only be used when followed by other functions that specifically set options. This command allows starting from a 'blank page'; if not followed by any other options selection, the report will be totally empty!

Example:

// Reset all options to default, and change report format to CSV
gexOptions('reset','all','default');
gexOptions('output','format','CSV');	// Default was HTML, switch to CSV
 

Section: databases (Examinator Pro only)

Field Value / Description
local_path <path> where <path> is the path to the local databases folders. If value is (default) or empty, then the default local ExamintorDB path is used:
<Examinator application path>/databases
server_path <path> where <path> is the path to the server databases folders. If value is (default) or empty, then the default server ExamintorDB path is used:
<Examinator server application path>/databases

Example:

// Sets paths to Local and Server databases
gexOptions('databases','local_path','/export/home/my_local_databases');
gexOptions('databases','server_path','/mnt/srv/server-databases');
 

Section: output

Field Value / Description
format html to have report generated in HTML format.
word to have report generated in MS-Word document (PC versions only)
ppt to have report generated in PowerPoint slides (PC versions only)
pdf to have report generated in PDF format.
csv to have the report generated in Excel/.CSV ASCII format

Note: If the script is launched under the Examinator user interface, all formats are accepted. If the script is executed by launching Examinator in command line mode, then only HTML mode will be accepted. For background script executions supporting other output modes than HTML you need the Quantix Yield-Man software (accepts all formats like PDF, Word, etc...)
front_page_text Text string to display on the report's first page (front page) when report format is Word, Power Point or PDF
front_page_image Path to the image / logo to display on the report's first page (front page) when report format is Word, Power Point or PDF. Image format can be any of the following: .gif .png .bmp and .jpg
truncate_names Set to 'yes' to have test names in Word, PDF and PowerPoint reports truncated (32 characters max.). Allows to maintain accurate page formatting.
Set to 'no' to keep test names to their full length (
paper_size A4 to have report page formatting set to A4 (applies to Word & PDF reports only)
Letter to have report page formatting set to A4 (applies to Word & PDF reports only)
paper_format portrait or landscape to define the page orientation. 'portrait' is the default. Applies to Word, PDF reports only.
location <path> where <path> is the path where the report should be created.
or
file_path to have the report created in the same directory as the file selected. If multiple files are analyzed, the path of the first file in the first group is used.

Example:

// Sets report output format to HTML and specify a path
gexOptions('output','format','html');
gexOptions('output','truncate_names','yes');
gexOptions('output','paper_size','Letter');
gexOptions('output','paper_format','landscape');
gexOptions('output','front_page_text','This is my report!');
gexOptions('output','front_page_image','/export/home/logo.jpg');
gexOptions('output','location','/export/home/report');
 

Section: speed

Field Value / Description
adv_stats Applies to Examinator-Pro edition only, let syou define when advanced statistics must be computed (includes: Skew, Kurtosis, Quartiles and Median computation). As such computation require significant CPU time, it can be wise to do disable it when very large datasets are processed, and when focus is not on these information. Value can be one of the following:

always to always compute advanced statistics, no matter how large are datasets
50mb to only compute advanced statistics if total dataset is 50Mbytes or less
100mb to only compute advanced statistics if total dataset is 100Mbytes or less
200mb to only compute advanced statistics if total dataset is 200Mbytes or less
300mb to only compute advanced statistics if total dataset is 300Mbytes or less
400mb to only compute advanced statistics if total dataset is 400Mbytes or less
500mb to only compute advanced statistics if total dataset is 500Mbytes or less
never to disable advanced statistics computation.
collect_samples Applies to Examinator-Pro edition only, lets you define when data samples are collected and available under the Interactive Charting mode. Value can be one of the following:

always to always collect data samples, no matter how large are datasets
50mb to only collect data samples if total dataset is 50Mbytes or less
100mb to only collect data samples if total dataset is 100Mbytes or less
200mb to only collect data samples if total dataset is 200Mbytes or less
300mb to only collect data samples if total dataset is 300Mbytes or less
400mb to only collect data samples if total dataset is 400Mbytes or less
500mb to only collect data samples if total dataset is 500Mbytes or less
never to disable data samples collection for Interactive Charting.
db_summary Applies to Examinator Pro edition only, lets you define when to query the database summary records instead of the database samples. Queries over the summary records are typically 50 to 100 times faster as much fewer information is processed. This is ideal when running queries over very large datasets and when only looking at test statistics, or binning information. When the database summary records are used instead of the data samples, no histogram, wafermap or trend chart on data is available. Trending on binning per lot is however possible.:

always to always use the database summary record (ignore data samples)
50mb to only use the database summary record if dataset is 50Mbytes or more
100mb to only use the database summary record if dataset is 100Mbytes or more
200mb to only use the database summary record if dataset is 200Mbytes or more
300mb to only use the database summary record if dataset is 300Mbytes or more
400mb to only use the database summary record if dataset is 400Mbytes or more
500mb to only use the database summary record if dataset is 500Mbytes or more
never to never use database summary records, only data samples.

Example:

// Optimize Speed Vs. Datasets size processed
gexOptions('speed','adv_stats','50mb');
gexOptions('speed','collect_samples','100mb');
gexOptions('speed','db_summary','200mb');
 

Section: duplicate_test

Field Value / Description
merge Means all tests with same test number ID will be merged, even if the test name string is sometimes different.
no_merge Means not to merge tests with same test number ID if test names are different (default)

Example:

// Sets to sort data by testing date.
gexOptions('duplicate_test','no_merge','');

 

Section: part_id

Field Value / Description
show Have the PartID displayed in the tables (under interactive mode), and also use it as the X axis scale in trend charts. Usually used when conducting characterization analysis where individual PartIDs are required (eg: parts tested at different temperature, etc....). This mode should not be used on large data files unless you have enough computer memory !
hide This is the default: test results do not display their PartID and the trend chart uses the test result sample index as the X axis scale.

Example:

// Characterization mission:  need to know each individual PartID
gexOptions('part_id','show','');

Section: sorting

Field Value / Description
none Means data are not sorted but processed in the order of data files listed (from first file to last file in each group). Select this mode when you want to maintain data in the order of the files selected. This is the default mode.
date Means to sort data sets based on the date of test. Under this mode files order doesn't matter. Select this mode when you want data to be organized according to the testing date.

Example:

// Sets to sort data by testing date.
gexOptions('sorting','date','');

 

Section: fail_count

Field Value / Description
all Means all test failures found in the data file flow are counted. This is the default mode.
first Means only the first failure in each test program execution is counted.

Example:

// Sets to count all failures detected.
gexOptions('fail_count','all','');

 

Section: multi_parametric

Field Value / Description
merge Means all pins results for a multi-results parametric test are merged under a same test number and test name. This is the default mode.
no_merge Means each pin result in a multi-results parametric test is assigned a unique individual test number (test#.Pinmap#) and name.

Example:

// Sets to have each pin parametric test result have a unique test name
gexOptions('multi_paramteric','no_merge','');

 

Section: stdf_compliancy

Field Value / Description
stringent Examinator expects the file to be 100% compliant with the STDF specifications.
flexible Examinator does its best to extract all data found, even if not all STDF records and sequences are compliant. This is the default mode.

Example:

// Sets to quietly handle any type of STDF file, do best to extrat data no matter what
gexOptions('stdf_compliancy','flexible','');

 

Section: outlier

Field Value / Description
removal Value must be one of the following outlier removal strategy:
none for no outlier removal: all parts are processed!
100% for only processing good parts
150% for removing parts over 150% of the limits range
200% for removing parts over 200% of the limits range
250% for removing parts over 250% of the limits range
300% for removing parts over 300% of the limits range
1sigma  for removing parts outside the 1sigma  space centered on the mean
2sigma  for removing parts outside the 2sigma  space centered on the mean
3sigma  for removing parts outside the 3sigma  space centered on the mean
4sigma  for removing parts outside the 4sigma  space centered on the mean
5sigma  for removing parts outside the 5sigma  space centered on the mean
6sigma  for removing parts outside the 6sigma  space centered on the mean
7sigma  for removing parts outside the 7sigma  space centered on the mean
8sigma  for removing parts outside the 8sigma  space centered on the mean
9sigma  for removing parts outside the 9sigma  space centered on the mean
n_sigma  for removing parts outside the +/-N*sigma, where N is specified in the following argument.
iqr for removing samples outside of Q1-1.5*IQR, Q3+1.5*IQR space (parts are not removed)
n_iqr for removing samples outside of Q1-N*IQR, Q3+N*IQR space (parts are not removed), where N is specified in the following argument.

Example1:
// Set outlier removal to only process good parts
gexOptions('outlier','removal','100%');

Example2:
// Set outlier removal to parts 200% outside test limits (+/- 100% from the center of the limits)
gexOptions('outlier','removal','200%');

Example3:
// Set outlier removal to remove parts outside the + or - 3sigma space from the mean
gexOptions('outlier','removal','6sigma');

Example4:
// Set outlier removal to remove parts outside the + or - 6.23 sigma space from the mean
gexOptions('outlier','removal','n_sigma','6.23');

Example5:
// Outlier rule: Q1-N*IQR, Q3+N*IQR, where N= 1.6
gexOptions('outlier','removal','n_iqr','1.6');

 

Section: statistics

Field Value / Description
computation Defines how the test statistics percentages and values must be computed; can be one of the following:

samples_only makes the statistics computed from the data samples only, and all summary records are ignored. This selection is very convenient when the summary is corrupted or was not reset prior to start a production lot.

samples_then_summary : statistics are computed reading the data files samples records (if any), and summary records otherwise.

summary_only : statistics are computed reading the data files summary records only.

summary_then_samples  this is the default mode: statistics are computed reading the data files summary records (if any), and sampled data otherwise.
 
cp_cpk_computation Defines how the test Cp and Cpk statistics are computed for non-normal distributions ; can be one of the following:

standard makes the Cp and Cpk standard formulas used:
Cp = (High_Limit - Low_Limit)/6*Sigma
Cpk Low = (Mean - Low_Limit)/3*Sigma
Cpk High = (High_Limit - Mean)/3*Sigma
Cpk = min(Cpk Low, Cpk High)

percentile makes the percentile formula used instead of the classic sigma one:
 Cp = (High_Limit - Low_Limit)/(P99.5 - P0.5)
Where P99.5 represents the 99.5% percentile
and P0.5 represents the 0.5% percentile

Cpk Low = (Median - Low_Limit)/(Median - P0.5)
Cpk High = (High_Limit - Median)/ (P99.5 - Median)
Cpk = min(Cpk Low, Cpk High)
 
field Defines which fields to include in the test statistics tables. Value can be one
of the following field names:
test_name to include test name
test_type to include test type (digital, parametric,...)
limits to include the test limits
shape to tell the distribution shape/type (gaussian, lognormal, bi-modal,...)
test_flow_id to display the test order in the testing sequece
exec_count to include the execution count
fail_count to include the failures count
fail_percent to include the failure percentage over parts produced
outlier_count to include the outlier count
mean to include the mean value
mean_shift to include the mean shift (only used when comparing files)
sigma to include the sigma value
sigma_shift to include the sigma shift (only used when comparing files)
2sigma to include the computed 2xSigma value
3sigma to include the computed 3xSigma value
6sigma to include the computed 6xSigma value
min to include the minimum value found
max to include the maximum value found
range to include the range value (max-min)
max_range to include the max of all ranges in same test (only used when comparing files)
cp to include the Cp value
cp_shift to include the Cp shift (only used when comparing files)
cpk to include the Cpk value (min of CpkH and CpkL)
cpkL to include the Cpk_Low value (the Cpk computed with the Low limit)
cpkH to include the Cpk_High value (the Cpk computed with the High limit)
cpk_shift to include the Cpk shift (only used when comparing files)
yield to include the computed yield value for the test
ev to include Gage EV (Repeatability / equipment variation). Note 1
av to include Gage AV (Reproducibility / appraiser variation). Note 1
r&r to include the Gage R&R value. Note 1
pv to include Gage PV (Part Variation)
tv to include Gage PV (Total Variation)

Note1: implies advanced report box-plot selected

Advance statistics fields:
Skew to include the degree of asymmetry of a distribution around its mean
Kurtosis to show relative peaked ness or flatness of a distribution compared with the normal distribution
 P0.5% to include the percentile 0.5%
 P2.5% to include the percentile 2.5%
 P10% to include the percentile 10%
P25% or Quartile1 to include the first quartile (percentile 25%)
P50% or Quartile2 to include the second quartile also called Median (percentile 50%)
P75% or Quartile3 to include the third quartile (percentile 75%)
 P90% to include the percentile 90%
 P97.5% to include the percentile 97.5%
 P99.5% to include the percentile 99.5%
InterQuartiles to include range between 3rd and 1st quartiles
SigmaInterQuartiles to include the standard deviation of the population between 1st and 3rd quartiles
smart_scaling Specifies if statistics computed (mean, sigma, etc...) must be rescaled for easier reading purpose ; eg: 1mV instead of 0.001V (applies to parameters with units in Amps, Volts, Ohms, Watts, and Hertz only). Field value can be either yes or no:
yes to enable smart scaling (then a mean of 0.01V will show 10mV)
no to disable the smart scaling, in which case no rescaling take place.
normalized to normalize all test results and test units (e.g: A, V, etc...).
to_limits to normalize all test results to the same scale as the test limits (High limit in case low and high limits not sharing the same scaling).
sorting Specifies which field type to sort the generated table. Can be one of the following:
test_number to sort by ascending test number
test_name to sort by ascending test name
test_flow_id to sort test by execution order
mean to sort by descending mean
mean_shift to sort by descending mean-shift
sigma to sort by descending sigma
cp to sort by ascending Cp
cpk to sort by ascending Cpk
r&r to sort by descending gage R&R value

Note: Only one sorting mode can be selected at a time. Also when comparing files or groups of files, sorting is done on the Reference group (group #1).
alarm_test_cpk Allows setting an alarm level for tests with a Cpk value lower than a specified value. If the alarm level is reached, the cpk will appear on a red background in the statistics report table (except under CSV report ), or with *F* for .CSV report.

Note: Any negative number disables the alarm
warning_test_cpk Allows setting an alarm warning level for tests with a Cpk value lower than a specified value and higher than the alarm (alarm_test_cpk) level, if defined.. If the alarm level is reached, the cpk will appear on a yellow background in the statistics report table (except under CSV report ), or with *F* for .CSV report.

Note: Any negative number disables the alarm
alarm_test_yield Allows setting an alarm level for tests with a Test yield value lower than a specified value. If the alarm level is reached, the Yield will appear on a red background in the statistics report table (except under CSV report ), or with *F* for .CSV report.

Note: Any negative number disables the alarm
warning_test_yield Allows setting an alarm warning level for tests with a Test yield value lower than a specified value and higher than the alarm (alarm_test_yield) level, if defined.. If the alarm level is reached, the Test yield will appear on a yellow background in the statistics report table (except under CSV report ), or with *F* for .CSV report.

Note: Any negative number disables the alarm
mean_drift_formula Used to define how the Mean drift must be computed. Can be either:
  • Drift of the absolute value (field value value)
  • Drift as a percentage of the test limits (field value limits)
alarm_mean Allows setting an alarm drift level when you compare data samples and the mean drift exceeds a specified value (a percentage of value drift or a percentage of the limit space, depending of the value defined for the field mean_drift_formula above). If the alarm level is reached, the mean shift will appear on a red background in the statistics report table for HTML reports, or an *F* for .CSV reports. The value specified must be the maximum percentage of drift allowed, for example: 5%.

If 'mean_drift_formula' is set to 'value', formula used is:
MeanDrift = 100*((Mean / ReferenceMean) -1)

If 'mean_drift_formula' is set to 'limits', formula used is:
MeanDrift = 100*(Mean - ReferenceMean)/LimitSpace

Note: Any negative number disables the alarm
alarm_sigma Allows setting an alarm drift level when you compare data samples and the sigma drift exceeds a specified value (percentage of value drift). If the alarm level is reached, the sigma shift will appear on a red background in the statistics report table for HTML reports, or an *F* for .CSV reports. The value specified must be the maximum percentage of drift allowed, for example: 1%.

Formula: SigmaDrift = 100*((Sigma/ReferenceSigma)-1)
Note: Any negative number disables the alarm
alarm_cpk Allows setting an alarm drift level when you compare data samples and the cpk drift exceeds a specified value. If the alarm level is reached, the cpk shift will appear on a red background in the statistics report table (HTML report only), or with *F* for .CSV report. The value specified must be the maximum percentage of drift allowed, for example: 5%.

Formula: CpkDrift = 100*((Cpk/ReferenceCpk)-1)
Note: Any negative number disables the alarm
section_name Lets you define the section name in the value field.

Example:

// Generates a table with pareto of Cpk values
gexOptions('statistics','section_name','Tests Statistics');
gexOptions('statistics','field','test_name');
gexOptions('statistics','smart_scaling','yes');
gexOptions('statistics','field','limits');
gexOptions('statistics','field','exec_count');
gexOptions('statistics','field','fail_count');
gexOptions('statistics','field','mean');
gexOptions('statistics','field','range');
gexOptions('statistics','sorting','Cpk');
gexOptions('statistics','alarm_mean','5%');
gexOptions('statistics','alarm_sigma','1%');

// Highlight tests with Cpk < 1.33
gexOptions('statistics','alarm_test_cpk','1.33');

// Highlight in RED tests failing over 85% of the time, and in Yellow if in 85-90%
gexOptions('statistics','alarm_test_yield','85.0');
gexOptions('statistics','warning_test_yield','90.0');

// Mean drift to be computed as a percentage of test limits
gexOptions('statistics','mean_drift_formula','limits');

Section: wafer

Field Value / Description
chart_view Defines the chart orientation view; can be one of the following:
mirror_x to mirror X values (left becomes right)
no_mirror_x to disable mirror X
mirror_y to mirror Y values (top becomes bottom)
no_mirror_y to disable mirror Y
shape_round to resize die size so wafer always looks round
shape_default to disable die resize (wafer my not look round if die count in X and Y highly differs)
low_yield_pattern Defines the low-yield threshold used to detect failing patterns in stacked wafers (any die location on the stacked wafermap that has a yield lower than this limit is considered a failing pattern). The value can be any number from 0 to 100% (eg: 33.5)
 
chart_show Defines the types of wafermaps to create and display. the parameter is an integer number with the following valid data bits:
bit0: set to '1' to show STACKED wafer (one stacked wafermap per group/query)
bit1: set to '1' to show all individual wafers.
bit2: set to '1' to show the dies not matching (when comparing multiple wafers)

Examples:
gexOptions('wafer','chart_show',2); // Only Show individual wafers
gexOptions('wafer','chart_show',4); // Show dies not matching
chart_size Defines the size of the wafer map image created in the HTML report. Can
be one of the following:
small for small image or mosaic if multiple wafer maps reported
medium for medium image
large for large image
auto to let GEX determine the size depending on the number of charts in the report
chart_view Lets you force to mirror dies over the X or Y axis (to compensate wrong probing directions). Can be one of the following for the X probing direction:
mirror_x to swap left dies with right dies
no_mirror_x to disable X mirroring
mirror_y to swap top dies with bottom dies
no_mirror_y to disable Y mirroring
all_parts to display a full wafer with all dies no matter the part filtering enabled (filter then only applies to samples and statistics but doesn't prevent from having a complete wafer image)
filtered_parts to only report dies matching the filtering criteria (eg: list of parts, or bin list, etc)
retest_policy Lets you define how to handle the binning for a dies tested multiple times. Can be one of the following :
last_bin to keep the bin of the last retest (default mode)dies
highest_bin to promote the highest bin value found in the retests
 
marker Define which markers to display on the wafer map chart. Markers support include:
bin to have the value displayed in each wafer map die (measurement for parametric wafer map, or Bin# or bin count for stacked wafer).
retest to have the dies retested show a black outline border (this only applies to wafer maps of software or hardware binnings; wafermap for parametric tests ignore this option).
section_name Lets you define the section name in the value field.
gross_die Lets you define the gross die count per wafer to use (instead of the physical dies tested in each wafer) when computing Yield related results. The Gross die count is specified as an integer in the value field; a negative value disables the gross die count support.

Example:

// Define wafer map options: adaptive size, show binning in wafer map
// Mirror in X and in Y !
gexOptions('wafer','section_name','Wafer maps');
gexOptions('wafer','chart_view','mirror_x');
gexOptions('wafer','chart_view','mirror_y');
gexOptions('wafer','chart_size','auto');
gexOptions('wafer','marker','bin');
gexOptions('wafer','marker','retest');

 

Section: pareto

Field Value / Description
section Defines if the pareto section is to be included in the report generated. Value must be one of:
disabled : section is not included in the report
enabled : all pareto sections are included in the report
<integer value>: When the value specified is an integer number then bits are decoded as following:
bit0 (0x1): set to display the 'Pareto of Cp' paragraph
bit1 (0x2): set to display the 'Pareto of Cpk' paragraph
bit2 (0x4): set to display the 'Pareto of test failures' paragraph
bit3 (0x8): set to display the 'Pareto of Failure signatures' paragraph
bit4 (0x10): set to display the 'Pareto of Soft Bin' paragraph
bit5 (0x20): set to display the 'Pareto of Hard Bin' paragraph
cutoff_cp Defines the highest Cp value to be listed in the Pareto reports. Parameters with a higher Cp value will not be listed. E.g: if this value is set to 1.67, then only parameters with a Cp lower or equal to 1.67 (including negative values) will be listed. Specify a negative number (typically -1) to disable the cutoff limit and have all parameter's Cp listed.
 
cutoff_cpk Same as above, but for Cpk values.
 
cutoff_failure Defines the maximum number of failing parameters to report (Top N tests). Specify a negative number (typically -1) to disable the cutoff limit and have all failures listed.
 
cutoff_signature_failure Defines the percentage of signature failures to report (value can range from 0% to 100%). Specify a negative number (typically -1) to disable the cutoff limit and have all signature failures listed.
 
section_name Lets you define the section name in the value field.

Example:

// Makes Pareto report ALL Cp, and only Cpk <= 1.33
// report first 20% of signature failures detected (failing patterns)
gexOptions('pareto','section_name','Pareto reports');
gexOptions('pareto','cutoff_cp','-1');
gexOptions('pareto','cutoff_cpk','1.33');
gexOptions('pareto','cutoff_signature_failure','20.0');
gexOptions('pareto','cutoff_failure','10');	// Display top 10 failing tests.
gexOptions('pareto','section','3');		// Report Cp and Cpk pareto

 

Section: binning

Field Value / Description
section Defines if the binning section is to be included in the report generated. Value must be one of:
enabled : section is included in the report
disabled : section is not included in the report
 
computation Defines how the binning percentages and values must be computed; can be one of the following:

samples makes the binning computed from the data samples only, and all summary records are ignored. This selection is very convenient when the summary is corrupted or was not reset prior to start a production lot.

summary  this is the default mode: binning counts are computed reading the data files summary records (if any), and sampled data otherwise. Use this mode unless you process wafer map data that include retests. In such situation, this mode may count retested dies, and the binning/yield values wouldn't be totally accurate

wafer_map makes the binning computed from the wafer map data only. Under this mode, only the last bin of each die retested is counted, making the binning chart and yield information always accurate. If you select this mode, you must activate the wafer map report generation otherwise no binning chart will be created (as no wafermap data would be found).
 
section_name Lets you define the section name in the value field.

Example:

// Makes Binning computed from summary data (default)
gexOptions('binning','section_name','Binning Summary');
gexOptions('binning','computation','summary');


Section: adv_histogram

Field Value / Description
chart_size Defines the size of the histogram image created in the report (HTML, Word, PDF, etc). Can
be one of the following:
small for small image or mosaic if multiple histograms are reported
medium for medium image

Section: adv_pearson

Field Value / Description
cutoff Defines the lowest Pearson's absolute correlation value a test must have to be part of the report. Any correlation value higher means test strongly correlates. Default value is 0.8; and in no event should it be lower than 0 or higher than 1.0

It is usually considered that tests with a correlation value of 0.7 or higher strongly correlate; therefore favor any value from 0.7 and higher.
sorting Specifies which field type to sort the generated table. Can be one of the following:
test_name to sort by ascending test name
pearson to sort by descending Pearson's correlation ratio (first show tests with the highest correlation value)

Note: Only one sorting mode can be selected at a time. Also when comparing files or groups of files, pearson report is done on the first group only.

Example:

// Set 'pearson's correlation' Advanced report options: show all tests with a correlation

gexOptions('adv_pearson','cutoff','0.8');
gexOptions('adv_pearson','sorting','test_name');

 

Section: toolbox

Field Value / Description
csv_sorting Defines how the CSV export parser (and CSV editor) displays tests. Two options are valid:

test_id : This is the default mode, where tests are sorted in increasing order (first column show test with lowest test#)
flow_id : This mode makes tests appear sorted by execution order (testing sequence).
