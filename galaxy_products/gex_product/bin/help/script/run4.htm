 When calling the GEX Quantix Examinator program with a script as an argument, you can specify runtime
options. Runtime options let you decide how GEX must behave while processing the script.

Syntax to launch a script with options:

gex -options scriptfile

The scriptfile extension may be omitted if it is .csl, the additional parameters depend on your script file.
The -options parameter  (case sensitive, order of options doesn't matter) lets you provide run time options to the Examinator such as:

  • Specify a Script to be executed at launch time of the application
  • Hide application (minimize in icon) while running the script
  • Close the application once script execution is completed

Important notice: If you plan to have the Examinator run script in hidden mode...ensure your script is not requesting any information to input (i.e.: do NOT include calls to sysPrompt !). Failing to do so would make the GEX run forever, until you kill the process!

You can combine any of the following non-case sensitive options:

option Description
W Forces GEX Welcome page to display, allows you to change the running mode and/or change the server name/Socket port#. Depending of your license, you may then run some or any of the following modes:
  • Evaluation
  • Client/Server
  • Standalone
S Examinator has to run a script at launch time.
H Examinator is Hidden (minimized) when running script. If this option is not specified, the Examinator application is visible during the script execution.
Note: This option requires the S option to be set
C Examinator is Closed once script execution is completed. Useful when you want a script
to be executed periodically and have the Examinator launched every X minutes using a
timer application (e.g. crontab, task windows, etc.).
Note: This option requires the S option to be set

 

Example1: A script launched with GEX hidden, and closed once script is completed: 

gex -HCS /export/home/myscripts/hello.csl

Example2: A script launched with GEX. Application is visible and report is shown once script is completed: 

gex -S /export/home/myscripts/hello.csl
Note: If the script created a report, only HTML and CSV formats will be accepted.
To have reports created in PDF, Word or PowerPoint format you need to run this
script from the Yield-Man scheduler instead (contact Quantix to learn more about Yield-Man)
