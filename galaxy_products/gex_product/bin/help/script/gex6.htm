gexReportType(
  const section,      // section entry name
  const type,         // report type
  const value,        // field value
  const customvalue)  // custom value used in some report types

Sets the type of report to generate (wafer map, histogram, etc...). Fields are not case sensitive.

For a complete example, take a look at the ready-to-use scripts templates:

Note: Advanced reports always display one report per page except when the chart size is set to auto; in
this case a mosaic of small charts are displayed on a same page (see gexOptions for more details)

Example:

// Create a wafer map of Hardware bins, 
gexReportType('wafer','hard_bin');
// Because parameters are not case sensitive, it is the same as: 
gexReportType('Wafer','hard_bin');
// or even the same as...
gexReportType('wafer','HARD_BIN');

Here is the list of report sections you can configure using this script function:

Section Description
stats Define the type of Test Statistics table to generate
wafer Define the type of wafer map to generate
histogram Define the type of histogram to generate
adv_histogram Define the advanced report section to be a histogram bar chart
adv_trend Define the advanced report section to be a trend control chart
adv_correlation Define the advanced report section to be a correlation (XY-scatter) chart
adv_production_yield Define the advanced report section to be Production Yield charts
adv_boxplot Define the advanced report section to be a Gage R&R - box plot chart
adv_datalog Define the advanced report section to be a datalog listing
adv_pearson Define the advanced report section to be the list of tests that highly correlate (based on Pearson's correlation formula)
adv_pat Define the advanced report section to display the PAT history report created into the test data file by Quantix PAT-Man or Examinator-PAT (post-processing PAT). Allows to review all outliers detected, their value, partID, etc...
adv_my_report Define path to a template file (for creating custom reports & sections).
adv_global_options Defines global options that can be used by any of the advanced reports
Note: Only one advanced section can be selected per report generation. If more than one advanced section is set using multiple calls to the gexReportType function, only the last call takes effect, previous calls are simply ignored.

 

Section: stats

Type Value / Description
disabled Specifies not to create the test statistics report section. No value parameter required. 
all Test statistics table includes all tests. No value parameter required. 
fails Test statistics table only includes tests that failed at least once. No value parameter required. 
outliers Test statistics table only includes tests that have outliers, result depends of the outlier removal definition you've made. No value parameter required. 
tests Test statistics table includes a test or a range of tests. The value parameter must set to the test number to process, for example: 1045 to 1055.
Alternatively, the value parameter can be a file name that holds your test list (see file format in the Appendix 1 of the visual tour help page)
cp Test statistics table includes all tests with a Cp lower or equal to a given value. The value parameter must be set to the Cp limit you want, for example: 3.0
cpk Test statistics table includes all tests with a Cpk lower or equal to a given value. The value parameter must be set to the Cpk limit you want, for example: 1.33

Example1:

// Create a test statistics table of failing tests only 
gexReportType('stats','fails');

Example2:

// Create a test statistics table of tests with a Cpk <= 1.33
gexReportType('stats','cpk','1.33');

Example3:

// Create a test statistics table of tests specified in a configuration file
gexReportType('stats','tests','c:/scripts/tests.txt');

See also the ready-to-use script: statistics.csl

 

Section: wafer

Type Value / Description
disabled Specifies not to create the wafer map report section. No value parameter required. 
soft_bin Create a wafer map using the software binning information. No value parameter required. 
hard_bin Create a wafer map using the hardware binning information. No value parameter required. 
param_over_limits Create a wafer map using parametric test results, and using the test limits range for the color spectrum. The value parameter must be set to the test number to process, for example: 1045
param_over_range Create a wafer map using parametric test results, and using the test results range for the color spectrum. The value parameter must be set to the test number to process, for example: 1045
stack_soft_bin Create a stacked wafer map combining all wafers in the group. The value parameter must be set to the list of bins to merge, for example: 1-10,12
stack_hard_bin Create a stacked wafer map combining all wafers in the group. The value parameter must be set to the list of bins to merge, for example: 1-10,12
stack_param_over_limits Create a stacked wafer map using parametric test results of all wafers in the group, and using the test limits range for the color spectrum. The value parameter must be set to the test number to process, for example: 1045
stack_param_over_range Create a stacked wafer map using parametric test results of all wafers in the group, and using the test results range for the color spectrum. The value parameter must be set to the test number to process, for example: 1045

Example1:

// Create a wafer map using software binning, 
gexReportType('wafer','soft_bin');

Example2:

// Create a gradient wafer map of parametric test 1045 over its limits
gexReportType('wafer','param_over_limits','1045');

Example3:

// Create a Stacked wafer map of software binning: Bin1, 
gexReportType('wafer','stack_soft_bin','1');
 

See also the ready-to-use script: wafermap.csl

 

Section: histogram

Type Value / Description
disabled Specifies not to create the histogram report section. No value parameter required. 
advanced Forces the standard histogram section to use the advanced histogram plotting engine instead of plotting the standard 3D predefined histogram bars. This mode is more resource intensive and slower as each histogram image has to be fully built then saved to disk as an individual image. The benefit of using this mode is that markers and all the other options relative to the advanced histogram section can be leveraged. No value parameter required. 
test_over_limits Create the histogram report section showing charting test results over the test limits space. The value parameter can be one of:
 all to have all tests histogram charts created
 XX where XX is the specific test histogram to create
 XX to YY to create the histograms of the tests XX to YY, e.g.: 1040 to 1060

Alternatively, the value parameter can be a file name that holds your test list (see file format in the Appendix 1 of the visual tour help page)
cumul_over_limits Same as above, except it creates cumulative histograms.
test_over_range Same as above, except it creates histograms over test results range instead of test limits.
cumul_over_range Same as above, except it creates cumulative histograms.
adaptive Adjust the charting dimensions to make all data points and test limits visible

Example 1:

// Create the histogram of ALL tests. Chart over test results
gexReportType('histogram','test_over_range','all');

Example 2:

// Create histograms for tests 1040 to 1060, Chart over test limits
gexReportType('histogram','test_over_limits','1040 to 1060');

Example 3:

// Create histograms for tests defined in a configuration file
gexReportType('histogram','test_over_limits','c:/scripts/tests.txt');

See also the ready-to-use script: histogram.csl

test_over_range Same as for histogram (test list or file name that holds the test list)
cumul_over_range Same as for histogram (test list or file name that holds the test list)

Section: adv_trend

Type Value / Description
disabled Specifies not to create the advanced report section. No value parameter required. 
test_over_limits Same as for histogram (test list or file name that holds the test list)
test_over_range Same as for histogram (test list or file name that holds the test list)
adaptive Adjust the charting dimensions to make all data points and test limits visible
difference Build the trend chart resulting from the difference between two test values.
The value parameter must be the two tests to compare with a coma separator. 
E.g.: 1040,1060    will create the trend chart of Test 1040 - Test 1080
mean Creates the trend chart plotting the mean value for all given tests (one point per sub-lot). The value parameter can be one of:
 all to have all tests histogram charts created
 XX where XX is the specific test histogram to create
 XX to YY to create the histograms of the tests XX to YY, e.g.: 1040 to 1060

Alternatively, the value parameter can be a file name that holds your test list (see file format in the Appendix 1 of the visual tour help page)
sigma Creates the trend chart plotting the sigma value for all given tests (same syntax as for the mean)
cp Creates the trend chart plotting the cp value for all given tests (same syntax as for the mean)
cpk Creates the trend chart plotting the cpk value for all given tests (same syntax as for the mean)
soft_bin_sublots Build the trend chart of the software binning list specified in the value parameter. Each plot is the sub-lot yield level for the given binning selected.
The value parameter must be the bin list (or file name that holds the bin list)
E.g.: 2 to 7,9    will create the trend chart of combined bin 2 to 7 and 9.

Alternatively, the value parameter can be a file name that holds your test list (see file format in the Appendix 1 of the visual tour help page)
soft_bin_parts Build the trend chart of the software binning list specified in the value parameter. There is one plot per part tested (unless data file was generated with sampling...). The curve shows the evolution of the instant yield level over all the samples. It always starts from 0, then each point is the resulting current yield level considering all previous point. Therefore, the last point on the curve shows the exact yield level reached at the end of the test batch
The value parameter must be the bin list (or file name that holds the bin list)
E.g.: 2 -7, 9-12    will create the trend chart of combined bin 2 to 7 and 9 to 12.

Alternatively, the value parameter can be a file name that holds your test list (see file format in the Appendix 1 of the visual tour help page)
hard_bin_sublots Same as above soft_bin_sublots but applies to Hardware binning instead.
soft_bin_parts Same as above soft_bin_parts but applies to Hardware binning instead.
soft_bin_rolling Plots two yield charts at once: the first chart plotted in the software binning yield level after each device tested; the other chart plotted shows the rolling yield level computed over 200 consecutive parts (you can change this number with the function gexOptions). As the rolling yield is computed over windows of 200 consecutive parts, it is very reactive to yield change and can drop or increase much faster than the overall yield.
hard_bin_rolling Same as above soft_bin_rolling but applies to Hardware binning instead.

Example:

// Create Trend charts for tests 1040 to 1080, chart over test limits
gexReportType('adv_trend','test_over_limits','1040 to 1080');
// Create a Trend chart for Test 1040 - Test 1080 
// (difference between T1040 and T1080)
gexReportType('adv_trend','difference','1040,1080');
// Create a Trend chart for Binning 1 over all parts tested
gexReportType('adv_trend','soft_bin_parts','1');
// Create a Trend chart for Bins specified in a configuration file
// See file format in the Appendix 1 
gexReportType('adv_trend','soft_bin_parts','c:/scripts/binlist.txt');

See also the ready-to-use script: adv_trend.csl

 

 

Section: adv_correlation

Type Value / Description
disabled Specifies not to create the advanced report section. No value parameter required. 
test_over_limits That value lists the pairs of tests to plot on the scatter chart over a windows using the tests limits (first test is for the X axis, next test test is )Y.

Alternatively, the value parameter can be a file name that holds your test list (see file format in the Appendix 1 of the visual tour help page)
test_over_range Same as above, but charting over the window of data samples (Min & Max)

Example1:

// Correlation (XY-Scatter) between test 1040 and test 1080, using test results
gexReportType('adv_correlation','test_over_range','1040 1080');


Example2:

// Correlation (XY-Scatter) between test 10.45 and test 10.57, using test limits
gexReportType('adv_correlation','test_over_limits','10.45 10.57');


Example3: Two Scatter plots (two pairs of tests)

// Scatter tests 1000 vs 1020 and test 2000 vs 2050
gexReportType('adv_correlation','test_over_limits','1000 1020 2000 2050');


Example4: Scatter plots from a list of tests pairs defined in a file (see file format specs)

// Scatter tests defined in a file
gexReportType('adv_correlation','test_over_limits','c:/scripts/filelist.txt');
 

See also the ready-to-use script: adv_correlation.csl 

 

 

Section: adv_production_yield

   
Type Value / Description
title Specifies the chart title
sublot Tells to create a Yield/Volume chart where each plot is a sublot (or wafer). The value field specifies the Bin# or Bin list to focus on (typically Bin1)
lot Tells to create a Yield/Volume chart where each plot represents a full lot. The value field specifies the Bin# or Bin list to focus on (typically Bin1)
group Tells to create a Yield/Volume chart where each bar represents a group (or query). Typically used when comparing lots / datasets. The value field specifies the Bin# or Bin list to focus on (typically Bin1)
daily Tells to create a Yield/Volume chart where each plot represents all lots and sublots tested the same day. The value field specifies the Bin# or Bin list to focus on (typically Bin1)
weekly Tells to create a Yield/Volume chart where each plot represents all lots and sublots tested the same week. The value field specifies the Bin# or Bin list to focus on (typically Bin1)
monthly Tells to create a Yield/Volume chart where each plot represents all lots and sublots tested the same month. The value field specifies the Bin# or Bin list to focus on (typically Bin1)
yield_axis Specifies the view port to be used for the Yield axis. Possible values are:
'0-100' (default): view port is from 0% to 100%
'0-max': view port is from 0% to max yield
'min-100': view port is from min yield to 100%
'min-max': view port is from min yield to max yield
volume_axis Specifies the view port to be used for the Volume axis. Possible values are:
'0-max' (default): view port is from 0% to max volume
'0-customvalue': view port is from 0% to volume specified in customvalue arg
'min-max': view port is from min volume to max volume
'min-customvalue': view port is from min volume to volume specified in customvalue arg
yield_marker Specifies the yield position at which a red marker should be set on the graph. possible values are between 0 and 100, default is 0 (no marker).

Example:

// Sets the title of the Yield/Volume chart
gexReportType('adv_production_yield','title','Weekly Yield/Volume Chart');
// Display the Weekly Bin1 yield over the query period
gexReportType('adv_production_yield','weekly','1');

// Set Yield axis view port to be min-100
gexReportType('adv_production_yield','yield_axis','min-100');

// Set Volume axis view port to be 0-20000
gexReportType('adv_production_yield','volume_axis','0-custom','20000');

// Set Yield marker at 80%
gexReportType('adv_production_yield','yield_marker','80');


Tip: for accurate yield value when retest is involved, make sure the binning is computed over the wafermap/stripmap and not from samples or summary.

 

Section: adv_boxplot

Type Value / Description
disabled Specifies not to create the advanced report section. No value parameter required. 
tests Create the Gage R&R - box-plot report section . The value parameter can be one of:
all to have all tests boxplot charts created
XX where XX is the specific test boxplot to create
XX to YY to create the boxplots for tests XX to YY (e.g. 1040 to 1060)

Alternatively, the value parameter can be a file name that holds your test list (see file format in the Appendix 1 of the visual tour help page)

Example:

// Includes all tests in the Gage R&R - box-plot report
gexReportType('adv_boxplot','tests','all');
// Gage R&R - box-plot report for the tests listed in the configuration file
gexReportType('adv_boxplot','tests','c:/scripts/testlist.txt');

Section: adv_datalog

Type Value / Description
disabled Specifies not to create the advanced report section. No value parameter required. 
all

Advanced report section will include datalog of ALL tests. No value parameter required. 
fails

Advanced report section will include datalog of failing tests only. No value parameter required. 
outliers

Advanced report section will include datalog of outliers for the given tests in the value parameter.

Alternatively, the value parameter can be a file name that holds your test list (see file format in the Appendix 1 of the visual tour help page)
tests

Advanced report section will include datalog of the specified test list given in the value parameter. Test results, part ID, and BIN results are reported.

Alternatively, the value parameter can be a file name that holds your test list (see file format in the Appendix 1 of the visual tour help page)
tests_only

Advanced report section will include datalog of the specified test list given in the value parameter; only test values are reported (no part ID, nor BIN result).

Alternatively, the value parameter can be a file name that holds your test list (see file format in the Appendix 1 of the visual tour help page)

Example 1:

// Generate datalog of all failing tests
gexReportType('adv_datalog','fails');

Example 2:

// Generate datalog of test results for test 10 to 50
gexReportType('adv_datalog','tests','10 to 50');

Example 3:

// Generate datalog for a test list defined in a configuration file
gexReportType('adv_datalog','tests','c:/scripts/testlist.txt');

See also the ready-to-use script: adv_datalog.csl

 

Section: adv_pearson

The report created lists all tests that strongly correlate (the correlation threshold over which tests are listed can be configured using the gexOptions function).
 

Example:

// Create the list of tests that highly correlate
gexReportType('adv_pearson','','');

 

Section: adv_pat

The report created displays the complete PAT history report created into the test data file by Quantix PAT-Man or Examinator-PAT (post-processing PAT) . Allows to review all outliers detected, their value, partID, etc...

Example:

// Display the PAT Outlier report summary.
gexReportType('adv_pat','','');

 

Section: adv_my_report

This function lets you define the report template file (.grt) to use for creating a custom report. This report template file must be created using the Examinator Wizard available from the Examinator Settings page, when selecting More reports = My Report. Such file includes the list of report sections to create (eg: Histograms, Statistics, Scatter plots,) as well as the layout (overlay charts, split charts, etc). This is the ultimate flexibility for tailored reports.

Example:

// Display the PAT Outlier report summary.
gexReportType('adv_my_report','template','c:/templates/my_report.grt');

 

Section: adv_global_options

This function lets you define global options that can be used by any of the advanced reports.

Type Value / Description
binning Can be hard_bin or soft_bin : allows to define over which binning type the advanced report must be generated.
Note: This scripting function is currently only supported by the Production yield report. All other advanced reports ignore this selection.

Example:

// Production report to focus on hard-bin only.
gexReportType('adv_global_options','binning','hard_bin');
