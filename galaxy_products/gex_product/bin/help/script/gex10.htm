gexToolbox(
  const action,     	// Action type
  const field,         // Parameter field to set
  const value)	       // Field value

This scripting function allows you to automate some of the Examinator toolbox functions.
Note: Fields are not case sensitive.

 

Action descriptions

action Description
stdf_dump Function to perform the ASCII dump of a STDF data file

 

 

Action: STDF_DUMP

 

The stdf_dump action allows to dump STDF files to ASCII. This function supports STDF and compressed STDF (.gz .zip .Z). Note that if the file is compressed, Examinator first decompresses the file into a folder of the same name without the compressed extension 'i.e: file.zip in decompressed in sub-folder ./file/), and the dump is created in this sub-folder too. It is your responsibility to erase such temporary folders when dump files are no longer required. Alternatively, if you define the optional output path, you don't have to perform any housekeeping! Therefore, the output path is strongly recommended when processing compressed data files.

field Description
file Define the STDF file name to dump (and optional output path). Note that if you call multiple times this function, you can trigger multiple dumps at once!
records Defines one of the STDF record to dump. To include multiple records, you must call this function multiple times. Following are the only valid values for the 'value' field:
all  : to default all records to be dumped (this is the default)
clear  : to clear all records
and one of the following STDF records: ATR, BPS, DTR, EPS, FAR, FTR, GDR, HBR, MIR, MPR,  MRR, PCR, PGR, PIR, PLR, PMR, PRR, PTR, RDR, SBR, SDR, TSR, WCR, WIR, WRR
style Defines if the records are dump in the short form (record name only) or detailed form (all record fields listed). Valid values for the 'value' field are:
short : for only listing record titles
detailed : for detailed records, with all fields listed (this is the default)
size Defines the maximum dump size in Kbytes (convenient when only the first 500K of first few mega bytes of dump is needed). Typical  values for the 'value' field are:
0 (or any negative value) : no limit defined, the full dump is created (this is the default)
500 : for 500 Kbytes of dump
1000: for 1Mbytes of dump
 
consecutive Tells if ignore or display consecutive reports f a same type (eg: when many PTR records are found, you can instruct to only display the first one or all PTRs ). Valid values for the 'value' field are:
yes : to display all records, even consecutive ones (this is the default)
no: To only  display the first record when multiple records of a same type are detected.
dump Instructs to dump the file, using all options defined in previous calls.

 

Example1:
// Dump STDF file: ALL STDF records, detail all fields
gexToolbox('stdf_dump','file','c:/files/file.std');
gexToolbox('stdf_dump','dump');	// Trigger dump now!

 
Example2:
// Dump STDF file in a specific output folder: ALL STDF records, detail all fields
gexToolbox('stdf_dump','file','c:/files/file.std','c:/output_folder');
gexToolbox('stdf_dump','dump');	// Trigger dump now!

 
Example3:
// Dump 2 STDF files: ALL STDF records, detail all fields
gexToolbox('stdf_dump','file','c:/files/file_1.std');
gexToolbox('stdf_dump','file','c:/files/file_2.std');
gexToolbox('stdf_dump','dump');	// Trigger dump now!

Example4:
// Dump STDF file: MIR and PRR records, up to 500 Kbytes of dump
gexToolbox('stdf_dump','file','c:/files/file.std');
gexToolbox('stdf_dump','records','clear');	// clear all STDF records to dump
gexToolbox('stdf_dump','records','MIR');	// Dump MIR record
gexToolbox('stdf_dump','records','PRR');	// Dump PRR records
gexToolbox('stdf_dump','size','500');		// Dump up to 500 Kbytes of text
gexToolbox('stdf_dump','dump');			// Trigger dump now!
