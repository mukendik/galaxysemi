gexParameter(
  const parameter,     // Test/Parameter name
  const name,          // Parameter name
  const field,         // Parameter field to set
  const value,	       // Field value
  const label          // Label assigned to field.

This scripting function lets you insert your own markers you want to see in the charts generated. It also lets you define the chart section to display in the report for advanced reports (offers custom zooming for advanced histograms, trends and scatter plots). This function allows to add markers to be plotted when drawing the Advanced Histogram or Trend charts. Another typical application is to force histograms to display a predefined area; making histogram reports comparisons easier.
Note: Fields are not case sensitive.

Example:

// Add marker at position 3.0V : 2 pixel wide line, color: Red (RGB = 255, 0, 0)
gexParameter('1234','Output voltage','marker','3 2 255 0 0','Target voltage');
 

Fields descriptions

Fields Description
parameter This is the parameter/ test number (eg: 1000 or 1000.1 if a pinmap number exists).
name This is the parameter name (test name)
field This is the parameter field to overwrite or create. Must be one of:
Marker to set a line marker to plot on the parameter advanced histogram or trend chart.
Viewport to define on which charting area to focus (lets you define the zoom-in charting section).
value This is the value to force for the field.

If field is set to Marker, then this string includes the position where the marker must be plotted as well as the line width and color (RGB format). The marker position must be normalized (eg: 20000  for 20Khz). Marker string format is:
<position> <line_width> <Red> <Green> <Blue> where the RGB (red green blue) are the color channel level ranging between 0 and 255. See example2  below

If field is set to Viewport, then this value field includes the charting area to display. It must at list define the LowData and HighData values (used for X axis for Histograms , Y axis for trend charts, data axis for scatter plot (X or Y, depends how the test position in the scatter plot list)); and optionally may also include the LowCategory and HighCategory values (used for Y axis on histograms, or X axis of trend charts; ignored in scatter plots). The numbers must be normalized (eg: 2e-3  for 2mV). See examples 3 to 5 below.
label If field is set to Marker, this label is the marker label text to write on the chart when drawing the marker line. 
Example1:
// Add marker at position 20.3Khz : 2 pixel wide line, color: Red (RGB = 255, 0, 0)
gexParameter('1000','RF test ABC','Marker','20300 2 255 0 0','20.3Khz marker');
Example2:
// Forces to have Histograms on the 10mV-20mV results space 
gexParameter('1000','RF test ABC','Viewport','0.01 0.02','');
Example3:
// Trend charts over the 10mV-20mV results space (first two viewport values
// are set to 0 as they represnet the X axis and are ignored in Trend chart mode.
gexParameter('1000','RF test ABC','Viewport','0.0 0.0 0.01 0.02','');
Example4:
// Forces to have Histograms on the 10mV-20mV results space, and Y distribution
// percentage to range between from 0% and 25%
gexParameter('1000','RF test ABC','Viewport','0.01 0.02 0 25','');
Example5:
// Forces to have scatter plot on the the 10mV-20mV results space
gexParameter('1000','RF test ABC','Viewport','0.01 0.02','');
gexParameter('1020','RF test YRO','Viewport','0.07 0.09','');
