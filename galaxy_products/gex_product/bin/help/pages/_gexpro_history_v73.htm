GEXYM-30 H7433: Shift Analysis - add a new shift report called Actual Value Shift The purpose is to implement in C++ as usual. As a quality engineer, I want to compare data before and after reliability tests relative to the actual value of the baseline data point (before) rather than relative to the width of the test limits (HiLim - LowLim). This is important because measurements in the nano range could have significant shifts (e.g. 40 nA to 100 nA) that are not significant relative to the width of test limits. Measuring shifts relative to the actual value of the baseline data point would correctly identify the nA shift as significant. The algorithm for the actual value shift is below: Given:
t0 = measurement before reliability trail;
t1 = measurement after reliability trial; t1 Actual Value Shift % = (Data@t1 – Data@t0) / (Data@t0);
t2 Actual Value Shift % = (Data@t2 – Data@t0) / (Data@t0); Mel suggests that we give users a choice to measure the shift relative to actual value of baseline measurement or relative to width of test limits with shift relative to actual value as the default. Also go to links to see:
  • email conversations with customer in SalesForce
  • technical details in wiki page
