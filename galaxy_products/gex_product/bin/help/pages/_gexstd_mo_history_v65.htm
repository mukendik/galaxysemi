V6.5.14 (31 July 2012) release history (Top)
Yield-Man:
  • Case 6460 - Yield-Man crashes when using Reports Center.

V6.5.13 (27 July 2012) release history (Top)
Yield-Man:
  • Case 6453 - atd file is being treated as if there is a missing mrr record: atdf file that is complete with an mrr record is being rejected for having a missing mrr record.
  • Case 6427 - When sorting by flow ID in the CSV dump, we get additional commas at the end of each part results: In the CSV toolbox dump option we can sort by test id, or by flow id. Sorting the CSV by flow id lead to unnecessary commas at the end of each part results. This is not expected by the customer who uses a very old version (6.1B155): we want them to move to 6.5 but this issue is currently preventing us from doing so. The goal being to show them that we can get the exact same output in 6.5 as in 6.1B155 This issue has been found in YieldMan and uses the automated file conversion features, but it can be recreated with GEX only since it comes from the toolbox CSV dump itself.
  • Case 6393 - data pump tasks with scan subfolders delete files: when you create a datapump task with the scan subfolders box checked and you have delayed files set to go to a folder, files go missing.
  • Case 6392 - creating a task from another task (copy) deletes all rdbs from pulldown: database list is eliminated when a task is created from a copy.
  • Case 6388 - spil file not recognized by gex parser: file that is supported by gex is not recognized.

V6.5.11 (27 June 2012) release history (Top)
Yield-Man:
  • Case 6362 - SPEA parser: update for SPEA340 and SPEA372 format: The SPEA tester has a variation in output between * the AtosC Datalogging - Specifications (Release : 1.40) * the SPEA340 * the SPEA372. Update to the parser to accommodate all this 3 formats..
  • Case 6357 - Linux bug causing Yield-Man to close unexpectedly: A very large .tar.gz file should be rejected but instead causes ym to close unexpectedly.
  • Case 6300 - Gross die not updated correctly: With V6.5, the Gross die is not updated as expected when a wafer is inserted through several files, and no specific gross die overload method is used. The gross die takes the value of the nb_parts from the first file inserted.

V6.5 (8 March 2012) release history (Top)
Yield-Man:
  • Case 5975 - Gex-Email deletes attached files after a delay: Gex-Email deletes files attached to sent emails only after these files are at least 1 hour old.
  • Case 5971 - Intermediate Consolidation at Wafer Sort level: Support intermediate consolidation based on test conditions at Wafer Sort level.
  • Case 5787 - Make YmAdminDb server mandatory: YieldMan must now be used with the YmAdminDb server. This option allows YieldMan to access all new functionnalities from the YmAdminDb server.
  • Case 5696 - Allow an option to use the max start time as the test date in the consolidation table for every lot: Allow an option to use the max start time as the test date in the consolidation table for every lot. This will allow the automated yield report for the last 7 days to also be included when new data arrives (but when 7 days after the first files start time) and shows up in the weekly gex automated report for that product.
  • Case 5693 - Insert v4 2007 stdf files into db with warning: allow the user to insert v42007 files into YM db and give a warning.
  • Case 5658 - Update FETTEST parser: update the FETTEST parser so that wafer test FETTEST file testing HVM data can use a different bin mapping and / or promis data file than what is specified in today's env variables.
  • Case 5419 - Add 'Save to file' and a 'Save to clipboard' buttons in the SYA compute&simulate interface: Add 2 buttons in the SYA compute&simulate interface, 1 to export the SYA computation/simulation results into a CSV file, the other to save the SYA computation/simulation results into the clipboard in CSV format and offer to launch the spreadsheet application.
  • Case 5297 - Automatic/manual GEXDB connections: Add an option to specify for each GEXDB link, whether the connection at startup should be automatic or manual.
  • Case 5213 - Gex-FTP: add an option to set the max number of files to download before going to the next profile: To ensure rotation among the various FTP servers, add an option to set the max number of files to download before going to the next profile. To be set in 'General settings'
  • Case 5212 - Improve Yield-Man task management: Improve the way the user can interact with Yield-Man tasks: 1) To force the execution of a given task, add 'Execute task now' menu entry when the user right-clicks on a task. This would force the execution of this task ONLY. This new menu entry would be available only when the scheduler is paused. 2) The 'Execute task now' button should be renamed to 'Execute all tasks now', and would force the execution of all tasks. 3) Enable multi-select of tasks to enable/disable several tasks at the same time
  • Case 5208 - Enhance Gex-Ftp so that if the server cuts it off, it tries to reconnect before going to the next profile: In 'General Settings' add an option to allow the Gex-ftp to attempt reconnect to the ftp server X times after: being cut-off by ftp time-out on the server side, being unable to connect to the ftp server, being cut off while transfering a file
  • Case 5198 - Information about Quantix database update DIALOG: When user wants to update a Quantix database, YM displays now the 'Quantix Database Update' dialog to indicate the different actions implemented during the update. Then the user can choose to do the update or not.
  • Case 5192 - Please provide a tool to specify the order of file loads into YM: Please provide a tool to specify the order of file loads into YM. Options would be: 1) FIFO - First In First Out. 2) LIFO - Last In First Out. 3) Alphabetical by file name.
  • Case 5189 - [Yokogawa parser] test numbers repeated with different test names: test numbers are being repeated for tests with different test names creating confusion for the user.
  • Case 5040 - Allow for the user to output to CSV the SYA simulation and/or limits: Add an output to CSV button on the SYA task simulation and limits sections.
  • Case 4976 - Provide the ability to port SYA simulation results to CSV: Please provide the ability to port the SYA simulation results to a CSV file for reading into a spreadsheet.
  • Case 4950 - Pearson's reports enhancement : boxplot-whisker chart: add option to view in box-whisker plots instead of in scatter plot.
  • Case 4901 - SYA enhancement for display of median and robust sigma: When the user selects median+/-N*robust sigma, both the median and the robust sigma should be in the display.
  • Case 4653 - Log system enhancements for 6.5: Examinator/Yield-Man applications now log messages using a GEXLOG log module, which supports the syslog standard severity levels, and supports several output formats (rtf, xml, sqlite, syslog TCP/IP).
  • Case 4597 - Multi insertion support from several nodes: Allow several Yieldman nodes running on the same GexDb database to insert data into the same testing stage.
  • Case 2761 - Allow a bin Yield-Man alert where it looks at the Yield difference between sites:
  • Case 5968 - No date fields in consolidated table if meta_data on start_t field defined: No date fields in wt_consolidated_wafer table if meta_data on start_t field defined (Y for consolidated_field).
  • Case 5966 - Force testing stage to Etest doesn't work: When using the gexdbkeys ForceTestingState to E, the file was rejected by Yield-Man (invalid WaferId).
  • Case 5936 - Outlier removal don't works with space in the product ID: When creating an outlier removal task, if there is a space in the product ID the task won't be executed on this product.
  • Case 5926 - Yield or parameter Alarms cannot be generated: When pumping in a specific file, we cannot have the yield monitoring or parameter monitoring tasks create emails if we have space in the product ID.

  • Case 5331 - Error while creating database link not handled correctly: While creating a database link, if the name of this database link is already used for another database link. No error message appears, but the new link will not be created.
