V7.7.32 (Top)
Enhancements
Case Number Subject Description
GCORE-17609 The ASE Statec Summary Parser (.sum) has been enhanced to better support files with different column alignments The ASE Statec Summary Parser (.sum) has been enhanced to support files using different alignments in the "<< SORT PLAN REPORT >>" section. Instead of expecting the columns to be aligned with the columns in the header row, each row is parsed using the space separator. The 2 fields of interest ("NAME" and "COUNT") are extracted using the 2nd field and the 2nd to last field respectively.
Bug fixes
Case Number Subject Description
GCORE-17753 Wrong error message when trying to read a corrupted STDF file When analyzing a corrupted STDF file with Examinator, an error message gets displayed stating Examinator failed to read the Bin mapping file, instead of an error stating the file format is invalid.
GCORE-17710 The "Speed optimization" option is obsolete The speed optimization option ("Speed optimization..."->"Database queries..."), allowing to use summary data instead of Stdf raw data, was implemented for the initial flat file DB. This DB type is not supported anymore, but the option is still available. It is confusing because the user may think this applies to TDR extraction, which is not the case. The option has been removed.
GCORE-17618 Characterization hierarchy feature is broken when filtering on test conditions When filtering on test conditions, the rebuild hierarchy feature of the Characterization wizard does not output anything if a filtered condition is removed from the list of hierarchy conditions in the left pane of the interface. It is expected that the hierarchy gets rebuilt even if a field on which a filter is applied is not part of the conditions to be included in the hierarchy.

GCORE-17610 Application crashes when analyzing a SkyWorks NP CSV file with no raw results The application crashes when analyzing a SkyWorks NP CSV file that has no raw results, but only the header and metadata information.
GCORE-17482 More Reports combo grayed After using the "Shift Analysis" wizard, which forces the "More Reports" combo box to "Shift report", the first time you try to generate a standard report (ie using "Analyze one file"), the "More Reports" combo is still grayed with "Shift report" selected.
GCORE-17470 Execs and Fails count inconsistencies when a test has multiple result instances in the same run When a test has multiple result instances in a same run (same test number, same test name, multiple PTR records), the exec and fail counts in the statistics table are sometimes inconsistent, depending on the options. For instance, if the option is set to keep all results, the exec count will include all results, but the fail count includes only maximum 1 fail per run, even if more than one result is fail in that run for that test.
GCORE-17411 When comparing files with the option to compare all group combinations, the computed max shift is wrong When comparing files, the statistics table can display the inter-group shift for some stats (mean, sigma, cpx). With the option "Test statistics..."->"Compare calculations to include" set to "Reference only", it will only display the shift of each group with the reference group, whereas if this option is set to "All group combinations", the table will display the shift of all groups against each group. If including advanced charts in the report, the max shift displayed in the statistics table above the chart is wrong if the option is set to "All group combinations". It is one of the group to group shifts, but not necessarily the max shift.
GCORE-16976 Examinator crashes when converting some Eagle Test Data files (.log) When converting some Eagle Test Data Files (.log), Examinator crashes. This is due to new line types in the format that are not correctly ignored by the parser and cause Examinator to crash.
GCORE-16195 Despite being disabled from within the options, the Binning section is still present in HTML reports When the user disables the Binning section in Examinator's options, the Binning report section is still generated if the output format is HTML. It is expected that when the user clicks on the "Binning" link in the generated HTML report, a page gets displayed specifying that such report has not been requested.
GCORE-15234 Confusing scroll bars in the "Part results" interactive table The table displayed in the "Part Results" UI has a vertical scroll bar, but as the table is not adapting to the window size, this scroll bar is sometimes not visible. The user needs to use the window's horizontal scroll bar to see and have access to the table's vertical scroll bar. Moreover, the containing window's vertical scroll bar may mislead the user with the impression it is the table's scroll bar.
GCORE-13502 When extracting data from the TDR, Examinator always extracts the Gross-Die, which may generate an inaccurate Binning Summary When extracting data from the TDR, Examinator always extracts the Gross-Die, even if it has not been set through stored procedures or the config.gexdbkeys configuration file, which may generate an inaccurate Binning Summary.
GCORE-11502 Inappropriate "nan" results in Gage R&R CSV reports When generating the Gage R&R report using a CSV output, some "nan" values are output in some columns (ie AV column) when the corresponding value cannot be computed. "Nan" (not a number) is a technical notation. It is expected that the report outputs "n/a" (not available) instead.
GCORE-3991 When working with datasets and using the "Pick list of parameters..." button, the "Pin name" column is useless With Examinator-Pro, when using a Database Analysis, and using the "Pick list of parameters..." button in the "RDB options" tab to select the parameters to be extracted, the dialog box listing available tests includes a "Pin name" column. This column is always empty and Examinator does not allow filtering on specific pins, it only allows filtering specific tests (including all their pins). This column is useless and confusing. It has been removed.

V7.7.31 (Top)
Enhancements
Case Number Subject Description
GCORE-17460 A feature has been added to allow the user to filter test results at loading time A new feature has been added that allows the user to filter the test results at loading time based on a set of pre-defined statistical filtering formulas, such that the statistics and charts do not get affected by extreme values or aberrations. The feature can be enabled through the "Data processing options" in the "Options" tab, by setting the entries "Data cleaning mode" and "Data cleaning value". For more information, refer to the on-line documentation or to the documentation available in the help center. Both are accessible from within the "Help" tab.
GCORE-17451 SPM deprecated The SPM feature is not supported anymore. The related SPM UI has been removed from the product, and the documentation updated.
Bug fixes
Case Number Subject Description
GCORE-17615 The Options tab is not refreshed when a profile CSL file is loaded When loading a profile CSL file, the UI in the options tab is not refreshed, so that the options stored in memory may not be aligned with the ones shown in the UI, and a generated report will not necessarily reflect options shown in the Options tab.
GCORE-17548 On Linux, impossible to save selection to clipboard for interactive tables Within the interactive tables, when selecting rows and right-clicking, there should be a "Save FULL table to clipboard" and a "Save selection to clipboard" entry in the popup menu that allows copying the selected rows to the clipboard. These entries are missing on Linux.
GCORE-17481 Test List edit box displayed even if "All Tests" is selected In the "Settings" tab, if in the "More Reports" section you select "Histogram" or any other report that allows to select a test list, the test list edit box and picker are displayed to let you select your test list. If you switch back to "All tests", these controls disappear as expected. If you change the chart viewport (ie from "Chart over test results" to "Chart over test limits"), the controls re-appear, even if you are still on "All tests".
GCORE-17476 Rounding errors impact on pass/fail decision when values are equal to a limit (ATDF) When inserting an ATDF file into a TDR and extracting the corresponding data, in some cases, if a value is equal to one of its limit, and has the flag to use non-strict limits (test pass if value equals limit), and has a value expressed in scientific notation in the original ATDF, the extracted value might be considered fail instead of pass. This is due to some rounding issues on measurements when converting from ATDF to STDF prior to the insertion.
GCORE-17468 MPR split option shows an additional test in the stats table When using the "Do NOT merge" option with a file having MPR records, the stats table shows 1 test per "pin", but also 1 test without any pin index (with 0 execs). It is expected that only the per pin tests are displayed.

Case Number Subject Description
GCORE-16842 The FetTest datalog parser has been enhanced to support the latest .dat format specification The format specification for FetTest .dat files has changed. It is now able to support Serial Numbers up to 1 000 000. To get this possible, some minor changes have been made in the specification of some records. Basically, a new flag value is used in an existing byte of the header that indicates whether the file uses large serial numbers. If large serial numbers are turned on, any serial number (in the header and data records) is encoded on 4 bytes instead of 2.
