V3.02 B26 release history (Top)
    Introduced in V7.3
  • Allow BIG index for PinMap records
    • Change tpin_pmrindex field to unsigned for xx_pin_map and xx_mptest_results
  • Change BLOB type to TEXT
    • Change global_files.file_content from mediumblob to mediumtext
  • SYA enhancement
    • Add bin type to SYA tables: wt_sya_set, ft_sya_set, et_sya_set
  • Run/Result enhancement
    • Add part status to RUN tables: wt_run, ft_run, et_run
  • Add primary key to all tables
    • Add testseq to results tables
    • Add some auto increments
    • Change default value of run_id to 0 in ft_dtr, wt_dtr, et_dtr
  • Create SDR tables
    • Following tables have been added to the TDR schema: et_sdr, ft_sdr, wt_sdr
    • These tables are populated with data from the STDF SDR record
    • 1 row is inserted for each site in the SDR group
    • Files with multiple SDR records are now not rejected anymore, they are inserted with a warning
    • If multiple SDR records are found in the file, all SDR data is inserted into the xx_sdr table, but only data from the last SDR gets populated into the legacy xx_splitlot fields
  • Create lot and splitlot level metadata tables
    • Following tables have been added to the TDR schema: xx_splitlot_metadata (primary key is splitlot_id), xx_lot_metadata (primary key is lot_id, product_name)
    • These tables are created only with the fields composing the primary key. They can be customized with additional fields for specific requirements (ie data coming from the MES system)
    • These tables are by default not populated during the insertion process. They need to be populated through customizing one of the various stored procedures called during the insertion process.
  • Add table token
    • This table is used to share specific tokens between Yield-MAN nodes
    • This table used to be created dynamically, but cannot be created dynamically anymore in a SPIDER architecture
