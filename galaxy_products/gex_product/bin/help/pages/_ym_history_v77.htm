V7.7.32 (Top)
Enhancements
Case Number Subject Description
GCORE-17609 The ASE Statec Summary Parser (.sum) has been enhanced to better support files with different column alignments The ASE Statec Summary Parser (.sum) has been enhanced to support files using different alignments in the "<< SORT PLAN REPORT >>" section. Instead of expecting the columns to be aligned with the columns in the header row, each row is parsed using the space separator. The 2 fields of interest ("NAME" and "COUNT") are extracted using the 2nd field and the 2nd to last field respectively.
Bug fixes
Case Number Subject Description
GCORE-17753 Wrong error message when trying to read a corrupted STDF file When analyzing a corrupted STDF file with Examinator, an error message gets displayed stating Examinator failed to read the Bin mapping file, instead of an error stating the file format is invalid.
GCORE-17617 SYA option renamed SYA option for removing abnormal data points during SYA limits computation has been renamed to "Data cleaning during computation"
GCORE-17610 Application crashes when analyzing a SkyWorks NP CSV file with no raw results The application crashes when analyzing a SkyWorks NP CSV file that has no raw results, but only the header and metadata information.
GCORE-17185 Yield-Man scheduled reports running at wrong time A reporting task created with Yield-Man and configured to execute at a specific time will not necessarily execute at that time, depending on when it was last executed.
GCORE-16976 Examinator crashes when converting some Eagle Test Data files (.log) When converting some Eagle Test Data Files (.log), Examinator crashes. This is due to new line types in the format that are not correctly ignored by the parser and cause Examinator to crash.
GCORE-16195 Despite being disabled from within the options, the Binning section is still present in HTML reports When the user disables the Binning section in Examinator's options, the Binning report section is still generated if the output format is HTML. It is expected that when the user clicks on the "Binning" link in the generated HTML report, a page gets displayed specifying that such report has not been requested.
GCORE-13349 When a FT consolidation cannot be completed due to missing data, the root cause is not specified in the TDR When a FT consolidation cannot be completed, not due to a query or computation error, but because some data is missing (ie some retest_index 1 data inserted and no retest_index 0 data yet inserted), the part counts in the ft_sublot_info table remain at 0 which is expected, the consolidation_status is set to 'P', which is also expected, but the consolidation_summary is empty. It is expected that the consolidation_summary contains the root cause preventing the consolidation computation to be completed.
GCORE-1503 When creating a "Yield Monitoring" task, no error gets displayed when the mandatory field "Bins list to Check" is left empty Yield-Man should display an error dialog when the user creates a Yield Monitoring task and leaves the "Bins list to check" field empty. This field is mandatory, and if not populated, nothing will be monitored.

GCORE-17476 Rounding errors impact on pass/fail decision when values are equal to a limit (ATDF) When inserting an ATDF file into a TDR and extracting the corresponding data, in some cases, if a value is equal to one of its limit, and has the flag to use non-strict limits (test pass if value equals limit), and has a value expressed in scientific notation in the original ATDF, the extracted value might be considered fail instead of pass. Case Number Subject Description
GCORE-16842 The FetTest datalog parser has been enhanced to support the latest .dat format specification The format specification for FetTest .dat files has changed. It is now able to support Serial Numbers up to 1 000 000. To get this possible, some minor changes have been made in the specification of some records. Basically, a new flag value is used in an existing byte of the header that indicates whether the file uses large serial numbers. If large serial numbers are turned on, any serial number (in the header and data records) is encoded on 4 bytes instead of 2.

Case Number Subject Description
GCORE-17207 Yield-Man job posts for analytics agents has been updated for agents v3.0.0 and higher Instead of posting 1 job for each analytics agent, Yield-Man now only needs to post maximum 1 single job. The post of this job can be enabled though the TDR global setting EXT_SERV_AGENT_WORKFLOW_ENABLED from the Housekeeping dialog. This option is turned off (FALSE) by default.
