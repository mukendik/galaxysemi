Dataset Analysis

To analyze a dataset, all you have to do is:

  • 1: Launch the Analyze a Dataset assistant.

  • 2: Enter a Dataset name (used as your query name in the reports)

  • 3: Select the database from which the data will be read

  • 4: Specify your filter criteria (e.g.: LotID, Process ID, Temperature, etc)

  • 5: Go to next assistant page to define what to include in your report: charts types, and which specific tests / parameters to plot (settings). Default report includes test statistics tables, Soft bin wafermap, and histogram of all tests.

For more details, see also the 'Data' tab description

 

Filters

The Filters parameters let you narrow your query to only work on the Dataset that matches your needs. Examinator lets you define multiple Filters that can be activated at the same time. List of filters available:

Combine filters?

Define if filters have to be combined or not. Selection available are:

  • Yes: all filters are combined which means you run in AND mode. Data retrieved are the one that match (pass) ALL the filters.
  • No: filters are not combined which means you run in OR mode. Data retrieved are the data that match ANY of the filters (pass one or more filters).

 

Map Test/Parameter Numbers

One of the filter field lets you select a text file (comma separated format) that includes the list of parameters / tests to map to new indexes. This is quite useful when you need to compare data sets created on different test environment where the same parameter is declared under a different ID. Full details about the file format can be found in the relevant scripting section
