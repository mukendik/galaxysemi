'Data' Tab: Select the data files to process

Once you have selected a specific assistant from the home page, the Data page is displayed. Its layout will vary slightly depending on the assistant selected. If you select the assistant to analyze a single file, then the page will only let you select one file. If you select  compare or merge files, you will be able to select multiple files and add them to the list. Lastly, if you select the assistant to compare groups of files, you get a dedicated button that lets you create and label groups.

On the Data page, you'll find some of the following icons:

Icon Description

(Smart Import)		 Lets you automatically find and select all the files matching a given criteria (date window, tester name, lot #, sub-lot #, etc...). Very powerful and easy to use assistant to quickly process big databases.

(Select files)		 Lets you select the files to analyze. Unless you're analyzing a single file, you can select multiple files at once after clicking this icon: simply press and hold the CTRL key so each file name clicked remains selected. 
When files are selected, the examinator assumes you plan to include all data found in the file (any site, any test result). If you want to change this, you have to change the file properties as described hereafter. 


(File properties)

 Once you have selected all the file to process, they appear in the list of files to analyze. Select any file in the list, then click the properties icon in order to view the file details (lot id, tester id, etc.), and define which test results to process (all data, good bin only, all sites, site X only, etc.). Double-clicking on a file name in the list also invokes the file properties dialog box.

(New group)		 Lets you create a group so you can then add files to merge into it. This icon only appears when using the compare groups of files assistant.

(Remove file)		 Lets you remove a file from a list of files (the file is not erased, it is simply removed from the list!). If you have selected the process a single file assistant, you don't see this function since you don't need it.

File properties

When you double click a file from the list (or click the file properties button), Examinator displays the properties dialog box that lets you review file header information such as LotID, tester name, etc., and data filter fields you can modify as described hereafter:

Field Description
Process Defines the type of parts (devices) to process. Default is all parts; you can change this to process only failing parts, specific parts, specific bins, etc... If your selection relates to a list (eg: list of Bins or parts), then you must also enter the list in the edit box next to the drop-down list.

The format for your list is:
XXX for a single part/bin, or
XXX to YYY for a range or parts/bins, or
XXX, YYY to ZZZ, WWW . For a list, use ',' as separator

e.g: 0 to 10, 21, 23, 25 to 29
Valid Sites Default is all test results from all sites will be processed.  If you want to filter on one specific site (eg: site # 2), then simply enter the site ID to filter.
Multi-Sites data This drop-down list allows you to force Examinator to either merge all testing site into one dataset and chart, or split each testing site data so to have as many datasets and charts to overlay as testing sites.


Data file details tab:
This tab lets you see global information about your data file such as: Testing date, test program revision, operator name, etc... If no header information can be extracted from your data file, then this list remains empty.
 

Advanced tab:

Field Description

Map tests

 Lets you map some or all the test numbers in your data file to new numbers. Also allows you to merge multiple tests under a single test number.

You have to select an ASCII file (.CSV comma separator format) that includes the list of tests to map to a new test number. For more details about the file format, see the relevant scripting section
Temperature Lets you enter the testing temperature (in °C) for this data file. Leave it empty if you don't know it or if this information is irrelevant. If you enter a temperature (E.g.: 125.0), then Examinator will create a Parameter result holding the temperature value. This allows you to very easily chart Scatter plots to show Temperature Vs Parameter and is very practical if you compare multiple files done at different temperature (characterization study)


Note: You can return to the Data page at any time to edit the list of files to process and/or the files properties. There is no need to start-over from the home page and select all the files again, just click the Data tab!
