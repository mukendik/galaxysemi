V6.6.02 (20 July 2012) release history (Top)
Examinator:
  • Case 6362 - SPEA parser: update for SPEA340 and SPEA372 format: The SPEA tester has a variation in output between * the AtosC Datalogging - Specifications (Release : 1.40) * the SPEA340 * the SPEA372. Update to the parser to accommodate all this 3 formats.
  • Case 6304 - HVM parser update for bin mapping: the parser for the bin mapping files at vishay for hvm searchs only Bin Names which starts with "001" and ends with "/S/L". need to update the parser such that this is a valid line: 1,1,001 ,PASS.
  • Case 6237 - Add the ability to choose chart over limits/results/adaptive in pearson's correlation: On the settings tab, give a pull down that allows the user to select 'chart over results', 'chart over limits', or 'chart adaptively over limits and results'.
  • Case 6410 - In Pass/Fail mode, Stacked Wafer Map report on bin 1 is not producing consistent report: The intent of this case is to get the same Stacked Wafer map when choosing bin 1, than when using default Soft Bin Wafer map.
  • Case 6406 - Low Yield pattern threshold seems not be working as expected: When setting the Low Yield threshold to a value > 90%, we expect to get more blue dies than what is actually shown (7 dies only). Indeed, the 10 wafers stacked attached have a lot more XY locations where the total number of pass bin is below 9.
  • Case 6398 - Examinator crash with do not merge MPR results option: Examinator closes unexpectedly with certain bad stdf files and do not merge MPR option selected.
  • Case 6397 - Datalog output issue: Datalog output is incorrect.
  • Case 6375 - Tool tip on stdf records dump needs updated for v4 2007 records: The tool tip on the stdf records dump in the toolbox has the same tool tip as the FAR record for all of the stdf v4 2007 records.
  • Case 6374 - Die counts are incorrect for wafer data: Incorrect stats/bin data from the Quantix prod DB extraction.
  • Case 6347 - Wrong limits when not merging tests with the same test name: When selecting to not merge tests if the test name is different, the limits for the first instance of this test is used for all instances of this test.
  • Case 6294 - The marker for the mean value on the interactive wafer map scale is incorrectly placed: When viewing a parametric wafer map in the interactive wafer maps section, the scale illustrating the min, max and mean in the right pane has the mean pointing to the min regardless of the mean value.
  • Case 6128 - Parser for Maxwell Freescale format: New Parser for csv format for Freescale whcih uses an external file for the bin #'s and names.
  • Case 6065 - Why do histograms in multi-chart from More report are cumulative histograms: When we generate a multi-chart from the More report, the histograms are plotted over cumulative results. We expect to have a chart not cumulated. If we chart over test limits, histograms are not cumulated however.
  • Case 5974 - Test names are replaced by test numbers since v64.8: In this specific configuration where PTR.test_txt is not valid and TSR.test_name holds the actual test name, Examinator replaces the test name by the test number instead. This occurs since v64.08. Workaround consists in using "ignore test name, always merge on test number" to get the expected results.
  • Case 5867 - Limits reached when querying db: A popup warning shows up when moving from interactive histogram to box plot. It tells that no samples are available because dataset is bigger than the limits allowed. This warning is inconsistent since no particular limits have been set in the options tab.
  • Case 5830 - Softbin table has incorrect headers in PPT/PDF/DOC reports: In the second part of the soft bin table, the third column header is missing (Pass/Fail) leading to an inconsistency where "Count" header replaces the "Pass/Fail" one. This also affects the other column header: "Percentage" replaces "Count", "Software Bin Chart" replaces "Percentage" and finally the last header has no header. There is also an inconsistency where "Count" should be renamed to "Total count". This affects PPT/DOC and PDF reports.
  • Case 5770 - Pareto functional failure signatures count issue: Total Fail count is wrong in the Pareto of Functional failure signatures.
  • Case 5707 - Stats tab not updated in Interactive reports and Box Plot More Report for same test number, different test name: In compare mode, the stats tab of interactive reports are not updated when moving from one parameter to the other. Also when generating a word report with box plots, the stats going with each box plot are always the ones from the first parameter. This probably comes from the fact Test numbers are the same while the test names differ.
  • Case 5697 - Incorrect adptative scale in plot when high limit is below all results: If the high limit is below all results, you get incorrect scales in plots (whether interactive or static). Indeed, when choosing adaptive scale we can't see the high limit while it is far below the results. Also, when choosing the box plot and charting over test limit, the Y axis get reversed.
  • Case 5329 - File using MPR records will not process as expected: File has MPR.RTN_INDX defined for the first PIR/PRR record pair but not for subsequent PIR/PRR record pairs. As a result, GEX assumings that the testing for the first part follows the order in the MPR.RTN_INDX but the testing for the subsequent parts follow the order laid out in the PMR.
  • Case 4717 - ATDF / STDF conversion problem with summary records: * PCR.HEAD_NUM : empty in ATDF if PCR field contains summary of the part counts for all test sites * PCR.SITE_NUM : empty in ATDF if PCR field contains summary of the part counts for all test sites
  • Case 3934 - Iddq delta does not work as expected.
  • Case 3915 - Quantix doesn't read test names from TSR's - regression: In the attached data file, test names are present in the TSR's but not in the PTR's. Quantix reads the test names and displays them in test number pick lists, but does not display test names in the Test Statistics table or in the interactive histograms. This is a regression since over 6.4.07.
  • Case 3076 - Graphs are appearing for functional tests when they shouldn't when using My Reports: when running reports, the functional tests are not graphed. however, when running custom reports using my reports, the functional tests are graphed.
  • Case 2376 - Display changes to soft bin map in the middle of a data session: The software doesn't stay on the parametric map when the map has been previously pinned.

V6.6.01 (14 May 2012) release history (Top)
Examinator:
  • Case 6228 - crash in interactive tables: GEX crashes while user is in interactive tables switching between individual part datasheets.
  • Case 6084 - User can't change the layer name in interactive histogram. Regression over 6.3: When using the edit layer properties, there is a field displaying the layer name which can be edited. Unfortunately, changes that one can do in this field don't reflect in the layer tab of the interactive chart.

V6.6 (10 May 2012) release history (Top)
Examinator:
  • Case 6146 - PAT limit display in interactive charts When Check/unckecK Org.Limits boxplot is resized: Check/unckecK Org.Limits and see that it resize the boxplot whereas it shouldn't.
  • Case 6039 - Generate a post-pat output wafer map with failing bins from input Map file: In Gex-PAT, when an AVI/MAP file is used in addition to the source STDF, the resulting post-pat output Wafer map does not contain the failed bins from the AVI/MAP file. It only contains the bins from the source STDF, and the PAT bins. The user expects to also have the failed bins from the input AVI/MAP file.
  • Case 6108 - Several dies have an inconsistent rebin in what-if: Several dies in this dataset which have a failing bin, are after a what-if reassigned to pass bin 1, while the what-if pass bin is 11 and the what-if fail bin is 99. There is no reason why we do a binning reassignment on those. Another set of die have incorrect reassignment to a What-if pass while they don't have any measures for the test on which we do the what-if. They should not be reassigned neither.
  • Case 5962 - Force binning options to sample data when using filters from the Data tab: Force option to compute stats (tests and binnings) from samples if parts filters are selected.
  • Case 5915 - Linear approximation in Pearsons/Scatter plots
  • Case 5914 - Remove data points in scatter plot mode
  • Case 5982 - Compute and display correlation in Scatter interactive charts in multi dataset.
  • Case 5789 - Do not override patman conf file with CSV recipe options.
  • Case 5525 - Update Bin Trend gui to allow the selection of multiple bins: Allow the user to select more than one bin to trend.
  • Case 5361 - Make the default "keep viewport" option in the interactive graph to unchecked: If you check the keep viewport box, then close Examinator and open it again, when you run a report the box stays checked.
  • Case 5061 - When saving a report center script, remove unnecessary paths from the options list: In the .csl file from a Reports Center script, the options section has some options that are system specific. For instance, the location and name of the r&r alarms criteria file is listed in the options section for a Reports Center report. This item will create an error when the user tries to install this report as an automated report on the Yield-Manserver.
  • Case 4375 - Merge STDF Files From Command Prompt: Allow the ability to Merge STDF Files From Command Prompt.
  • Case 4019 - In the Reports Center, allow soft bin name to be in the report title: In the Reports Center templates, allow Soft Bin Name (not just #) to be shown along with the bin number in the bin trend and bin pareto reports.
  • Case 3896 - Make the default "keep viewport" option in the interactive graph to unchecked: Provide the ability to turn off or on PAT limit display in interactive charts.
  • Case 2888 - Allow for splitting the dataset by selecting certain groups of outliers with the mouse.
  • Case 2825 - CSV Output from Database Query.
  • Case 6203 - Memory Leaks in GEX-LM: After a certain amount of time and number of connection to GEX-LM it is noticed that GEX-LM consume to much memory.
  • Case 6127 - GexPAT: No Data Samples for results at first access: When accessing PAT Report, get No Data Samples for results.
  • Case 6199 - Fix 'Last N X' in compare dataset mode
  • Case 2741 - Snapshot feature in when using What-If analysis is not functioning as expected: When using the What-If feature, the snapshot tool add an an item to the snapshot gallery but does not take a snapshot in the Windows clip board.
