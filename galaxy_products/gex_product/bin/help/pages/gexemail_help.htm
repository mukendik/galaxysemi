Introduction

This application is a companion to Examinator-Monitoring and is designed to send email messages files created by Examinator-Monitoring using the SMTP protocol. Prior to install and run this application, we assume that your Examinator-Monitoring module is already installed and works properly (it must have at least one  DataPump task and the Create Web Status task created).

The Examinator Email Dispatcher application includes the following menus in its toolbar:

  • Help
  • General
  • SMTP
  • Activity log

These menus have the following purpose:

Help: Includes the HTML on-line help documentation, details how to setup the application to have the Examinator-Monitoring email messages processed properly and sent to the relevant user.

General: Lets you define where and how often to look for Examinator-Monitoring incoming email files. The spooling folder is read from the Examinator Monitoring configuration file. It has the same path as the Web Site created by the Examinator-Monitoring, sub-folder examinator_monitoring/.emails

SMTP: Lets you define the SMTP server name and port to use when sending email files found in the email spooling folder. To test that your SMTP details are accurate, click the Test... button so the application performs a live connection and transaction handshake with your SMTP server.

Activity log: Logs exceptions such as errors while sending emails or connecting to the SMTP server. The log file doesn't require any purge admin as it automatically removes any exception older than 1 week.
 

When all settings have been set, click the Save Settings button to save information on disk.

 

Setup

To have Email dispatcher correctly process and send Examinator-Monitoring emails, you need to:

  1. Have your Examinator-Monitoring module installed and configured properly (it must have at least one  DataPump task and the Create Web Status task created)
  2. Start the gex-email application using the Windows Start menu "Start->Quantix Examinator Monitoring->Email service"
  3. Enter all setup information (spooling frequency, SMTP server and port)
  4. Save your settings using the Save Settings button
  5. Start the spooler using the Start button
  6. Have Examinator Monitoring create a few email files to make sure your SMTP settings are correct, and the emails are correctly sent.

When running the Email dispatcher in manual mode (started from the Windows Start menu "Start->Quantix Examinator Monitoring->Email service", user interface visible), you must click the Start button in order to start the email spooling process. Unless you click the Start button, the application doesn't scan the email spooling folder! Once started, you can stop it at any time clicking again the same button (toggle Start/Stop function).

Installing the application as a Windows service

Once you have entered all the appropriate setup information, it is recommended that you first start the spooling process in Manual running mode (using the Windows Start menu "Start->Quantix Examinator Monitoring->Email service"), and have it send out a few emails, so to make sure all SMTP settings are working, and the emails are correctly sent . Then you may install the application as a Windows service, using the following procedure:

  • exit the application if it is running (clicking the Exit button)
  • install the application as a Windows service using command line "gex-email.exe -i"
  • start the installed service using command line "gex-email.exe -e"

Once the application is installed as a Windows service, it will automatically be started on system boot. You don't need to be logged to the system for the service to start, and when you logout, the service doesn't stop. The application runs in background with no visible user interface. You may use the Pause function from the Windows Services control panel, or the command line "gex-email.exe -p". This brings up the application's GUI, enabling you to eventually stop the spooler, change/save some settings, and restart the spooler. Using the Resume function from the Windows Services control panel, or the command line "gex-email.exe -r" will hide the GUI, and the application will continue in the background.

Command line arguments summary:

Command line Description
gex_email.exe  -i Install application as a service (doesn't start it)
gex_email.exe  -e Execute the service
gex_email.exe  -t Terminate the service
gex_email.exe  -u Uninstall application from the service list (you should stop it first)
gex_email.exe  -p Pause the service
gex_email.exe  -r Resume the service
