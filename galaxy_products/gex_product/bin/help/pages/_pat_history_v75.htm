GCORE-4515 New rules for PAT reticle algorithm PAT reticle algorithm supports the following new rules:
* Corner Rule 1 - When there are N bad dies occupying the corner of a field, allow the user to specify an x, y, and diagonal boundary for inking.
* Corner Rule 2 - If there is a PCM die in the corner, then the die adjacent to the PCM die becomes the corner die.
* Edge Non-Corner Rule - When there are N bad dies occupying an edge of a field, allow the user to specify an x, y, and diagonal boundary for inking.
GCORE-4514 Allow for use of custom reticle maps for PAT reticle algorithms When a custom map type specified as an optional source in PAT contains the reticle configuration information, the PAT reticle algorithms can leverage that information when applying PAT.
