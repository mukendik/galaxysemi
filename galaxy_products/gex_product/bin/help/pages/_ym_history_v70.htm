V7.0.12 (14 August 2013) (Top)
Bug fixes
Case Number Subject Description
7366 GDF Validation error: Found tests with the same test number and different test name Customer is frequently getting the old "tests with same test number and different test name" Yield-Man loading error that he used to get for invalid STDF files, only now it is for GDF files, which have no test numbers. So this looks like an issue with Quantix's GDF parser where, however it is adding test numbers, it is assigning the same TN to different tests.
7435 Case sensitivity discrepancy between regular expression checker and YM/TDR insertion process Regular expressions in gexdbkeys are not sensitive to the case, which leads to process lower case letters and capital letters the same way.
However in the regular expression checker from the tool box the case sensitivity does apply and lower / upper cases letters are different.
It is expected to make the regular expression checker not case sensitive so that it behaves just like the regexp tool in YM/GEX TDR.
