V6.3.16 (12 July 2011) (Top)
Yield-Man:
  • case 4907 - Datapump loses the path of the "Execute batch on insertion" with YieldMan Administration Server: When using YieldMan Administration Server, Datapump loses the path of the "Execute batch on insertion" fields.
  • Case 4882 - Error connecting to a GEXDB from a system, if the link has been created on a system using a different Endianness: If a GEXDB link has been created from a system using a specific Endianness (ie Windows using little endian), the connection will fail if attempted from a system using a different Endianness (ie Linux using big endian). The same applies when connecting to the YM Admin DB.
  • case 4881 - Yield-Man crash while inserting in Black Hole database: Yield-Man crash while inserting in a Black Hole database if a SYA task exists for the inserted product.
  • case 4873 - All Data going into splitlot 0: All data files are being inserted into Splitlot 0 when the AUTO_INCREMENT of table ft_splitlot is undefined.
  • case 4829 - UMC - PCM Parser: The parser recognizes the UMC PCM file. However, it does not load properly. Only the first column of data is loaded. If line 5 is deleted from the csv file, then the data loads properly, however X and Y position information is lost.
  • case 4814 - Yield-Man: Wafer consolidation fails during Data insertion under ORACLE: CREATE TEMPORARY TABLE for wafer consolidation fails.
  • case 4809 - 7C7 PCM Parser error: 7C7 file not converted.
  • case 4786 - PAT server no_update key is affecting the application of the cluster algorithm: when running PAT-server with the cluster algorithm turn on using an external map, if the trigger file uses no_update when specifying the external map, the cluster fallout is not reflected in the pat output stdf, log and sinf files although it should be.
