Merge data files analysis

To analyze data from a list of files, all you have to do is:

  • 1: Launch the Merge files assistant and select the files to merge and process

  • 2: Edit the filter by double clicking a file name (or clicking the properties button) to edit which test sites to process, and/or which parts to analyze (e.g. Bin1 only, Failures only, specific list of parts, etc.). Repeat for each file as necessary.

  • 3: Go to the next assistant page to define what to include in your report (settings), then have it created. Default report includes test statistics tables, Soft bin wafermap, and histogram of all tests.

Smart Import: Query over Time Period

You may find it very convenient to import all files that match a date and filters criteria. This very powerful function lets you only import test data coming from files generated between specified starting and ending dates. Furthermore, you can also define filters such as tester name, lot name, device type etc... For full details about this function, check the Smart Import page.

: Click this icon to import files based on creation dates and filters.

 

Load/Save your File List

Examinator allows you to save your list of files and filters to disk so you can later reload it with just one mouse click! This is a very convenient feature that boosts your productivity on repetitive tasks involving the same data files.

: Click this icon to save your current list of files and filters to disk (saves a script file)
: Click this icon to reload a list of files and filters from disk (reads a script file)

 

Filter: Parts and Sites to process

When editing the file properties to show data file details and/or change the type of parts and sites to process, the following dialog box appears:

Map Test Numbers

From the Properties page, you can select a text file (comma separated format) that includes the list of tests to map to new test indexes. This is quite useful when you need to compare data files created in a different test environment, or need to merge files with test numbers that no longer match because of new tests added. Full details about the file format can be found in the relevant scripting section
