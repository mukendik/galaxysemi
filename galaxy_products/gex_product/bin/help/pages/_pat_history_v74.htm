V7.4.0 (1 July 2015) (Top)
Bug fixes
Case Number Subject Description
GCORE-4250 Changing the hard bin in a PAT recipe does not change the hard bin for NNR, IDDQ, GDBN, and Clustering Changing a soft bin in a PAT recipe changes the soft bin for NNR to match the PAT recipe, but changing a hard bin in a PAT recipe does not change the hard bin for NNR. The hard bin also does not change for IDDQ, GDBN, and Clustering.
GCORE-4184 The deviceid and part_id from the IFF data parser do not match From run 1, part_id = 0 and devid = 1. Expected result is part_id = devid.
GCORE-4167 Probe data merged with AVI map cannot be inserted into Yield-Man TDR When a user merges probe data with an AVI map, Yield-Man rejects it because the output STDF contains some TSR records before the PIR/PRR records from the AVI map. This only affects the Functional TSR. PCR record may also not be correctly located in the STDF.
GCORE-4158 Some JaveScript API calls do not work properly via HTTP requests The following JavaScript API calls accept file paths as method argument but fail when they are called from an HTTP request:
* GSEngine.RemoveFileFromDisk(<filepath>)
* GSFile.Remove(<filepath>)
* GSFile.Copy(<filesrc>,<filedest>)
GCORE-4144 PAT-Man outputs an invalid (empty) CSV map when it detects an error during processing If PAT-Man detects an error during PAT processing (e.g. no hard bin for a tested die), it should never output an invalid CSV map.
GCORE-4010 When a user deletes an option from the admin options window, it is no longer visible in options list When a user deletes an option, the option is no longer visible in the list of options. To make the deleted optoion re-appear in the list of options, the user must close the window and re-open it.
GCORE-3868 Missing parts from wafermap After PAT processing a wafer sort file, a section of the wafer map is missing in the generated report. When performing single file analysis on the same file, this section is not missing.
GCORE-2934 Enabled tests are are not appearing in a PAT recipe In Examinator-PAT, the user selected a few enabled tests and selected "Keep Selected tests in PAT recipe." The selected tests were listed under Tests PAT-Ready tab. The user then generated the PAT recipe, but the recipe was blank, and the enabled tests were not visible in the display.
GCORE-910 It takes too long to open the filter pick list from the interactive tables In the interactive tables, there is a funnel icon that is used to filter tests which are displayed. When a user clicks on the funnel icon there is an unacceptable delay before the test list window appears. Users expect to immediately see the test list window after clicking on the funnel icon.

V7.4.RC1 (2 June 2015) (Top)
Bug fixes
Case Number Subject Description
GCORE-3845 IFF data parser is extracting the wrong wafer ID The IFF parser assigns WAFER_NUMBER to the wafer_id field in the Test Data Repository (TDR), which is wrong. The IFF parser should assign WAFER_ID to the wafer_id field in the TDR, and WAFER_NUMBER can be ignored by the IFF parser.
GCORE-3780 Final test PAT simulator generates a gtl_init error When using the final test PAT simulator in GEX-PAT, the user receives a gtl_init error.
GCORE-3721 Insertion of EDF file causes Examinator and Yield-Man to crash This EDF file causes many crashes. For instance, making a default Examinator report causes Examinator to crash.
GCORE-3687 Final Test LVM FETTEST data is incorrectly processed as wafer sort data In Examinator, Final Test (FT) LVM FETTEST data is processed as final test data, as expected. But when Examinator parses FT LVM FETTEST data in a PROMIS data file, it rejects this data with the error message "wafer not found in PROMIS info file."
GCORE-3686 Tests are in different order when saving a JSON recipe making it not compliant with diff and versioning tools The test list in Examinator-PAT is expected to be ordered in a consistent way so that users can use a diff tool or versioning tool to compare two versions of a recipe and identify whether there are changes. Currently, this is not possible because the list of tests is reordered each time it is saved with or without modifications.
GCORE-3644 NNR PAT algorithm failed all good die on a wafer (100% fallout) This 100% fallout event is likely caused by a large cluster event in the center of the wafer, but 100% fallout is excessive.
GCORE-3628 The masking function in GDBN or other PAT clustering algorithms does not work The masking function in the reticle PAT algorithm works, as expected. But the masking function is not working in GDBN (Good Die in a Bad Neighborhood) and other clustering algorithms.
GCORE-3618 GDBN identifies die next to PCM die (not on the map) as edge die The GEX-PAT Good Die in a Bad Neighborhood (GDBN) algorithm currently identifies neighboring die around a wafermap location with a missing die as edge die. For example, a wafermap could have a PCM die in the middle of the wafer. This PCM die would be missing in the wafermap, and the GDBN algorithm would identify the neighboring die surrounding the PCM die as edge die. The GDBN algorithm would be more useful if it treated die next to empty PCM die as normal, non-edge die.
GCORE-2862 When “PAT limits built from” is set to “All data / parts (any bin)”, TSR record has wrong value If user sets the option "PAT limits built from" to "All data / parts (any bin)", the TSR record in the post PAT STDF contains an incorrect value of 0 fails for a test that is known to have 2 fails.
GCORE-1315 Examinator does not generate the wafer map on an "inline" G85 XML map The user could not generate a wafer map for a particular.XML file. This is likely due to the file containing inline data, which prevents Examinator from parsing it correctly. Inserting line breaks to separate the lines in the file allows the wafer map file to be generated.

V7.4.E (10 May 2015) (Top)
Enhancements
Case Number Subject Description
GCORE-3629 Allow final test PAT to use test name as key instead of test number Not all testers support both test numbers and test names. For testers that support test names only, PAT recipes must be able to use test names as keys instead of test numbers.
GCORE-3176 When merging two datalogs containing different tests, show all test results of a die XY in the interactive part results table After merging two datalogs containing different sets of test, show all test results for a die XY in the same interactive part results table. In previous versions when datalogs were merged, the interactive table showed different entries for the same die XY making it harder to interpret the results.
GCORE-3084 Enhance PAT recipe editor to activate/deactivate tail management per test when using Smart and Adaptive algorithm The enhancement to the Smart & Adaptive algorithm is as follows:
If the DPAT rule selected *is not* "smart & adaptive", then the option is disabled. If the DPAT rule selected *is* "smart & adaptive", then the user can choose between: Disabled / Conservative / Medium / Liberal. For compatibility with previous versions, the *default* value is "Liberal".

If *Disabled* is selected: no tail management will be applied
If *Conservative* is selected: a hole of more than 1*IQR will be searched
If *Medium* is selected: a hole of more than 2*IQR will be searched
If *Liberal* is selected: a hole of more than 3*IQR will be searched
GCORE-205 Grey out the GUI area for the js hooks when ym_admin_db is not in use If a user is not using ym_admin_db, the js hooks entered in the GUI will not be saved in the .gexmo_tasks.xml for use/reload later. In this case, the hooks will be lost when Yield-Man is restarted. To prevent users from losing their work, grey out the GUI area for the js hooks when ym_admin_db is not in use.
GCORE-35 Remove ZPAT 3D GDBN option shown in GUI This option was not implemented and should be removed from the GUI.
Bug fixes
Case Number Subject Description
GCORE-3558 Geographic PAT rules are not reloaded when disabled When a user disables a Geographic PAT (GPAT) algorithm, the rules for this algorithm are saved into the recipe file. But when Examinator-PAT later reads this recipe, it does not reload the rules for the disabled GPAT algorithm.
GCORE-3346 Geographical PAT always ignores Parametric PAT even when it should not ignore it Geographical PAT (GPAT) always ignores Dynamic PAT (DPAT) failures even when the GPAT option to ignore Parametric PAT (PPAT) failures is not checked. PPAT comprises the following algorithms: Static PAT (SPAT), Dynamic PAT (DPAT), and MultiVariate PAT (MVPAT).

GPAT does ignore DPAT fallout.
GCORE-3191 If ATDF file is missing an MRR, the toolbox should not try to convert the last record The toolbox expects the last record in an ATDF file to be an MRR record. When an ATDF file is missing an MRR, the toolbox should ignore the last record in the file instead of converting it, which will cause problems.
