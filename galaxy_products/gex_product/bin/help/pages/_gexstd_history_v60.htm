V6.0 B154 (18 September 2009) (Top)
Examinator:
  • Case 2195 - Enhanced Tango Parser: Implemented several enhancements to the Tango parser.
  • Case 2160 - Title for PNG Wafermap export: Added a title line to the exported PNG Wafermap.
  • Case 2159 - Wafermap export: Added a feature to the Export Wafermap utility. When using the 'Export wafermap' feature the first time, the path for the output file will be set to the 'Report Output' dircetory, then it will be remembered throughout the session.
  • Case 2112 - EDL Parser: Added enhanced functionality to the EDL data parser.
  • Case 2102 - Interactive Chart Display Modes: Added a feature that allows the user to specify a display type in the interactive mode, and that same display type will carry through all of the interactive charts during that particular session.
  • Case 2100 - New Markers for Histograms: Added Min, Max, and Test Limits to the banner mode histograms.
  • Case 2096 - Enhanced SPEA Datalog support: Added enhanced suppport for the SPEA datalog .log format.
  • Case 2049 - Units in CSV Export: Added the capability to export CSV format with non-normalized units.
  • Case 2007 - Scaled Units in Toolbox CSV Output: Added a capability to export data into a CSV file with scaled units in addition to normalized units.
  • Case 2241 - Hyperlinks not working: Fixed a bug that would cause the Test Statistics page hyperlinks to act improperly at times.
  • Cases 2240 & 2223 - Software Crash: Fixed a bug that caused Quantix to crash when viewing a 3D wafermap under certain conditions.
  • Case 2197 - Error in Statistics on CSV Output: Fixed a statistics calculation error that would cause a floating point discrepancy between the Test Statistics table and the Toolbox - CSV output.
  • Case 2190 - Incorrect pass/fail: Fixed a bug that would cause parts that are between the spec limits to register as failing parts unders certian conditions. This would appear when Normalize/Scale Statistics was set to Normalize All Units.
  • Case 2184 - Test results not properly aligned: Fixed a bug that would cause Quantix to misalign results for STDF files with interlaced results.
  • Case 2181 - Save Settings: Fixed a problem with the Save Settings page. Prior to this fix when saving settings from the Settings page, if some settings contained a list of tests with one or more commas, the test list would be truncated before the comma upon reloading.
  • Case 2175 - AWW Files: Fixed a problem that would cause Quantix to crash when loading multiple AWW files.
  • Case 2161 - Image PNG Wafermap export: Fixed a bug that caused an exported wafermap to be oriented improperly.
  • Case 2147 - Test Number Pick List: Fixed a problem that would cause Quantix to freeze when typing a test number into the test number pick list that was not present in the database.
  • Case 2144 - Titles misaligned: Fixed a glitch that caused titles to be misaligned on PDF reports.
  • Case 2126 - Pearson Correlation Report: Fixed a problem that caused Quantix to crash when running the Pearson Correlation report under certain conditions.
  • Case 2125 - Incomplete Optical Files causing freeze: Fixed a bug that would cause Quantix to freeze, with no error message, when loading incomplete Optical files such as Imagemaster, Talysurf, & Voyager.
  • Case 2116 - Individual retested die display: Fixed a bug that caused an individual die on the 3D wafermap to display the first-pass Min/Max values on wafermaps and histograms.

V6.0 B153 (7 August 2009) (Top)
Examinator:
  • Split Dataset by Two Fields: Added a feature that allows for splitting a dataset by a combined database field: Lot;Wafer.
  • Merge Dataset Feature. Added a feature that allows for the merging of multiple datasets. This allows for calculating consolidated yield when there are multiple first pass and multiple retest splitlots.
  • Interactive Dataset Split Feature: In the interactive mode, Quantix now supports the capability to split a dataset and maintain the split while scrolling through other test id's.
  • Fault Tolerance:
    • Added additional fault tolerance to prevent Quantix from crashing when it encounters corrupt files or missing fields in data files.
    • Added the ability to detect if Sites are logged out of order in the data file.
  • New Parsers:
    • Added suppport for the Teradyne Catalyst .log format.
    • Added support for Optical testers: QV302, Voyager, & Talysurf.
  • Time Zone Error: Fixed a bug that caused Quantix to apply an incorrect time zone adjustment under certain condiditons, causing the time to display 8 hours ahead of the correct value.
  • Error on Opening Interactive Tables Fixed a bug that would cause Quantix to crash under certain conditions when opening the Interactive Data Tables, Trend Charts, or Histograms.
  • Missing Test Names during TSMC Conversion to CSV. Added code to include all test names when converting from TSMC WAT format to CSV format.
  • Powerpoint: Fixed a bug that would cause Quantix to close an existing Powerpoint window when creating a new report in Powerpoint. Under certain conditions, this could cause work in Powerpoint to be lost.
  • Fixed a problem that would cause some charts to not display properly in Powerpoint when using the Powerpoint template and the MyReports format.
  • Yield & Bin Paretos:
    • Fixed a bug that would cause yield and bin paretos to display incorrect values when the Last Test Instance filter is used.
    • Fixed a bug that caused the failing die contribution to display 'n/a.'
    • Fixed a discrepancy between Enterprise reporting of yield and detailed reporting of yield that occurred under certain circumstances, particularly when a query spans more than one month.
  • Remove All Matching Parts: Fixed a bug that caused some statistics to be calculated incorrectly when using the Remove All Matching Parts feature in the interactive report.
  • Reporting Units of Percentages: Fixed some problem that occurred when working with units of percentage. Percentage now is reported correctly when using data extraction from the Relational Database, and when performaing calcuations in What-If / Virtual Retest.
  • Converting GZ Compressed Files: Fixed a problem that caused Quantix to run very slowly when de-compressing GZ files.
  • Gage R&R: Fixed a problem that caused negative R&R percentage values to display under certian conditions.
  • Wafermap X-axis swap, Y-axis swap: the wafermap X-axis swap and Y-axis swap options (Options->Wafer map...->Visual options...) were not functional anymore in V6.0.
  • What-If: fixed a bug that caused Examinator to use an incorrect scaling factor when the limit was manually modified while using What-If / Virtual Retest.
  • KLA Wafer XML parser: Corrected to insert the proper PosX and PosY information. Examinator used the default PosX and PosY values (PosX=Right, PosY=Down), so that the wafer map could be swapped in both X and Y.
  • Trend chart on custom parameters: fixed a bug that caused Examinator to crash when displaying a trend chart for a custom parameter created with the 'Create a custom parameter...' button in the 'Interactive Charts' window.
  • Functional tests when filtering parts: fixed a bug where the functional test results of excluded parts were not removed when the dataset was filtered (ie 'Last test instance' filter).
  • 'Pick tests from list...' dialog box: fixed a bug that would cause only the last character to appear in the edit box, overwriting the characters previously entered, when entering a string into the 'Search' edit, .

V6.0 B152 (27 May 2009) (Top)
Examinator:
  • New option for bin stacked wafer maps: an option was added to specify how the stacked binning wafer map should be generated. Available Options are:
    • Bin Count: the map will display the count of matching bins for each X/Y location.
    • Pass/Fail: the map will display a merged pass/fail information for each X/Y location. An X-Y location is considered a Pass Only If all the die for all wafers at that particular location are Pass. A missing die is considered a fail.
    The option is accessible through the GUI at 'Options->Wafer map, Strip map->Bin stacked wafermap', and through the scripting language using "gexOptions('wafer','bin_stacked','');".
  • New option for trend chart X-axis: an option was added to specify the iformation to use on the X-axis for trend charts. Available Options are:
    • Run ID: the trend chart X-axis will contain the Run ID (1,2,...n).
    • Part ID: the trend chart X-axis will contain the Part ID (alphanumerical value read from the data file).
    The option is accessible through the GUI at 'Options->Trend (Control chart)->X-Axis scale type)', and through the scripting language using "gexOptions('adv_trend','x_axis','');".
  • Support for 'Laurier Die Sort' wafer map format: added support for the 'Laurier Die Sort' wafer map fomrat. This format is supported both as an input file, and as an output file (wafer map export function).
  • 'Datalog' section added to the 'My Reports':
  • Display of the PartID: the PartID is now displayed in various locations:
    • when searching for a particular part in the interactive wafer map.
    • when searching for a particular part in the interactive table ('Part results' tab).
    • in the tool tip when hoovering the mouse over a specific data point in the trend chart (if option 'Options-> Data Processing options->PartID display' is set to 'Show PartID').
    • in the interactive chart's 'Table' tab (if option 'Options->Data Processing options->PartID display' is set to 'Show PartID').
  • Support for 'SPEA datalog': added support for the 'SPEA datalog' data fomrat. This format is generated by SPEA testers.
  • Teradyne Image datalog parser: the parser for the 'Teradyne Image datalog' ascii format has been enhanced:
    • lines with non standard content (unknown keywords) are ignored, rather than generating a parser error.
    • test numbers with more than 6 digits are supported.
    • test numbers from the datalog file are used, rather than auto-generated test numbers.
  • Additional fields in the CSV export file: when exporting a data file to the Quantix CSV format, additional fields are written to the header (if populated in the data file):
    • ModeCode
    • RtstCode
    • ProtCode
    • CmodeCode
    • BurnTime
  • Wafermap retest policy option: fixed a bug that caused an inconsistency in bin counts between the 3D Interactive wafermap and the Binning section of the report. This error was previously observed only under the following Options settings.
    • 'Options->Wafermap->Retest Policy' = 'Highest Bin'.
    • 'Options->Binning->How is Binning Computed' = 'From Wafer map'.
  • <end dataset> tag in CSV files: fixed a bug that caused <end dataset> tag in Quantix CSV files to be read improperly.
  • SQL error when querying sample data: fixed a bug that caused Examinator-Production to display an SQL error when extracting data from an RDB database. This error occurred when a query included a list of tests for which the raw data should be extracted. If the dataset was composed of multiple splitlots, the error would appear for any splitlot having no executions for one of the selected tests.

V6.0 B151 (10 April 2009) (Top)
Examinator:
  • Parametric test stacked wafermap using median: an option was added to let the user select whether the mean or the median should be used to generate the parametric test stacked wafer map.
  • Parametric Pass/Fail wafer map: Examinator now allows (from the 'Settings' page) to select Parametric test Pass/Fail wafer maps. Each individual Pass/Fail wafer map can be viewed in the interactive wafer map view, and the user can dynamically switch between 3 possible views: 'Over test limits', 'Over test results', 'Pass/Fail'.
  • Parametric failure stacked wafer map: When multiple wafers are merged, and Parametric test Pass/Fail wafer map is selected, a stacked Pass/Fail wafer map is also generated.
  • Multi-selection in layers list view: the layers list view now supports multi-selection, enabling to hide/show/remove multiple layers at a time.
  • Display of specification limits: if specified in the STDF file, Examinator now has the option to display specification limits along with the test limits (Options->Test statistics->Fields to include in report table).
  • Labels in interactive box-plots: the angle used to display the X-axis labels in interactive box-plots has been modified to enhance the lisibility of the labels. Also, the labels now start properly below the corresponding tick, where the tick used to be aligned with the middle of the text. The zone used for displaying the labels now automatically adapts to allow displaying the labels without being truncated. When the labels are too long, and the label zone is too big, reducing the zone available for the graph, the option 'Options->Report Output->Truncating test names' can be used to automatically truncate the labels at a fixed length.
  • Split fields in Enterprise Reports couldn't be removed: in the Enterprise Report yield wizard, when selecting fields to be used to split the dataset into several graphs (page 3/4), or each graph into several layers (page 4/4), the selected fields could not be removed from the list. The user had to do a 'Reset'.
  • Crash when re-building Examinator test list: fixed a bug where Examinator crashed when re-building it's testlist. Also, in some circumstances, the application was unnecessarily asking the user if the testlist should be rebuilt or if Examinator should use the existing one. When selecting to rebuild the testlist, the application crashed.
  • Binning type (P/F) not initialized if no summary records available: fixed a bug where the binning type (Pass or Fail) was not initialized when no summary data (SBR,HBR) is available in the STDF file, and no wafermap stacking is performed. Examinator now initializes the binning type using as default: Bin1=Pass, other Bins=Fail. This problem could affect the binning mismatch table when comparing 2 datasets.
  • Show all test results for run#: fixed a bug introduced recently, where nothing happened when right-clicking on a data-point in the trend chart, and selecting 'Show all test results for run#'.
  • Line type for gaussian curve not changing: fixed a bug where when comparing datasets, and then changing the line type for the gaussian curve in the interactive histogram, it had no effect on the graph.
  • "-1.#IND V" displayed in the statistics table: fixed a bug where invalid mean values (ie no executions) were displayed as "-1.#IND V" in the test statistics table, instead of "n/a".
  • Pin names not displayed in the Datalog report: fixed a bug where pin names were correctly appended to the test names ( - ) in the statistics table (when selecting the option to slit pins), but not in the datalog report.
  • Database upload: fixed a bug where Examinator failed uploading some zipped stdf files (*.STD.gz) into a database. It would log a "File copy failed" error.
  • Test & Re-test merge tool/Yield trend: fixed a bug where the Yield trend production report was not displaying the right yield on a merged test and re-test file.
  • Log scale looks linear: fixed a bug where the probability plot scales didn't change when selecting the 'Logarithmic scale' in the "Edit Legends, Scales..." menu.
  • Scales not changing in interactive histograms: fixed a bug where the histogram scales didn't change when modifying the 'Low X:' and 'High X:' values in the "Edit Legends, Scales..." menu.
  • Compare parametric wafermaps: fixed a bug where Examinator crashed when comparing 2 datasets and selecting 'Parametric wafermap' in the 'Settings' page.
  • Interactive charts: fixed a bug where when using a custom number of histogram bars (Options->Histogram->Total bars/bins to plot histograms), after creating a standard Examinator HTML report and clicking on the 'Charts' link from the 'Report' tab, the application would crash.
  • Gex-Lm on Windows Server: fixed a bug where Gex-Lm would not start properly on a Windows Server system, despite the gex-lm.exe process being still alive.
  • Uncompressing zipped .STD files: fixed a bug where Examinator would fail processing a zipped file (.zip, .gz) containing files with a .STD extension in uppercase.

V6.0 B150 (17 March 2009) (Top)
Examinator:
  • Gage R&R: added an option letting the user select how Examinator should compute the variation: over TV or over limit space.
  • Multi parametric wafermaps: when selecting 'Parametric test...' for the 'Wafermap' section in the 'Settings' page, the user can now select a list of tests. This results in multiple parametric wafermaps in the resulting report.
  • Re-test Filtering now available for Final Test Data: the 'First Test instances...' and 'Last Test instances...' now also works on Final Test data, provided the PartID string contains numerical data. It uses X/Y coordinates computed by Examinator to create the Strip map (based on the number extracted from the PartID) to identify re-tests.
  • Plug-in data extraction: data extraction from a SQL database through a Quantix plug-in can now be aborted (Abort button).
  • Test mapping enhanced: the test mapping is now done through a test setup XML file, and offers more flexibility. Test number and test name can be mapped for the same test.
  • Up-front test filtering added: tests can now be filtered up-front, prior to report creation, by using the Test Setup XML file. Only those tests that are specified in the filter will be present in the static reports, and in the interactive tables & charts.
  • New inkless wafermap formats supported: following inkless wafermap formats are now supported as input files:
    • Semi G85 (XML)
    • Semi E142 (XML)
    • SPIL
  • Semi G85 ascii inkless wafermap format: Examinator now supports loading Semi G85 (ascii) inkless wafermap files containing multiple wafermaps.
  • Interactive charts zoom: fixed a bug where the chart viewport was limited to the original selection (over data, over limits, adaptive), preventing the user to further zoom out (ie to view the 6*Sigma marker).
  • Extraction performance with Oracle: resolved an issue where the extraction performance on Oracle databases was 10+ times slower than with V5.7. The performance is now back to normal (same as with V5.7).
  • SPD parser: fixed a bug where the SPD parser would reject SPD files containing empty lines.
  • Strip map: corrected a bug where the Strip map was not correct in certain circumstances.
  • Dates in the 'Compare datasets' query page: fixed a bug where the dates selected for an individual dataset (when using 'Time period'='Pick dates from calendar...') were reset to year 1752 when re-opening the 'Properties' dialog for this dataset.
  • STDF V3 X/Y coordinates: fixed a bug introduced in V6 where the X/Y coordinates read from a STDF V3 file were not stored correctly, and as a consequence, the resulting wafermap was wrong.
  • Word report generation: fixed a bug where the Word report generation was very slow, and the resulting Word report didn't have any layout (page breaks).
  • Interactive/advanced histograms: fixed a bug where interactive/advanced histograms were empty if all test values were identical. The histogram will now display one bar.
  • Filter button in interactive windows: fixed a visual bug where the filter button was in checked state, even if the user clicked the 'Cancel' button in the filter selection dialog box.
  • What-If: fixed a bug that caused a revised limit to be ignored in reports if this limit was not present in the original data set.
