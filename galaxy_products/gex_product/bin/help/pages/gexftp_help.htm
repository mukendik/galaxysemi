Introduction

This application is a companion to Examinator-Monitoring and is designed to get test data files from remote servers, and ftp them to the local Examinator Monitoring server. Prior to install and run this application, we assume that your Examinator-Monitoring module is already installed and works properly (it must have at least one  DataPump task and the Create Web Status task created).

The Examinator FTP Client application includes the following menus in its toolbar:

  • Help
  • General
  • FTP
  • Activity log

These menus have the following purpose:

Help: Includes the HTML on-line help documentation, details how to setup the application to have the test data files correctly retrieved from one or several remote servers, and ftp'd to the local Examinator Monitoring server.

General: Lets you define how often the tool should scan the remote server(s) for test data files to transfer.

FTP: Lets you define several remote servers from which you want to retrieve test data files. 

Activity log: Logs exceptions such as errors while getting files from remote servers. The log file doesn't require any purge admin as it automatically removes any exception older than 1 week.
 

When all settings have been set, click the Save Settings button to save information on disk.

 

Setup

To have Examinator FTP client correctly retrieve test data files from one or several remote servers, you need to:

  1. Have your Examinator-Monitoring module installed and configured properly (it must have at least one  DataPump task and the Create Web Status task created)
  2. Start the gex-ftp application using the Windows Start menu "Start->Quantix Examinator Monitoring->Ftp service"
  3. Enter all setup information (spooling frequency, remote servers)
  4. Save your settings using the Save Settings button
  5. Start the spooler using the Start button
  6. Have some files posted to one of the remote servers, to make sure Examinator FTP client correctly retrieves them.

When running the FTP client in manual mode (started from the Windows Start menu "Start->Quantix Examinator Monitoring->Ftp service", user interface visible), you must click the Start button in order to start the ftp spooling process. Unless you click the Start button, the application doesn't scan the remote servers! Once started, you can stop it at any time clicking again the same button (toggle Start/Stop function).

Installing the application as a Windows service

Once you have entered all the appropriate setup information, it is recommended that you first start the spooling process in Manual running mode (using the Windows Start menu "Start->Quantix Examinator Monitoring->Ftp service"), and have it transfer a few files from one or all of the defined remote servers, so to make sure all FTP settings are working, and files correctly transferred to the local Examinator Monitoring server. Then you may install the application as a Windows service, using the following procedure:

  • exit the application if it is running (clicking the Exit button)
  • install the application as a Windows service using command line "gex-ftp.exe -i"
  • start the installed service using command line "gex-ftp.exe -e"

Once the application is installed as a Windows service, it will automatically be started on system boot. You don't need to be logged to the system for the service to start, and when you logout, the service doesn't stop. The application runs in background with no visible user interface. You may use the Pause function from the Windows Services control panel, or the command line "gex-ftp.exe -p". This brings up the application's GUI, enabling you to eventually stop the spooler, change/save some settings, and restart the spooler. Using the Resume function from the Windows Services control panel, or the command line "gex-ftp.exe -r" will hide the GUI, and the application will continue in the background.

Command line arguments summary:

Command line Description
gex-ftp.exe  -i Install application as a service (doesn't start it)
gex-ftp.exe  -e Execute the service
gex-ftp.exe  -t Terminate the service
gex-ftp.exe  -u Uninstall application from the service list (you should stop it first)
gex-ftp.exe  -p Pause the service
gex-ftp.exe  -r Resume the service

Modifying the settings of a GEX-FTP instance running as a Windows Service

This section describes how to modify the setup of a GEX-FTP application running as a Windows service.

Local login

If you are logged on the server running the GEX-FTP service:

  1. Launch the Windows Services control panel (Windows Control Panel->Administrative Tools->Services)
  2. Select the "Examinator GEX-FTP Vx.y"  service
  3. Pause the service using the 'Pause Service' button, or the mouse right-click
  4. The GEX-FTP GUI will be displayed (if you cannot see it, click it's icon in the Taskbar)
  5. Stop the spooling process by clicking the 'Stop' button
  6. Modify your settings ('General' and 'Ftp' sections)
  7. Save your settings by clicking the 'Save settings' button
  8. Check the 'Activity log' section to make sure no error occurred during the save
  9. Re-start the spooling process by clicking the 'Start' button
  10. In the Windows Services control panel, resume the GEX-FTP service by using the 'Resume service' button, or the mouse right-click

Remote login with Windows Terminal Services

This section describes how to modify the GEX-FTP settings when the service is installed on a server running Windows Terminal Services, and you are connected to the server running the GEX-FTP service through a Windows Terminal Services client/server connection. The specificity of this situation is that if you pause the service, the GEX-FTP GUI will be displayed on the console of the server, but not on your terminal. This is because the GUI runs under the local service account, and this account has no access to the screen real estate of the user logged in via Windows Terminal Services. If the GEX-FTP application is started manually, the directory holding it's settings and log files is different from the one used by the GEX-FTP service (for obvious access privileges reasons). To workaround this situation, GEX-FTP includes, when started manually on a system having the GEX-FTP service installed, a 'Service' tab, that enables you to:

  • stop the service
  • load its settings file
  • save current settings to its settings file
  • display the service's log file

The process is as follows:

  1. Manually launch GEX-FTP using 'Windows Start menu->Programs->Quantix Examinator Monitoring->Ftp service'
  2. Select the 'Service' section (if you cannot see any 'Service' section in the left pane, this means that the service is not installed)
  3. If the service is running, stop it by clicking the 'Stop service' button
  4. Make sure the 'Service home directory' is correctly set. If not, select the correct directory using the 'Select the service home directory' button. If this directory is correctly set, the 'Load from service settings' button should be enabled when the service has been stopped. If you are not sure of the directory, search for the .gex_ftp.conf file on the server hard drive. It should be either in the Windows directory, or in the home directory of the local service account. Don't use your local home directory, which may also contain a .gex_ftp.conf file.
  5. Load the settings used by the service by clicking the 'Load from service settings' button
  6. Modify the settings ('General' and 'Ftp' sections)
  7. You can test the new settings by starting the spooler ('Start' button). Then click the 'Status' tab in the 'Ftp' section. Once the spooler has finished checking all Ftp sites you configured, or you aborted the process, stop the spooler by clicking the 'Stop' button
  8. Save the new settings to the settings file used by the service by clicking the 'Save to service settings' button in the 'Service' section.
  9. Re-start the service by clicking the 'Start service' button, and check the 'Service log' to make sure the service started correctly
  10. You may save the settings locally (in your user home directory) by clicking the 'Save settings' button
