GCORE-6223 Fix xml test mapping to handle no-limit tests When creating a new test with the xml test mapping file, if there are no limits set, Examinator arbitrarily sets the limits to 0. This now allows for having no limits.

V7.6.0-beta (Top)
Enhancements
Case Number Subject Description
GCORE-8313 Ability to monitor for Cr and Cp shifts When comparing data, there is now the ability to monitor for Cr and Cp shift such that shifts over a threshold can be highlighted.
GCORE-8175 Populate data tab when running csl When running a saved report from a csl in the Scripting Center, populate the data tab with the information specified in the csl report.
GCORE-7998 Customers with multiple limit sets can choose which set to do analysis on Now customers with multiple limit sets can choose which set to do their analysis on and all the statistics and charts will be based on the selected limit set.
GCORE-7258 Replace the use of xls with csv Examinator does not support xls files specifically but rather outputs csv files that can be viewed with any spreadsheet tool. To prevent confusion, the use of this output format is now referred to as csv.
GCORE-7235 IFF Parser update - Rely on binning only for P/F flag For all IFF parsers (.ftpi, .prbd, .pcme, .pcmd formats), the part P/F flag should be set using the binning information present in the IFF file, rather than comparing test results to their limits.
GCORE-7206 IFF Parser update for Genealogy - Load the site_config for E-Test The IFF parser has been updated to support the IFF.site_config field. Following DTR record is used to store the information in the generated STDF file:
DTR.text_dat = {"KEY": "SITE_CONFIG","TYPE": "MD","VALUE": "5-zone-circle"}

This information is then used to populate the site_config field in the TDR (E-Test testing stage), which is required for the Genealogy feature.
GCORE-7205 IFF Parser update for Genealogy - Load the wafer number for E-Test and Wafer-Sort The IFF parser has been updated to write the wafer number into the converted STDF file for E-Test (ET) and Wafer-Sort (WT) testing stages, in addition to the wafer_id field, which is alphanumeric.
Following DTR record is used to store the information in the generated STDF file:
DTR.text_dat = {"KEY": "WAFER_NUMBER","TYPE": "MD","VALUE": "8"}

This information is then used to populate the wafer_nb field in the TDR (E-Test and Wafer-Sort testing stages), which is required for the Genealogy feature.
