V6.0 B154 (18 September 2009) (Top)
Yield-Man:
  • Oracle - New insertion modes: An environment variable (GEXDB_ORACLE_INDEX_INSERTMODE) now allows the user to select between 2 insertion modes:
    • DISABLE: the index on the xx_ptest_results table is disabled on the current partition. It is built just before changing to a new partition. This enables a faster and constant insertion time, but with the effect that the data extraction on recent data residing in the current partition is slower. A dialog box prevents the user that the extraction will be slower if part of the data he extracts is in the currentl partition.
    • NORMAL (default): the index on the xx_ptest_results table is not disabled on the current partition. For each insertion, the index is rebuilt by Oracle just after inserting the data. Data insertion gets slower as more data is inserted in the current partition .
  • Oracle - New partition management mode: An environment variable (GEXDB_ORACLE_SPLITLOTS_PER_PARTITION) now allows the user to specify the frequency at which new partitions should be created in terms of number of splitlots. Before, a partition was created every new day. Now, a new partition is created every new day, and every N splitlots (if N defined with the GEXDB_ORACLE_SPLITLOTS_PER_PARTITION environment variable).
  • Oracle - Automatic creation of new datafiles: A new datafile is now automatically added to the current tablespace if the current datafile becomes too big, so that a datafile won't exceed the 32Gb limit under Windows.
  • Datapump - Task execution window: The datapump task execution window ('Do not execute before', 'Do not execute after') used to be checked only at the time the datapump was executed (depending on the frequency). If inside the window, the datapump task would insert all files from the 'Data source' folder, even if not anymore in the task execution window. The task execution window is now checked at every file inserted (except when the datapump task is started manually with the 'Execute task now!' button), so that if the task is started when inside the window, but the insertion overlaps the window, it will stop at the first file outside the execution window.
  • Datapump - Post-processing action for delayed files: It is now possible, in the Datapump definition dialog box, to define a post-processing action (Leave, Rename, Move) for delayed files, the same way it can be done for files that have been successfully inserted, and for rejected files.
  • Case 2243 - PAT crash on multi-site file if not data on one site: Fixed a bug that would cause Quantix to crash when reading a multi-site file for PAT analysis if one site had no data.
  • Case 2234 - PAT Performance Degradation: Fixed a bug that would cause Quantix PAT Man to run very slowly under specific conditions when analyzing a file for yield alarm or PAT yield loss.
  • Oracle - Delayed files do not increment the splitlot_id: Delayed files used to increment the splitlot_id for the next insertion, this has been fixed.
  • Gex-Ftp - Truncated files after transfer: A bug has been corrected where some files may be truncated by a few blocks after transfer with Gex-Ftp.

V6.0 B153 (Beta) (7 August 2009) (Top)
Yield-Man:
  • Split Dataset by Two Combined Fields: Added a feature that allows for splitting a dataset by a combined database field: Lot;Wafer.
  • Merge Dataset Feature: Added a feature that allows for the merging of multiple datasets. This allows for calculating consolidated yield when there are multiple first pass and multiple retest splitlots.
  • GEXDB B9: added support in the GEXDB database and insertion procedures for Final-Test data files coming from multi-die parts.
  • Gex-LS - Display version number: Quantix License Status now displays the Examinator version number for connected clients.
  • Gex-Ftp - New option to prevent transfer of partial files: Gex-Ftp now has an option that allows the user to specify how long a file must be on the remote server prior to being downloaded the Quantix server. This prevents partial files from being downloaded.
  • Enterprise Report Save Script: A feature has been added to allow one-click save of the script used to create a Yield-man Enterprise Report.
  • Force Testing Stage: A variable has been added to support "forcing" data into a particular set of tables; E-Test, Wafer Probe, or Final Test.
  • Multiple Test Limits: A variable has been added to support datafiles that contain multiple test limits. Quantix will insert the minimum and the maximum limit within a single data file.
  • Out of resources error with MySQL: fixed a bug where a temporary table created during the wafer yield consolidation phase was not dropped, leading to MySQL 'out of resource' errors.
  • Yield & Bin Paretos:
    • Fixed a bug that would cause yield and bin paretos to display incorrect values in some circumstances.
    • Fixed a bug that caused the failing die contribution to display 'n/a.'
    • Fixed a discrepancy between Enterprise reporting of yield and detailed reporting of yield that occurred under certain circumstances, particularly when a query spans more than one month.

V6.0 B152 (Beta) (28 May 2009) (Top)
Yield-Man:
  • SYA computation: added an option to direct the SYA system to use the Gross Die count rather than the number of samples when computing wafer yield statistics.
  • SYA computation: added an option to direct the SYA system to ignore outliers when computing SYA limits.
  • Refuse to launch several Yield-Man (PAT-Server) instances: added a check to refuse launching several Yield-Man instances on the same system with the same GEX_SERVER_PROFILE.
  • Final Test binning consolidation: consolidation on software binnings has been dropped for Final Test. This is because it is not possible to guarantee a 100% correct consolidation on software binnings, as the only information available on final test re-test files is the list of hardware binnings that were re-tested. Yield-Man used to deduce the software binnings associated to each re-tested hardware binning to enable software binning consolidation, but there is no guarantee that this association is exhaustive in case of sampling.
  • Additional fields in the CSV export file: when exporting a data file to the Quantix CSV format, additional fields are written to the header (if populated in the data file):
    • ModeCode
    • RtstCode
    • ProtCode
    • CmodeCode
    • BurnTime
  • Gex-Ftp reading back invalid last modification date/time: the bug fixed in previous B151 release was not 100% fixed, the problem still happened under certain circumstances (some type of FTP servers, some specific date formats). It should now be cmpletely resolved.
  • SYA alarm check on insertion: fixed a bug where the SYA check on insertion used an invalid SQL query when the yield is to be computed using the summary data, rather than the samples.
  • Wafer Sort binning consolidation: fixed a bug where the wafer sort binning consolidation was wrong when a re-tested bin was not present anymore in the re-test data.

V6.0 B151 (Beta) (10 April 2009) (Top)
Yield-Man:
  • Final Test yield consolidation: the Final Test yield consolidation function has been improved to handle following situations:
    • after re-test yield consolidation, more parts are found than at initial test: the additional parts are considered new untested parts that were merged with the parts to re-test.
    • after re-test yield consolidation, more parts are found than at initial test: the missing parts are considered dammaged parts, and if a binning is specified in the config.gexdbkeys file (Ft_YieldConsolidation_MissingPartsBin variable), they are affected to this binning.
  • Final Test yield consolidation: the config.gexdbkeys file supports a new keyword for Final Test test/re-test yield consolidation:
    • Ft_YieldConsolidation_MissingPartsBin= <BinNb>: enables to specify the BinNb that should be used to hold missing parts between initial test and re-test (ie parts dammaged after initial test and before re-test).
  • Gex-Ftp reading back invalid last modification date/time: fixed a bug where for some Windows Ftp servers, the Gex-Ftp tool was sometimes reading back invalid last modification date/time (for files having 'last modification time' < 10AM). As a result, these files were never downloaded.
  • Yield-Man processing empty ZIP files: fixed a bug where Yield-Man exited when processing an empty ZIP file, instead of simply rejecting it.

V6.0 B150 (Beta) (17 March 2009) (Top)
Yield-Man:
  • Re-Test management: the config.gexdbkeys file supports new keywords to specify the re-tested binnings:
    • RetestBinList = all: all parts (PASS and FAIL) from the previous test level (initial test or previous re-test level) have been re-tested.
    • RetestBinList = all_pass: all PASS parts from the previous test level (initial test or previous re-test level) have been re-tested.
    • RetestBinList = all_fail: all FAIL parts from the previous test level (initial test or previous re-test level) have been re-tested.
  • SYA: the first version of the Quantix SYA feature has been implemented.
  • Gex-Ftp transferring no files when reading corrupted server date/time: on some type of Ftp servers, when retrieving the date/time of the server, at certain hours of the day, Gex-Ftp retrieves an invalid date/time. As a consequence, only old files are transferred (older than the corrupted date/time). The issue has been worked-around by using current UTC date/time -12 hours for the server date/time when reading back a corrupted date/time.
  • Gex-Email save settings: fixed a bug where the 'Save settings' button in Gex-Email remained disabled after changing the settings in the 'Smtp' tab.
