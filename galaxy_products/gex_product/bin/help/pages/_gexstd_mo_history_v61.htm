V6.1 B156.08 (21 July 2010) (Top)
Yield-Man:
  • Case 3440 - Yield-Man daily admin task generates thousands of emails: Yield-Man mistakenly generates tens of thousands of emails when the system clock is synchronized or changed. Gex-email just sends the emails without checking the dates which can be 100 years in the future.

Yield-Man:
  • Case 3288 - Gex-FTP transfer error - corrupted date/time property: Error due to unsupported datetime format from FTP server.

Yield-Man:
  • Case 3235 - Problem parsing CSM E-Test files: CSM E-Test parser rejecting files because of new foundry id on the first line.

V6.1 B156.05 (11 June 2010) (Top)
Yield-Man:
  • Case 3224 - The 'Settings' check implemented in case 3150 was not triggered: The 'Settings' check implemented in case 3150 is triggered when Gex-Ftp detected a new Gex-Ftp version. This detection is done using the new 'GexFtpVersion' field, but the settings check was not performed if this field is not present, so it was not performed for the first version implementing this check (B156.04).
  • Case 3167 - GEXDB insertion error message. More than one limit per site.: When inserting multi-site data files into a GEXDB, if 2 tests have the same test number, but a different test name, the insertion is rejected with following error message: "Insertion failed: Error in plugin Quantix SQL Database [Plugin V1.2 B9]. More than one limit definition per site for test number".

Yield-Man:
  • Case 3150 - Some ftp connections in GEX-FTP are giving an error "Changing directory failed": Some ftp connections in GEX-FTP are giving an error "Changing directory failed".

  • Case 3085 - FT yield consolidation incorrect in Enterprise Report: The consolidated ft yield numbers are correct for examinator report, but incorrect for the Enterprise Report. The FT yield consolidation was not correct when second level re-test data (Retest_index > 1) was inserted

V6.1 B156.02 (18 May 2010) (Top)
Yield-Man:
  • Case 3060 - Keep report on server option for all tasks: This option is not working correctly for the reporting task.

B153 processes the typeA and typeB files as well as converts them to csv. Will not convert to another format like stdf. The nd1_14 file appears to be a datalog and opens with a text editor.
  • Case 2911 - File causing crash: File with .dat extension does cause a crash. It's a binary format of some sort, not supported by Examinator.

V6.1 B156.01 (30 April 2010) (Top)
Yield-Man:
  • Case 3021 - Gex-Ftp reports date/time errors: On certain Ftp servers, Gex-Ftp doesn't parse the date/time property correctly and issues a transfer error ("Read N files with corrupted date/time property").

V6.1 B156 (29 April 2010) (Top)
Yield-Man:
  • Case 2982 - Add WaferID as OUT parameter in insertion_validation stored procedures: The goal is to enable the user to modify the WaferID in the xx_insertion_validation stored procedure (et_insertion_validation, wt_insertion_validation), teh same way the LotID can be modified. If the customer's xx_insertion_validation stored procedure does not have this addditional parameter, it will be called without.
  • Case 2899 - Links to reports in notification emails can use a HTTP URL: A new filed in the 'Web Site' task ('URL to reports') allows the user to specify a HTTP URL to be used for the link in the notification email (ie http://servername/path.pdf). The usual 'Reports directory' field is used to physically create the rerport on the local system, whereas the 'URL to reports' field, if not empty, is used to generate the link in the email body.
  • Case 1929 - Tests with 'test not executed' flag not inserted into GEXDB: When inserting into the GEXDB, tests having the 'test not executed' flag set in the STDF file (PTR record) were not inserted into the database, even if some measurement result is available.
  • Case 2933 - Only two tests appear in the file: File contains 57,000 some odd tests, and 52,000 of them are number zero. All are MPR tests. Customer would like to be able to convert to CSV, but only tests not numbered zero display.
  • Case 2891 - Date returned by the FTP server is not supported by the Gex-Ftp: Date returned by Japanese Solaris 8 Ftp server is not supported by the Gex-Ftp (it is not properly interpreted neither by the command prompt ftp client, but is correctly interpreted by FileZilla).
  • Case 2886 - Oracle error when inserting small files: Somertimes, when inserting small files into an Oracle GEXDB database, the file would be delayed due to a 'name is already used by an existing object' error when executing a 'CREATE GLOBAL TEMPORARY TABLE <table>' SQL query.

V6.1 B156 (preview 20100301) (Top)
Yield-Man:
  • Case 2737 - STDF files with NAN test results do not insert into GEXDB database: Any STDF file with NAN (not a number) floating point values as test results (ie PTR.result field) will not insert into the GEXDB database. It will be rejected with a SQL error on a SQL insertion statement.
  • Case 2728 - YM Crash, and error message prevents restart: Windows error message dialog box when YM crashes, requiring user interaction, and preventing YM to be automatically re-started.

V6.1 B156 (preview 20100217) (Top)
Yield-Man:

  • Case 2328 - Corrupt files causing PAT server to crash: Bad data file causing Quantix to crash. Requested enhancement is to add fault tolerance so that this type of file is rejected, but that Quantix continues to run.

V6.1 B156 (preview 20100126) (Top)
Yield-Man:
  • Case 2581 - Add a filter to reject files with too few parts: A filter was added in the DataPump task definition GUI ('Other' tab) to specify a minimum number of parts under which a file should be considered incomplete and thus rejected. This minimum number can be expressed in nb of parts, or in a percentage of the GDPW (Wafer Sort only: Gross Die Per Wafer).

V6.1 B155.04 (12 January 2010) (Top)
Yield-Man:
  • Case 2034 - New option to email the insertion log files daily: A new option has been implemented to email the insertion log files and a small insertion summary to the Yield-Man admin. To activate this option, create an 'Auto Admin' task (if it does not already exist), edit this task, enter a valid email address in the 'Sys. admin email' edit box, and check the 'Send daily insertion logs' check box.
  • Case 2553 - Optimized wafer consolidation (RDB): The SQL queries used during the wafer consolidation phase when inserting re-tested wafers into the GEXDB database have been optimized. On huge MySQL GEXDB databases, they could generate a timeout, where the optimized version executes in a few seconds.
  • Case 2555 - Wafer consolidation not working under Oracle (RDB): The wafer consolidation function executed when a re-tested wafer is inserted into a GEXDB running under Oracle was not working as expected.

V6.1 B155 (22 December 2009) (Top)
Yield-Man:
  • Case 2210 - E-mail sender can now be changed for all tasks: Added the capability to more precisely specify the YieldMan server alert emails' sender address. Now it is possible to specify a sender address for each individual YieldMan task.
  • Case 2460 - Gex-Ftp with new 'Scan sub-folders' option: Added the ability to scan subfolders on the remote ftp location using Gex-FTP.
  • Case 2446 - Option added in the datapump to reject files with too few parts: A new tab has been added in the Datapump definition dialog box ('Other'), which allows the user to define a minimum number of parts below which the file will be rejected. A value of -1 disables this option.
  • Case 2230 - Gex-FTP to search remote subfolders: Enhanced the capability of GEX-FTP to search subfolders on the remote folder to find files that match the specified criteria to return to the YieldMan server.
  • Case 2395 - Revised Database Administration Page: The Database Administration page has been updated to include additional valuable information such as 'Connected', 'Db version', and 'Update Status.'
  • Case 2442 - Save button for Enterprise report script does not actually save files: Fixed a bug that prevented the 'Save Script' button from properly saving a script.
  • Case 1909 - Handing current tasks during shut down: Enhanced the manner in which Quantix YieldMan manages existing processes when the user asks for a shutdown. Previously, it was possible to shut down YieldMan while an insertion, reporting, or some other task was underway. This had the potential to lead to system instability and corrupt data in the database. With the new method, Quantix will offer a dialog box to the user when a shut-down is requested and a process is underway. Typically, the user will allow Quantix to complete its current task and then shut down automatically. If necessary, the option is available to kill the process and shut down immediately.
  • Case 2229 - Yieldman executing the wrong script: Fixed a bug that would cause Quantix Yield man to automatically run a recently executed report, even if a report script was not saved and loaded into a Yield-man task.
