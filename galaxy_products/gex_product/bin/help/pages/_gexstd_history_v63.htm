V6.3.16 (12 July 2011) (Top)
Examinator:
  • Case 4986 - Examinator does not recognize the merge on test name option: When analysing a file with a particular test name that has multiple test numbers associated with it, it appears once for each test number even when the option to merge tests on test name (ignore test #) is chosen.
  • Case 4960 - Csv converter does not recognize the merge on test name option: When converting a file with a particular test name that has multiple test numbers associated with it, it appears once for each test number even when the option to merge tests on test name (ignore test #) is chosen.
  • case 4925 - Better support UMC data file format: GEX did not support a flavor of the UMC E-Test format. Instead of rejecting the file, the parser reads the first line as a wafer id. Therefore we get 26 wafers, one of which has the wafer id "WF#". The parser has been modified to support this format.
  • Case 4882 - Error connecting to a GEXDB from a system, if the link has been created on a system using a different Endianness: If a GEXDB link has been created from a system using a specific Endianness (ie Windows using little endian), the connection will fail if attempted from a system using a different Endianness (ie Linux using big endian). The same applies when connecting to the YM Admin DB.
  • case 4829 - UMC - PCM Parser: The parser recognizes the UMC PCM file. However, it does not load properly. Only the first column of data is loaded. If line 5 is deleted from the csv file, then the data loads properly, however X and Y position information is lost.
  • case 4825 - GEX crashes if the list of databases is refreshed during manual insertion: While running GEXDB without the YM Admin DB, if manually inserting multiple files into a RDB database, and trying to refresh the list of databases during the insertion, the application will crash.
  • case 4809 - 7C7 PCM Parser error: 7C7 file not converted.
  • case 4737 - File crashes the toolbox data editor: File crashes the editor in the toolbox because of too many columns.
  • case 4732 - File cannot be converted by converter: This file is converted but all the tests have no measured values in the resulting csv file. if the file is analyzed in gex, the results are there.
  • case 4691 - Convert to csv is not functioning properly: Convert to csv doesn't show up MPR records when Multi-Results Parametric option is set to "Do NOT merge".
