GCORE-5913 Incorrect number of test executions are displayed in Statistics Report when using the mapping option This bug occurs when the mapping option is used to add new tests defined in an xml file. GCORE-5437 Examinator generates an incomplete shift report When using the shift report feature available in GEX-Pro, you must load your dataset and declare your samples and control groups. Depending on the order in which the files are loaded, it may happen that Examinator doesn't create the shift report correctly. This problem was caused by an error in the initialization stage before applying the algorithm.
GCORE-5342 Bad welcome window when activating a standalone licence for the first time When activating a standalone licence for the first time, Examinator displays a bad welcome window and prompts the user to enter his/her name and product key ID. It was due to a misreading of js_lp.xml and js_lp.json files.
GCORE-3947 Group Name is changed to default when changing the file order in the "Compare files" GUI. When running a "Compare files" analysis and changing the default group name (DataSet_1) to something else, then trying to change the order of the files using the up/down arrows, the group name goes back to its default (DataSet_1).
GCORE-1158 Too long processing time to sort the interactive test stats There is a very long processing time to sort the interactive test stats table with many tests. The time was not that long in previous versions of Examinator like 6.5 and 7.0. The same kind of problems is seen when using the tests pick list in the interactive charts.
GCORE-1027 Allow for a wafer_id and lot_id mismatch between E-Test and Wafer-Sort in genealogy As a product engineer, I like to have the entire wafer scribe in order to identify my wafer. The wafer_id (string) and lot_id fields can eventually be different between E-Test and Wafer-Sort. This should not be an issue for the genealogy feature, which should use the wafer_nb (numerical) and tracking_lot_id fields to match E-Test and Wafer-Sort wafers.
The genealogy feature was failing in case of a lot_id or wafer_id mismatch between E-Test and Wafer-Sort, even though the tracking_lot_id and wafer_nb were matching.
GCORE-911 Loading of interactive table takes too long When using the interactive tables for the first time, a user must wait too long before seeing the tests displayed. The response time was better in V7.0. Users expect to see the interactive test table almost instantaneously.
