Single file analysis

To analyze data from a single file, all you have to do is:

  • 1: Launch the Analyze one file assistant and select the file to process.

  • 2: Edit the filter by double clicking the file name (or clicking the properties button) to edit which test site to process, and/or which parts to analyze (e.g. Bin1 only, Failures only, specific list of parts, etc.).

  • 3: Go to the next assistant page to define what to include in your report (settings), then have it created. Default report includes test statistics tables, Soft bin wafermap, and histogram of all tests.

For more details, see also the 'Data' tab description

Map Test Numbers

From the Properties page, you can select a text file (comma separated format) that includes the list of tests to map to new test indexes. This is quite useful when you need to compare data files created on different test environment, or need to merge files with test numbers no longer matching because of new tests added. Full details about the file format can be found in the relevant scripting section
