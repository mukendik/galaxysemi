Options tab

You may never even need to look at the Options tab! Since all fields are set to a default value that matches the most frequent needs, you will only have to read the following paragraphs if you want to review/edit the report and environment preferences. From this page, you can change the information listed in the report tables, set new markers to be added to the charts, etc.

Here are the options preferences you can edit:

Section: Databases (Examinator Pro)

This section is only available under Examinator Pro and lets your define Database related options.

Sub-section Value / Description
Locations

Lets you define the paths to the local and server databases folders

 

Section: Report output

Lets you define whether the report should be HTML format (default, strongly recommended) or ASCII .CSV (allows easy export to Excel-like applications). This section also lets you define where reports should be created (you can provide a fixed custom location if you want).

Sub-section Value / Description
Output format

Lets you determine if the report is to be created in HTML (default) or .CSV ASCII format. HTML is the most convenient format for sharing report forms, charts and tables, while .CSV is convenient for extracting the data used in the charts and tables.
Home page contents

Lets you customize text and logo to display on the cover page of your report (only applies to reports in Word, PowerPoint and PDF formats)
Paper size

Lets you define how page breaks in reports must be computed, based on your printer page format: A4 or Letter
Truncating test names size

Lets you enable test names truncation in reports. Allows to reduce the table size when test names are very long with unnecessary mid-strings. Truncation keeps first 32 characters only.
Output location

Lets you decide where the report files are created. If the report is HTML format, a folder named 'galaxy_examinator' will be created in this location (all report pages are visible from the index.htm page located in this folder). If the report is .CSV format, a single .csv file will be created in this location.

 

Section: Speed Optimization

This section (only applies to Examinator-Pro) lets you customize preferences that can improve the report generation speed.

Sub-section Value / Description
Data points available under 2D/3D interactive charting

Lets you define if Examinator collects all data samples so to allow you interactive charting, or only collect samples if the dataset selected is not too large (eg: 50 Mbytes). Examinator will always be able to create the HTML and Excel reports and charts independently of the selection made here. If you disable the data points collection, or process a dataset that exceeds the cutoff limit, then the Interactive charting will only include data samples for parameters selected in the 'More reports' of Examinator's settings page.
Computing advanced statistics

Lets you decide if Examinator computes the advanced statistics information (Skew, Kurtosis, Quartiles and Median). These parameters can require significant CPU time to compute, and are better disabled when you process large datasets (as you probably don't focus on these information anyway). Note that advanced statistics are computed form data samples, therefore forcing this option to 'always compute advanced statistics' doesn't have any effect if the above option (data points available under interactive mode) is disabled.
Data queries, use summary records

Only applies to Examinator Pro, lets you define if Examinator reads all data points or rather uses its internal summary records when running queries over very large amount of data. If you enable this option, you can customize the data set size threshold over which summary records are used instead of data samples. Processing speed is typically 10 to 100 times faster; ideal when running bin trend queries over a few months period.

 

 

Section: Data Processing / Handling duplicated tests

Lets you define if Examinator merges tests with same test number and different test name or keep them separate. Possible selections include:

Sub-section Description
always merge Makes tests with same test ID (test number) be merged, even if test names are different
never merge Do not merge test with same test number, but not matching in name.

 

Section: Data Processing / PartID display

Lets you define if Examinator extracts and displays data based on the PartIDs or only as samples. Used for characterization missions mostly.

Sub-section Description
Hide PartID Default mode: Examinator displays samples based on the position. E.g: First plot on trend chart is first data value from file, etc...
Show PartID Mostly used for characterization when trend charts need to display data sorted based on the PartID. Allows to display the first PartID even if it doesn't appear first in the data file. Also allows to display to which PartID each data point (in trend charts) belongs to.

 

Section: Data Processing / Sorting mode

Lets you decide how test data must be processed (processed in the order of files entered, or sorted by testing date). Possible selections include:

Sub-section Description
none Data files are processed in the order they have been selected: from the first in list to the last in list.
By date Data files are processed in the order of creation date. 


 

Section: Data Processing / Failures count mode

Lets you decide how test failures must be counted. Possible selections include:

Sub-section Description
Continue On Fail Count ALL tests failures found in the execution flow (default).
Stop On Fail ONLY count the first test that failed in the program execution flow.


 

Section: Data Processing / Multi-results parametric tests

Lets you decide if parametric tests with multiple results should be merged under one common name, or if each pin should be assigned a different test number. Possible selections include:

Sub-section Description
Merge Have all results (pins) of a parametric test merged under a same test name (default)
Do not merge Have each pin of a multi-results paramteric test have a unique test number.
e.g: 1.101, 1.102, etc.


 

Section: Data Processing / Handling STDF compliancy

Lets you decide how Examinator must behave when processing STDF test data binary files. Possible selections include:

Sub-section Description
Stringent Examinator expect the STDF files to be fully compliant with the STDF specification documents
Flexible Examinator does its best to smoothly and quietly process STDF data files (even if not fully compliant with the STDF specifications). This is the default operating mode.

 

Section: Data Processing / Intermediate files (STDF & Zip)

Lets you decide where to intermediate files are created when uncompressing files prior to the analysis or when importing non-STDF data files. This only applies to test data files analyzed under Examinator, not Examinator queries.

Sub-section Description
Same folder... Examinator creates intermediate files in the same folder as the original data file selected (default)
Other location... Examinator creates all intermediate files into the custom folder you define. Note: Intermediate files are always removed when closing the Examinator application. Select this mode if your non-STDF data files to analyze are stored on a folder where you don't have write access.

 

Section: Data Processing / Outlier removal

Lets you select the outlier removal policy: none (default) if all test results should be processed and computed for Cp, Cpk, Mean, Range, etc. You can define the outlier policy as a percentage of the test limits Any test result outside the specified window is ignored when computing the statistical values, and is classified as an outlier value. Possible selections include:

Sub-section Description
none No outlier removal enabled. All data are considered valid and are used to compute statistical results (Cp, Cpk, Mean, etc...)
100% Only tests within 100% of the test limits are used, tests outside 100% of the test limits are ignored. This selection basically makes you only process good test results.
150% Only tests within 150% of the test limits are processed. This is a +/- 75% limit space from the center of the test limits. This makes you compute all good parts and failing parts only up to 25% away from each test limit.
200% Same as above, but for 200%
250% Same as above, but for 250%
300% Same as above, but for 300%
1 sigma Only tests within the 1 sigma space (centered on the mean) are processed.
2 sigma Only tests within the 2 sigma space (centered on the mean) are processed.
3 sigma Only tests within the 3 sigma space (centered on the mean) are processed.
4 sigma Only tests within the 4 sigma space (centered on the mean) are processed.
5 sigma Only tests within the 5 sigma space (centered on the mean) are processed.
6 sigma Only tests within the 6 sigma space (centered on the mean) are processed.
7 sigma Only tests within the 7 sigma space (centered on the mean) are processed.
8 sigma Only tests within the 8 sigma space (centered on the mean) are processed.
9 sigma Only tests within the 9 sigma space (centered on the mean) are processed.

 

 

 

Section: Test statistics

Examinator always generates a test statistics table in its report. From this section, you can detail how statistics must be computed (from the data samples found in the file, or from the summary section if it exists), which information you want to be displayed, and how it should be sorted. You can also define alarms that can be triggered when you compare sets of data. Alarms can apply to any of the following:

Alarm type Description
Mean drift Allows setting an alarm level when you compare data samples and the mean drift exceeds a specified value (a percentage of the limit space). If the alarm level is reached, the mean shift will appear on a red background in the statistics report table for HTML reports, or with an *F* for .CSV reports. The value specified must be the maximum percentage of drift allowed, for example: 5%.

Formula: MeanDrift = 100*(Mean - ReferenceMean)/LimitSpace
Note: Any negative number disables the alarm
Sigma drift

Allows setting an alarm level when you compare data samples and the sigma drift exceeds a specified value (percentage of value drift). If the alarm level is reached, the sigma shift will appear on a red background in the statistics report table for HTML reports, or with an *F* for .CSV reports. The value specified must be the maximum percentage of drift allowed, for example: 1%.

Formula: SigmaDrift = 100*((Sigma/ReferenceSigma)-1)
Note: Any negative number disables the alarm
Cpk drift Allows setting an alarm level when you compare data samples and the Cpk drift exceeds a specified value. If the alarm level is reached, the Cpk shift will appear on a red background in the statistics report table for HTML reports, or with an *F* for .CSV reports. The value specified must be the maximum percentage of drift allowed, for example: 5%.


Formula: CpkDrift = 100*((Cpk/ReferenceCpk)-1)
Note: Any negative number disables the alarm

 

Section: Wafer map & Packaged map

This section lets you customize your wafer maps (wafer sort data) or packaged map (packaged parts tested) reports.

Wafermaps to include: Lets you decide how to deal with reports with multiple maps. E.g.: only display the stacked / combined wafers, or display all individual maps as well.

Chart size: Lets you define the chart size policy:

  • Small: image is 200x200 pixels, allows ultra fast processing. Right selection for a quick overview.
  • Medium: image is 400x400 pixels, allows more resolution but takes 4 times longer to create than a small image. 
  • Large: image is 800x800 pixels. allows the highest resolution but takes 16 times longer to create than a small wafer map. To be used only when few wafer maps have to be created (e.g.: 1-10).
  • Auto:  Lets Examinator decide on the chart size depending of the number of charts to create. This mode is strongly recommended as it lets the software find the best compromise. 

Markers: Let you add markers to the chart. Includes:

  • Outline retest: draws a black outline for all dies that have been tested more than once.
  • Write bin: writes the binning value onto each die.
 

Section: Histogram (advanced histogram only)

Chart size: Lets you define the chart size policy:

  • Small: image is 200x100 pixels, allows ultra fast processing. Right selection for a quick overview.
  • Medium: image is 400x200 pixels, allows more resolution but takes 4 times longer to create than a small image. 
  • Large: image is 800x400 pixels. allows the highest resolution but takes 16 times longer to create than a small histogram. To be used only when few histograms have to be created (e.g.: 1-10).
  • Auto:  Lets Examinator decide of the chart size depending of the number of charts to create. This mode is strongly recommended as it lets the software find the best compromise. 

Chart type: Lets you define how to draw the histogram:

  • Bars: this is the classic histogram display mode. Each class is drawn as a vertical bar.
  • Fitting curve: this draws a smooth curve that envelops the histogram classes. Very convenient mode when comparing distributions.
  • Bars + Fitting curve: above both drawing modes are combined.

Markers: Let you add markers to the chart. Includes:

  • Title: writes a title at the top of the chart.
  • Mean: draws a line to show the test mean (if applicable)
  • Test limits: draws lines to show the test limits (if applicable)
  • 2 Sigma: draws 2 lines showing the space at +-1 sigma around the test mean (if applicable)
  • 3 Sigma: draws 2 lines showing the space at +-1.5 sigma around the test mean (if applicable)
  • 6 Sigma: draws 2 lines showing the space at +-3 sigma around the test mean (if applicable)

Section: Gage R&R / Box plot

Fields in chart: Lets you define the fields to list in the table by the box plot images:

  • Limits: shows test limits
  • Mean: shows test mean (and mean shift in % when comparing groups of files)
  • Sigma: shows test sigma (and sigma shift in % when comparing groups of files)
  • Cp: shows test mean (and mean shift in % when comparing groups of files)
  • Cpk: shows test mean (and mean shift in % when comparing groups of files)
  • Repeatability: Gage R&R EV information
  • Reproducibility: Gage R&R AV information
  • R&R: Gage study information
  • Part variation: Gage R&R PV information
  • Total variation: Gage R&R TV information

Sorting mode: Lets you decide how the table must be sorted. Can be one of:

  • Test number: Table sorted with increasing test numbers
  • Test name: Table sorted with increasing test names
  • Mean: Table sorted with increasing mean
  • Sigma: Table sorted with increasing sigma
  • Cp: Table sorted with increasing Cp
  • Cpk: Table sorted with increasing Cpk

 

Section: Pareto

Lets you define cut-off limits so only tests with a Cp and/or Cpk lower than a given limit are listed.

 

 

Section: Binning

How is binning computed: Lets you define if the binning summary report is computed using the binning summary information found in the test data files (default), or is computed from the wafer map report:

  • From summary data (if any), otherwise from samples: This selection is the default. It works well and should be used in all situations except for wafer test results that include retest data. In such a situation, the summary information may be incorrect as it sometimes counts all parts tested including retested parts. So if you are in such a situation, use the option below.
  • From wafer map report only: Use this option ONLY if you want the binning charts created from the wafer map image in your wafermap report. In this mode, only the last bin of each die retested is counted, making the binning summary and Yield information always accurate. This requires that you force the wafer map report to be created (on hard or soft bin).

 

 

Section: Datalog

Format of the datalog output: Lets you define if the datalog output organizes data in one or two rows:

  • 1 row: Data (test results) are organized in a single row
  • 2 rows: Data (test results) are organized in two rows

 

Fields to include chart: Lets you define the datalog fields to include with test results:

  • Comments: include all comments section (may sometime create huge reports...)
  • Test number: include test numbers
  • Test name: include test names
  • Test limits: include test limits and units
  • Die XY : include XY die location for each part tested.

 

Section: Environment preferences

Lets you customize Examinator environment information such as:

  • Text editor: Either use the default editor or select the custom application to use for text editing
  • Spread sheet editor: Either use the default spreadsheet editor or select the custom application. Typically: wordpad  under windows, textedit under Solaris/HP-UX, and gedit under Linux (Open Office suite) to use for spreadsheet editing. Typically: excel under windows, and oocalc under Unix & Linux (Open Office suite)
  • Close application...: Define a timeout period after which the Examinator application closes (only applies to Examinator client applications running a floating license).
