License Activation

  • Standalone: uses local CPU license

  • Client/Server: uses floating license

Standalone mode

Select the 'Standalone' running mode to start the process of activating the release for this specific computer. You'll then have to:

  1. Enter your full name

  2. Enter the Product Key (given to you when you placed your P.O.)

  3. Click the 'Next' button

  4. Email the file gex_request.txt created to gex.license@galaxysemi.com

  5. Within few minutes, you will receive your license file and activation instructions (email auto-reply)


Client/Server mode

Select the 'Connect to server' running mode to start the software and have it request a floating license from the Examinator-Pro license server. You'll have to:

  1. Enter the license server name or IP address

  2. Enter the communication port # (Socket port#)

  3. Click the 'Next' button

Arguments at launch time

The Examinator-Pro application supports launch time arguments that can be very convenient. Note than under Unix, the application name is the shell "gexpro" while under windows it is "gex.exe"
 
Examples of command lines:
Line Description
gexpro -WPRO Displays the Examinator-Pro Welcome page. Lets you reset the running mode and/or server name+port number.
gexpro -S <file> Examinator-Pro has to run a script <file> at launch time
gexpro -SH <file> Examinator-Pro is Hidden (minimized) when running the script. If this option is not specified, the Examinator-Pro application is visible during the script execution.
Note: This option requires the S option to be set
gexpro -SCH <file> Examinator-Pro is Closed once script execution is completed. Useful when you want a script
to be executed periodically and have the Examinator-Pro launched every X minutes using a
timer application (e.g. crontab, task windows, etc.).
Note: This option requires the S option to be set

eg: gexpro -WPRO will let you display the Examinator Pro welcome page and allows to select any running mode available.


GEX-LM License manager

The gex-lm application handles all the floating license management and authorization mechanisms. It must be running on the server prior to running any client application (Examinator-Pro or GEX-LS). This application communicates with the other software using the TCP/IP protocol. The only setting you may want/need to change is the socket port number used for the communication channel. If GEX-LM is launched without any argument, the default socket port 4242 is used. To force the application to use a different port, simply launch the application as follows:

gex-lm -port <PortNumber>     where <PortNumber> is the decimal Socket port number to use.

eg: gex-lm -port 4243

GEX-LS License status

You can see the current license usage at any time launching the client application gex-ls. When executed, simply specify the server name (or IP address) that is running the gex-lm application, and the socket port number used to establish the communication (default is 4242 unless you have specified a custom number when launching the gex-lm application on the server).

 

Laptop users: installation prerequisites

When running Examinator-Pro software for the first time, a license request file with the computer hardware identification profile is created (except if running in Server mode). If your laptop sometimes connects to a docking station, or if you have set different hardware profiles on your computer, it may happen that Examinator Pro refuses to run if you do not follow the correct license activation process. Here is the process you MUST follow to ensure a smooth Examinator-Pro Standalone or Evaluation license activation on your laptop:

  1. Have the Examinator-Pro software installed on your notebook (you can be using a docking station at this stage, it's not a problem)

  2. Disconnect your laptop from its docking station

  3. Log to your laptop and activate your off-line profile if you have one (so as to have your computer in the exact same situation as when working disconnected from your network)

  4. Run Examinator-Pro , select the Standalone (or Evaluation) running mode and enter your Product Key ID then press the  Next button.

  5. Examinator-Pro will create the gex_request.txt license request file with the correct hardware profile information

Note: If you didn't follow the above steps, simply restart from step one, so your previous gex_request.txt gets updated.
