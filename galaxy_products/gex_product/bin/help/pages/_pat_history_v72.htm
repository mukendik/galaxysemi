Enhancements
Case Number Subject Description
6384 Yield-Man active-passive failover mechanism Active-passive failover mechanism requires the load balancing, and in addition a master-slave mechanism where the passive node is sleeping until the master node (or one of the master nodes) goes down. If the master node (or one of the master nodes) goes dfown, it becomes activ until the master node (or all master nodes) is back up and running.
6639 enhance xml mapping file to work with mpr records user would like to map new tests using the xml mapping function on mpr records such that the new test is created for each pin in the mpr record.
7216 Shift Analysis - Add a Column called 'Polarity' for polarity of shift, Preserve chronology, and sort by Max % The customer would like a column called 'Polarity' so they can sort by worst negative shift and worst positive shift. Currently it appears the shifts are absolute values, so the user cannot tell which way the shift occurred.
7248 many native KVD xml files do not support all tags in the spec provided update the kvd xml parser to allow for missing tags that are considered non-essential.
7323 Make the map geometry check optional when merging maps Currently we can merge external and STDF maps in memory but we impose to have an external map greater than the STDF map. Make this rule optional.
7324 Merge external map with STDF map before PAT, improve merge feature to allow merging external map smaller than STDF map Currently the merge feature is such that the STDF map should be contained into the external map. If this is not the case, an error message will be thrown saying that the map geometries don't match. It would be required to allow merging external maps smaller than the STDF map, while keeping the current functionality so that this verification can still be done if the customer needs it.
7325 Merge external map with STDF map before PAT, post PAT output wafer map and STDF should both contain bins from external map and STDF sources For the merge feature allowing to merge an external map with a STDF map before the PAT process, it is required to allow the user to output a post PAT stdf and a post PAT wafer map which will contain bins from both sources (external and STDF). Currently only the output wafer map contains the bins from the two sources.

7539 create an interface to connect to a daemon and interact with it for all daemons (PM, YM and GTM), have an app or api that can connect to a daemon and stop the scheduler (YM/PM) or prevent new connections (GTM) so that the user can see the status and safely stop the daemon for upgrades and server maintenance.

Test number is maintained as the MPR.test_num field as today. a new column to display MPR.rtn_index is added to differentiate the pins/index element for the mpr.
