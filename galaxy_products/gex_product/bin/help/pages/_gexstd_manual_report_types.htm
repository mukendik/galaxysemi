Report type Description
Statistics
Lets you define how to create the table with test statistics. Tables can be one of the following:
  • Disabled (no test statistics table created !)
  • All tests (it's the default): Have ALL tests listed in the table.
  • Failing tests: Only include tests that failed at least once.
  • Tests with outliers: Only include tests that have one or more outliers (outlier removal definition is set in the Options page).
  • List of tests : Specify the list of tests to report (e.g.: 1 to 10, 15, 19 to 23)
  • Tests with Cp lower or equal to... : Report all tests with a Cp lower or equal to the value specified.
  • Tests with Cpk lower or equal to... : Report all tests with a Cpk lower or equal to the value specified.
Wafermap, Packaged map
 		Lets you create a wafer map chart (wafer sort data) or packaged map chart (packaged devices). The chart can be one of the following:
  • Disabled (no map is created !)
  • Software binning (it's the default ; it uses the SOFT bin results)
  • Hardware binning (it uses the HARD bin results)
  • Parametric over test limits: shows how a specific test result behaves over the whole wafer; in this mode you need to enter the test number to monitor. Die color covers the whole spectrum: from blue if value is very close to the lower limit, to red if the value is very close to the upper limit. If your test is missing a limit, Examinator uses its Min or Max value instead. 
  • Parametric over test results: same as above except color spectrum covers the range of test values instead of the limits; can provide a more detailed view.
Histogram

Lets you create test results histogram charts (18 bars) for the list of tests selected. Charts can be one of the following:
  • Disabled (no chart created !)
  • Chart over test limits: for each test, a histogram that covers the test limits space is created.
  • Cumulate over test limits: same as above except each column includes the cumuli of the previous columns (this causes the last column to always be100%).
  • Chart over test results: is the default. For each test, a histogram that covers the test values space is created.
  • Cumulate over test limits: same as above except each column includes the cumuli of the previous columns (this causes the last column to always be100%).

About Fitting curve: More advanced histogram charts (such as to draw the histogram fitting curve), use the Advanced histogram report instead (in More reports).

Note: In all cases, you specify if you want the charts created for all tests or a specific list. You can reduce the histogram generation to your own list of test. E.g.: 1 to 10, 15, 19 to 23

Histogram (Bar chart)

 

Define bars / fitting curve rendering from the Options page.

 

Lets you create a more advanced histogram than in the option presented above. This histogram provides sharper (30 bars) and more accurate data as it is intended for investigation work--it also allows you to display additional markers and information as seen in the Options page. The default Examinator Options settings is to adapt the size of the image chart created depending on the number of charts to create: as number of tests to chart increases, the chart size decreases. This is a convenient feature as when you do a first pass with all tests, the result is a mosaic of small histogram images, then focusing on a subset of tests makes the Examinator increase the chart size created. If multiple files are compared, plots appear on the same chart with one different color per chart plotted.

Fitting curve: You can have the histogram draw bars, the fitting curve or both (simply select your chart type from the Options page!). The "fitting curve" drawing mode is the most convenient rendering type when comparing multiple distributions.

As with the standard histogram, you can select one of the following modes: Chart over test limits, Cumulate over test limits, Chart over test results (this is the default), Cumulate over test limits.

As for other reports, you can specify to create the report for all tests, or a specific list of tests. E.g.: 1 - 10, 15, 19 - 23
Trend (Control chart)


(Test result trend chart)


(Software binning trend chart)

Define Lines / spots drawing mode from the Options page.

Lets you create a trend chart of test results for the selected tests/binnings - the more tests you want the longer it takes! 

Test charts can be one of the following:
  • Chart over test limits: for each test selected, a test results trend chart that covers the test limits space in Y-axis is created.
  • Chart over test results: is the default. For each test, a test results trend chart that covers the test values space is created.

Note: In both cases, you specify if you want the charts created for all tests or a specific list. As for other reports, you can specify to create the report for all tests, or a specific list of tests. E.g.: 1 - 10, 15, 19 - 23

  • Chart difference between 2 tests: Specify the 2 tests to analyze, and have the difference of the two tests plotted (Test1-Test2). Chart is plotted over test results.

Note1: The test#1 is ALWAYS read from the first file/group of files, and the test#2 is ALWAYS read from the second file/group of files. If only one file/group exist, then both test results are extracted from the same file/group. If more than two files/groups of files exist, only data from the first two files/groups of files are listed.

Note2: If the number of samples found for the test#1doesn't match the number of samples found for the test#2, the minimum of the two sample set is used. Extra samples are simply ignored.

Binning charts can be one of the following:
  • Software binning (Sub-Lots): Specify the binning # or binning list to chart. Draws one plot per sub-lot. Useful report to trend yield in production. 
  • Software binning (all parts): Specify the binning # or binning list to chart. Draws one plot per part tested, each plot being the current binning yield level.  Useful to see how a binning yield evolves over the parts tested.
  • Hardware binning (Sub-lots): Same as Software binning (Sub-Lots) but for hardware binning instead. 
  • Hardware binning (all parts): Same as Software binning (all parts) but for hardware binning instead. 
  • Soft bin + rolling yield: Specify the binning # or binning list to chart. Two charts are plotted: one shows the current binning yield level, while the other one is the rolling yield every 200 parts.  Useful to spot sudden yield drops as the rolling yield is much more reactive to yield changes.
  • Hard bin + rolling yield: Same as Hard bin + rolling yield but for hardware binning instead. 

You can enter any list of binning to trend. E.g.: 1-5 to trend the combined yield of bin 1 to bin5.

 
Correlation (Scatter diagram)

Lets you create a XY-Scatter diagam of test results for the given tests. Convenient report when performing correlation analysis. Charts can be one of the following:

  • Chart over test limits: for each test selected, a test results scatter diagram that covers the tests limits space is created. This lets you see if the process is well centered or not.
  • Chart over test results: is the default. For each test, a test results scatter diagram that covers the tests values space is created. This is like zooming over the data.

Note1: In all cases, you specify the test to chart on the X-axis, and the test to chart on the Y-axis. For the two tests entered, ensure the format is: XXX or XXX.YYY where XXX is the test number and YYY the pinmap index (if any).

Note2: The test shown on the X-axis is ALWAYS read from the first file/group of files, and the test shown on the Y-axis is ALWAYS read from the second file/group of files. If only one file/group exist, then X and Y results are extracted from the same file/group. If more than two files/groups of files exist, only data from the first two files/groups of files are listed.

Note3: If the number of samples found for the test in X doesn't match the number of samples found for the test in Y, the minimum of the two sample set is used. Extra samples are simply ignored.

Multi-scatter plots: If you want to chart more that one scatter plot, then Examinator offers the possibility to enter the test pairs instead of only two tests. To have multiple scatter plots created from your pair of tests, simply enter the list of pairs into the 'Test in X' field and leave the 'Test in Y' field empty. Let's suppose you want to create three scatter plots as following:
plot#1 with test 1000 in X, test 2000 in Y
plot#2 with test 1001 in X, test 2001 in Y
plot#3 with test 1002 in X, test 2002 in Y

Then the 'Test in X' field should include the following string:
1000,2000 ; 1001,2001 ; 1002,2002

You can also import your list of pairs from an ASCII file by clicking the 'file open' icon. The file format is very simple and detailed below in Appendix 1.
Production Yield reports

The Production reports allows to easily generate Yield/Volume yield charts. From the 'options' tab you can define which plots to overlay (Yield, Volume, both) and select markers (yield level, volume count). The X axis of your Yield/Volume chart can be one of:

  • Sublots: each chart cell represents one sublot (or wafer).
  • Lots: each chart cell represents one lot (merging all sublots and/or wafers).
  • daily: each chart cell represents one day (merging all lots tested the same day).
  • weekly: each chart cell represents one week (merging all lots tested the same week).
  • monthly: each chart cell represents one month (merging all lots tested the same month).
Gage R&R - Boxplot

Lets you create a Gage R&R boxplot in only one click!. Convenient report when performing correlation analysis, and focusing on repeatability and reproducibility. The Gage R&R Boxplot chart shows candle-style graphics where extreme values are the min. and max. samples values, and the candle showing the 1st and 3rd quartiles position. If the analysis is conducted over only one dataset or file, then the viewport is normalized over the test limits space. If comparing multiple datasets or file, then the viewport space is the range of values measured for all datasets/Appraiser compared. Charting can be one of the following:

  • All tests: is the default. All tests are shown on the boxplot chart.
  • Test list: Lets you generate a boxplot chart for the list of tests specified. Test number, and range syntax is the same as for the other report types supporting a test list.

For this report, both limits must exist but since this is not always the case, a simple algorithm is used to create artificial limits for the purpose of charting:

  • If no low limit exist, we compute LowLimit = Minimum/2, where Minimum is the minimum test result value found in the data samples.
  • If no high limit exist, we compute HighLimit = Maximum*2, where Maximum is the maximum test result value found in the data samples.
Datalog

Lets you extract test results for the specified tests (all tests, failing tests, specific range of tests, etc.). As this output report may be huge, be careful to only extract the information you need, and activate a filter such as good parts only, or a range of parts to reduce the amount of data listed in the datalog output--you can quickly consume megabytes of pages! Datalog type can be one of the following:

  • All tests: all tests are listed in the datalog page; you should be very cautious when using this option as it will probably create huge page that will consume megabytes of space!
  • Failing tests: is the default. Only failing tests are listed in the datalog page.
  • Outliers: lets you define the  which tests outliers to be dataloged.
  • Test list: lets you define the list of tests to report in the datalog page. E.g.: 1 - 10, 15, 19 - 23
  • Test list: values only: Same as above except that only test results are listed; any other information such as part number, and binning result are not listed. This mode is convenient if you only want test result values to be listed.
Guard banding. What-if

Yield is the number of good parts you test...and yield will vary if test limits are changed. In the same way, yield will be affected if test distributions change. Such scenarios can be easily simulated using this drilling selection. You can play on test limits and see the impact on Cp, Cpk and yield. This function is present in both GEX-Instant Report (kept for historical reason) and GEX-Data Mining [more].

Pearson's correlation: Test Time Optimization

This report lists all test pairs that highly correlate (includes: correlation ratio, test number, test names, and scatter plot for the two correlating tests). When two tests strongly correlate, you may consider removing one of them from your test flow and therefore reduce the tests in your program as well as its overall execution time. The higher the correlation ratio is (1.0 is the maximum), the more tests correlate. Tests that with a low correlation value are not listed in the report ; the threshold used to classify tests with a high correlation can be customized under the Options tab. Default level is 0.8
Note: if comparing groups or dataset, this report will only process data from the first group / dataset.
PAT-Man History

 

This report extract the PAT-Man report found in the data file (report created by the PAT-Man solution when identifying outlier parts). This report provides full details about parts identified as outliers, the faulty tests, their value and the recipe settings used. This is a precious report for any company that wants to remain ISO while using PAT, or any engineer looking after historical PAT information.

 

Appendix 1: Test list file format

This section details the structure of the ASCII file you may create to define a list of parameters you can have Examinator import for you (saves valuable time when your list includes many parameters or when you use the same list periodically under Examinator).

Any line of the file starting with a # character is ignored and is considered a comment line. Other lines must include the parameter lists as detailed hereafter. When Examinator reads your file, it is not case sensitive , spaces are ignored, semicolon or coma have the same effect.

Note to scripting language users: if you're using the Examinator scripting language and specify a file path to the test list to import, remember that the backslash character needs to be doubled in the string.

A such, the following line is NOT correct:
gexReportType('histogram','test_over_limits','c:\temp\tests.txt');
 

While this line is correct:
gexReportType('histogram','test_over_limits','c:\\temp\\tests.txt');

 

Following cases studies show typical content of test lists file. Cases cover the histogram, trend and scatter plots types:

Case#1:
If your file is to hold tests for Histograms or Trend charts, then each line of the file can include list of tests in the usual format supported by Examinator. Example:

#List of parameters for Histograms
1000-2500, 2600, 2900-3100


Case#2:
If your file is to hold tests  for Scatter / Correlation plots, then each line must specify test pairs and must comply with the following format:
X=<List of tests in X> , Y=<list of tests in Y>

Where the <List of tests...> includes one or more tests coma-separated. Test numbers can include pinmap number as well (eg: 1000.16).

Example1: Two scatter plots

#Scatter plot#1
X=1000 , Y=2000
#Scatter plot#2
X=1000 , Y=2010

Example2: Same effect as Example1, but using a more compact definition

#Scatter plot#1
X=1000 , Y=2000,2010
 

Example3: Same effect as Example1, but using a more compact definition

#First 7 scatter pots with same parameter in X
X= 6001 , Y = 6004, 6006, 6008, 6019, 6033, 6047, 6053

#Next 7 scatter pots with same parameter in X
X= 7001, Y = 7004, 7006, 7008, 7019, 7033, 7047, 7053
 

Example4: All the scatter combinations for 4 tests!

#Build 4 scatter plots!
X= 6001 , 6002 , Y = 6004, 6006
