Standalone mode

Select the 'Standalone' running mode to start the process of activating the license for this specific computer. You'll then have to:

  1. Start the activation utility delivered within this package.

  2. Enter the activation code that you received by email. If you do not have it, you may also find it under Quantix Semiconductor Product and License Center (https://galaxysemiuat.flexnetoperations.com/control/glxy/index)

  3. If you have an internet connection, your license will be activated and installed in a few short minutes. If you do not have internet access, please generate a request.xml file and upload it under the 'Quantix and License Center/OfflineActivation' section. You will get a response.xml file and you will need to load it into the activation utility to activate and install your license.

  4. Now you can run your Quantix product in the standalone mode.


Client/Server mode

You'll have to: Select the 'Connect to server' running mode to start the software and have it request a floating license from the Quantix vendor daemon.

  1. Enter the license server name or IP address

  2. Enter the communication port # (Socket port#)

  3. Click the 'Next' button

    The activation process is the same then described previously in 'Standalone mode' but it is need to be done in the server machine

GEX arguments at launch time

The GEX application supports launch time arguments that can be very convenient:

Line

Description

gex -W

Displays the Welcome page and allows you reset the running mode and/or server name/IP and port.

gex -S <file>

Examinator has to run a script <file> at launch time

gex -SH <file>

Examinator is Hidden (minimized) when running script. If this option is not specified, the Examinator application is visible during the script execution.
Note: This option requires the S option to be set

gex -SCH <file>

Examinator is Closed once script execution is completed. Useful when you want a script
to be executed periodically and have the Examinator launched every X minutes using a
timer application (e.g. crontab, task windows, etc.).
Note: This option requires the S option to be set




Quantix Vendor daemon and License server manager

The “galaxylm” vendor deamon handles the floating license management and authorization mechanism. It must be running on the server prior to running any Examinator client application (Examinator, Examintor-Pro, ... ).

To launch the “galaxylm” vendor deamon, you must first set the GEX_SERVER_PROFILE enviroment variable.This variable should point to a location in which the daemon can store configuration and log files cleanly. Then run “lmadmin” utility that starts and maintains vendor daemons as required for serving license rights. This utility is delivered with this package or you can download it at http://www.globes.com/support/fnp_utilities_download.htm#downloads.

The basic command to run it : lmadmin -adminOnly no -allowRemoteStopServer yes

This application communicates using TCP/IP protocol. The only settings you may want/need to change is the socket port number. If lmadmin is launched without any argument, the default socket port 27000. To force “lmadmin” to use a different port, just add -licPort portNumber argument when starting the application.

You can also configure this utility through a web interface accessible at: http://hostname:8090 where hostname is the host name/IP of the machine from which lmamdin is run and 8090 is the default web port. (Do not confuse this port with -licPort option.) This port can be changed just by adding the -webPort webPort argument at start up.

Once "lmadmin" is running, start the "galaxylm" vendor deamon either by web interface "Administartion->Vendor Daemon Configuration->Import license". You can also start "galaxylm" directly when launching "lmadmin" by adding the -import galaxylm\galaxylm.lic to the arguments list at start up.

For more information please review the "LicAdmin.pdf" file delivered within this package."

License status

You can view the current license usage at any time by launching the activation utility or using the "lmadmin" dashboard web interface.
