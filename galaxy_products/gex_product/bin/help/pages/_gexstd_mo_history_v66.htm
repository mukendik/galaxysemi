V6.6.02 (20 July 2012) release history (Top)
Yield-Man:
  • Case 6362 - SPEA parser: update for SPEA340 and SPEA372 format: The SPEA tester has a variation in output between * the AtosC Datalogging - Specifications (Release : 1.40) * the SPEA340 * the SPEA372. Update to the parser to accommodate all this 3 formats..
  • Case 6304 - HVM parser update for bin mapping: the parser for the bin mapping files at vishay for hvm searchs only Bin Names which starts with "001" and ends with "/S/L". need to update the parser such that this is a valid line: 1,1,001 ,PASS.
  • Case 6357 - Linux bug causing Yield-Man to close unexpectedly: A very large .tar.gz file should be rejected but instead causes ym to close unexpectedly.
  • Case 6300 - Gross die not updated correctly: With V6.5, the Gross die is not updated as expected when a wafer is inserted through several files, and no specific gross die overload method is used. The gross die takes the value of the nb_parts from the first file inserted.
  • Case 6258 - converter_external_file.xml does not support multi lines: When using more than one line to define the BinMap file path, the converter_external file reader cannot open the BinMap file.
  • Case 6232 - Duplicate data in sbin summary table causing bad sya alarms: NoPAT data is being inserted into the database such that the wt_sbin_stats_summary table has two entries for one splitlot.
  • Case 6128 - Parser for Maxwell Freescale format: New Parser for csv format for Freescale whcih uses an external file for the bin #'s and names.
  • Case 5376 - Bin 0 is good but being reported as bad in GEX: GEX reports bin 0 as a failed bin but if you dump the file to csv, the software bin 0 is associated with hardware bin 1 and all these die are pass die.

V6.6 (10 May 2012) release history (Top)
Yield-Man:
  • Case 6071 - TestNum/TestName: key managment: When the PinName/SequencerName appears into the TestName, YieldMan spits the Test for each PinName. Yield-Man must provides a tools to clean the TestName and merge all splitted Test with the good TestName.
  • Case 6050 - 1 out of n sampling for parametric queries: Implemented 1 out of n sampling for parametric queries.
  • Case 5885 - Store email report condition/products per user in admin DB: Store list of conditions (products/tester/...) per user in admin DB : for each user, offer a way to associate a list of conditions (products,...) (available or not in any production DB) per user.
  • Case 5863 - Generate Report on trigger: Allow a new GTF/JS trigger action to create a report (and email it, or leave it on the server) WITHOUT needed to setup an SYA rule for the product.
  • Case 6028 - [MySql] To InnoDb converts all tables even if table is already in InnoDb Engine: The process to convert a MyIsam database to InnoDB engine doesn't process correctly tables already in InnoDb engine.
  • Case 6007 - [ORACLE] data insertion error with empty HBIN_NAME: When using a GexDb B17 from scratch, the insertion process fails when inserting empty hbin name into wt_wafer_hbin_inter.
  • Case 5979 - When using a gexdbkeys as a suffix for a specific file name the keys file gets deleted after it is read: For each data file found, Yield-MAN looks if the same name but with extension ".gexdbkeys" exists (data file configuration file). If so, this configuration file is used to overload the data file header info with the one read from the configuration file. This file is erased by Yield-MAN after being successfully read.
  • Case 5968 - No date fields in consolidated table if meta_data on start_t field defined: No date fields in wt_consolidated_wafer table if meta_data on start_t field defined (Y for consolidated_field).
