V6.2 B157 release history


V6.2 B157.13 (15 December 2010) (Top)
Yield-Man:
  • Case 4158 - Update to CSV Parser - some tests have no names or data: When some testName appears more than one time in a same run with a different testNumber, runs are incorrect and some test appears without the good testName.
  • Case 4141 - Update to SEMI E142 parser: SEMI E142 parser now include wafermap origin location.

V6.2 B157.11 (24 November 2010) (Top)
Yield-Man:
  • Case 4075 - Update to Mcube parser: Start time is taken from the DTG of the first part in the file. add finish time and take it from the last DTG in the file so that the probe time is no longer 0.
  • Case 4040 - Add site # capability to the Amida parser: Add the column to the parser that allows the site # column.
  • Case 3948 - DSTIM file won't process: Update the Verigy dstim parser such that unknown event types are ignored.

V6.2 B157.10 (10 November 2010) (Top)
Yield-Man:
  • Case 3749 - Verigy 93000 parser: The Verigy 93000 parser was enhanced to support files with data prior the TestFlowStartEventType marker.

V6.2 B157.09 (03 November 2010) (Top)
Yield-Man:
  • Case 4008 - KLA/INF parser enhancement: KLA/INF parser has been enhanced so that the setup_t, start_t, and finish_t timestamps are populated with current date/time at time of conversion, rather than with 0.
  • Case 4006 - Change MCube parser so that if good bins are not defined in the header, the file is rejected: If no good bins are defined, do not assume all bins are bad, reject the file instead.
  • Case 3973 - GDF files are crashing YM: Some GDF files are crashing Yield-Man on insertion. Crash only occurs when inserting into an RDB.

V6.2 B157.08 (27 October 2010) (Top)
Yield-Man:
  • Case 3974 - Incorrect detection of Adjacent edge die: Some adjacent edge die are not correctly detected when applying the GDBN algorithm.
  • Case 3966 - Excel converter truncates digits: When I convert our stdf datalog files to csv format with the converter, the data in the csv file is truncated to 6 significant figures. We are datalogging register values, and need the full value (up to ten digits, plus sign). If I do an STDF Records Dump (from the Quantix Toolbox), I can see the full number of digits, so I know the data is correct in the stdf file.
  • Case 3900 - Update to Mcube Parser: Provide parser update per provided documentation."
  • Case 3855 - CSV conversion has fewer delimiters than expected: When converting stdf to csv, the commas end on each part when the part has no more measurements. Even though there are more parameters. So that comma count for parameters is 889 but the comma count for PID-2 for example is 885 & PID-4 is 886. In previous versions, the comma count for each part matched that of the parameter count.
  • Case 3730 - CSV export doesn't export all the data: CSV export exports only the first line of data when tests have no name.
  • Case 3722 - Yield-Man and GEX crash if temporary STDF file already exist: When converting from a suported format other than STDF, Yield-Man and GEX crash if the target STDF file already exist.

V6.2 B157.06 (03 September 2010) (Top)
Yield-Man:
  • Case 3698 - Add a File_Out argument to the script called after DataPump insertion: The script that can be called after Yield-Man inserted a file through a DataPump ('Other' tab, 'Execute batch on insertion') will be passed an additional 11th argument, which is the destination file name (after the inserted file has been moved, renamed...).
  • Case 3685 - Application crash on ATDF->STDF conversion: On a specific ATDF file, the application crashes during the ATDF->STDF conversion. This caused Yield-Man to shut down during execution of the datapump. Manually analyzing the incriminated file with Examinator also causes a crash.

V6.2 B157.05 (30 August 2010) (Top)
Yield-Man:
  • Case 3527 - YM insertions slow down over time: YM insertions slow down significantly over time, ie going from about 30'' to about 10' in 24 hours of YM execution. Better control over 'Process events' messages during insertions, and more Trace messages to track the issue.

V6.2 B157.04 (04 August 2010) (Top)
Yield-Man:
  • Case 3505 - STDF files generated by PAT-Man are missing parametric data: A bug was introduced in late B157 previews and final B157 version where the STDF files generated by PAT-Man after PAT processing only contain the test result records (PTR, MPR, FTR) for the first run of each site.
  • Case 3500 - YM appears to attempt to insert data file that doesn't exist: YieldMan writes bogus entries to log file. The entries are written when YieldMan performs an RDB extraction, and then attempts to create the report.

V6.2 B157.02 (21 July 2010) (Top)
Yield-Man:
  • Case 3440 - Yield-Man daily admin task generates thousands of emails: Yield-Man mistakenly generates tens of thousands of emails when the system clock is synchronized or changed. Gex-email just sends the emails without checking the dates which can be 100 years in the future.

V6.2 B157.01 (13 July 2010) (Top)
Yield-Man:
  • Case 3411 - Problem with duplicate FT consolidated Soft bins after B10 update: The consolidated yield #'s from the enterprise report, and the report center don't match. There are also multiple FT soft bins with the same # and similar names when pulling a product bins details report.

V6.2 B157 (06 July 2010) (Top)
Yield-Man:
  • Case 3194 - Bin names disappear when you run a consolidated Gex report on either Wafer or FT data: Bin names disappear when you run a consolidated Gex report on either FT or Wafer data. They show up fine when running a "raw data" extraction.
  • Case 3163 - Add wafer ID to the FT_RUN table: Add wafer ID to the FT_run table so that wafer maps can be made from final test data.
  • Case 2939 - SYL/SBL Line Item recipe editor phase 2. New SYL/SBL algorithms: This enhancement would allow for individual line items to be edited in the SBL editor. In this phase, please provide for some new algorithms for SBL calculation:
    • Median +/- N* RobustSigma.
    • Percentile rank. (Fail 0 to 1 percentile and/or 99-100 percentile).
    • Q1 & Q3 +/- N*IQR.
  • Case 2807 - Progress bar while connecting to databases: A nice item to have would be a progress bar that displays while Quantix is attempting to connect to an external database. On startup, Quantix sits non-responsive for a 10-15 seconds while it attempts to find its database connections. A progress bar would prevent frustration in trying to click on items while Quantix is not responding.
  • Case 2735 - GEXDB B10 - Database format incremental release: Quantix database enhancements, required to support new ER reports (originaly planned under JAsper); + other DB enhancements
  • Case 2350 - Create a tool, such as trigger file, to run SYL/SBL at the Probe-Lot level: Provide the ability to run SYL/SBL at the wafer probe lot level. This requires a mechanism for informing Quantix when to run SYL/SBL. A trigger file mechanism has been implemented to support this feature.
  • Case 1794 - Check databases only once at startup or when going to database admin: The database connections should be checked only once at startup and when clicking the 'Reload' button in the 'Database Admin' page.
  • Case 3299 - Gex-FTP sequence stopped unexpectedly if a timestamp file is in a subfolder: Gex-FTP sequence stopped unexpectedly if a timestamp file is in a subfolder and the option ""scan subfolder"" is checked. Error message: FTP error: Removing file failed: /Remote_Folder/gex_ftp_timestampfile.temp.txt.gex: The system cannot find the file specified.
  • Case 3256 - Wafer consolidation tables in gexdb b10 are not populating correctly: When a retest of a wafer is inserted before the original test of the wafer, it uses the last inserted (test) instead of the retest die counts in the wt_consolidated_wafer table. Therefore it is not using the testing date, for populating this table.
  • Case 3223 - 'LOCKED' gexDB in Quantix plugin: When an error appear in consolidation/update process, functions are returned WITHOUT unlocking tables. Consequence is that the gexbd is totally locked inside gex until restarting.
  • Case 3180 - File wrongly rejected due to unexpected PASS bins: When the option to reject files with PASS Hard bins not in specific list is activated on Bin 1, Eagle wafer files get rejected, because the first PIR/PRR run is not a real valid run, but only contains statis test info. The binning is 65535, and considered PASS.
  • Case 3107 - Errors when updating to Gexdb B10: When updating to Gexdb_B10 there are errors due to the indexes in all the ptest_info, and mptest_info tables.
