V7.5.0 (Top)

V7.5.0-RC1 (Top)
Bug fixes
Case Number Subject Description
GCORE-8008 When a negative value is used for SYA value N During limit computations, when N is negative, SYA cannot compute according to any algorithm and may crash.
GCORE-7986 Examinator stops when processing certain ATDF files ATDF files with certain characteristics can cause Examinator to crash.
GCORE-7943 Ability to convert files with long fields in PLR into STDF The ATDF to STDF conversion feature is now able to process files with long fields (greater than 1024bits) in the PLR record.
GCORE-7894 Fix a memory leak in ATDF to STDF converter Fixed a memory leak occuring when converting GDR record from ATDF to STDF format.
GCORE-5821 Ignore custom views when purging data from the database During a purge action to remove bad splitlots, data in custom views is now ignored.
GCORE-4855 Indexes added to the TDR In order to improve the speed of consolidation for very large databases, additional indexes have been added to the consolidation tables.

Enhancements

To allow the upgrade, the users must be able to be forcibly disconnected from the administration database without disrupting their non-database Examinator usage.
GCORE-153 Update the SYA gui to all easier analysis of results The SYA simulation results are all shown in different significant figures. This enhancement forces all the results to 2 places after the decimal to make the report easier to read.
Bug fixes
Case Number Subject Description
GCORE-7038 The INCREMENTAL_UPDATE task can return a PASS or a DELAY or a FAIL status but this status doesn't appear correctly into the events logs This bug corrects an error in the incremental update status into the Events log.
GCORE-6789 Insertion speed issue with TEST_ALLOW_DUPLICATE_TESTNUMBERS option Improve the insertion speed when TEST_ALLOW_DUPLICATE_TESTNUMBERS global option is turned on.

GCORE-6733 Multi-Limit loading Issue when analyzing multiple files/splitlots When doing a 'Merge files' on multiple files or a 'Analyze a Dataset' or 'Merge Datasets' on (a) dataset(s) containing multiple splitlots, the tool will display N*M sets of limits, where N is the number of files/splitlots and M is the number of limits in each.
GCORE-6707 IFF Converter generates STDF files with test results only for the 256 first tests If the IFF file has more than 256 tests, the converted STDF file has the test information (i.e. test name, number) but no resulting data. In Examinator-Pro, the statistics table has n/a or 0 for all the statistics for the tests after the 256 first tests. In Yield-Man there is not ptest_results for the tests after the 256 first tests.
GCORE-5299 When running 20 Yield-Man instances, some Yield-Man instances stop inserting data When 20 YIeld-Man instances are running, Yield-Man is creating obsolete rows in ym_actions table that put Yield-Man instances in a waiting mode. To avoid this bug, the small ym_actions table was moved from the TokuDB engine to the InnoDB engine.
GCORE-4391 Yield-Man cannot find user Admin When trying to connect to the ym_admin_db user gets the error "cannot find user Admin"
GCORE-3283 Handle and repair summary records that contain bad "ÿ" character(s) When STDF files are repaired by software tools, these tools sometimes introduce a special character "ÿ" in HBR.BIN_PF and/or SBIN.BIN_PF records, which cause problems if this special character is inserted into the Test Data Repository. To avoid problems, Yield-Man will automatically handle and repair summary records that contain bad "ÿ" character(s).

Prior to this bug fix, this module strips out comments and empty lines from the SQL file. The bug fix will preserve these comments.
GCORE-6213 Custom PCM parser produces an invalid part PASS/FAIL flag when there is no HBin/SBin information If the PCM file does not contain any HBin/SBin information, all parts are flagged as PASS with HBin=SBin=0 even if they contain parameter values out of limits.

GCORE-6093 Custom parser has a multi-limits data loading issue There is a problem with the way multi limits are managed during the conversion process from a customer data format to STDF. For instance, the customer data file contains a test with four test limits. After converting this customer data file to STDF, there are only two DTR records with test limits for this test when there should be four DTR records.
GCORE-6085 There is a data scaling issue for data inserted from IFF data files As an example of this bug, a selected test inserted from an IFF data file has a high limit = 13 micro Amps, but in Examinator the high limit = 13 pico Amps.

GCORE-5890 Parser bug for Advantest T66xx .log datalog file Examinator doesn't load data from Site 1 for particular tests, but can load data of Site 2 on a Advantest T66xx datalog file.
GCORE-5781 Bin counts are not properly inserted into the Test Data Repository (TDR) When inserting data into the TDR where one bin is recorded multiple times in the STDF, the last record for this bin overwrites previous records. For instance, bin 12 is known to contain 3003 parts but if the last record for bin 12 has a count of 0, the summary count for bin 12 is 0, which is an error.

V7.5.0-gamma (Top)
Enhancements
Case Number Subject Description

Bug fixes
Case Number Subject Description
GCORE-5916 Intermediate consolidation does not correctly store the start time for each test insertion at FT When inserting lot data into Yield-Man that has two or more test insertions, the consolidated data stores the earliest start time for the entire lot as expected. However, the start times for the intermediate test insertions are the same as the lot start time, which is not correct. Each test insertion should have a different start time.
GCORE-5475 The BINNING_Yield-Man BINNING_FT_FORCE_INVALID_PART_BIN_WITH option is active for data files with no invalid HBR or SBR part counts The new BINNING_FT_FORCE_INVALID_PART_BIN_WITH option
should only be activated if the test data file has an HBR/SBR with an invalid part count or if there is a PRR with invalid HBin and SBin part counts. However, this bug also activates the BINNING_FT_FORCE_INVALID_PART_BIN_WITH option when the HBR/SBR does not have invalid part counts, which it should ignore.
GCORE-5447 Yield-Man data converter pump creates file names with time stamps that cannot be sorted in chronological order When using the Yield-Man data converter pump to convert data files to STDF or ATDF or CSV using the "unique name" option, a time stamp is added to each file name. Currently, time stamps are in the format yyyyMMMdd.hhmmss, where MMM is a three-letter abbreviation for the month. Files with this time stamp format cannot be sorted in chronological time stamp order. This bug can be fixed by a new yyyyMMdd.hhmmss time stamp format, where MM is a two-digit representation of the month.
GCORE-5358 Intermediate consolidation does not correctly store the start time for each test insertion at FT When inserting lot data into Yield-Man that has two or more test insertions, the consolidated data stores the earliest start time for the entire lot as expected. However, the start times for the intermediate test insertions are the same as the lot start time, which is not correct. Each test insertion should have a different start time.
GCORE-5286 Wrong X-Y die size in G85 ASCII wafer map when rotating the wafer When a user generates a G85 ASCII wafer map, the X_SIZE and Y_SIZE values are not correctly updated according to the position of the notch. For example, if a user rotates a wafer map 90 degrees, the dimension of every die is rotated 90 degrees, and the X_SIZE and Y_SIZE values should therefore also change but do not change.

I make a 90° rotation, the X_size and Y_size values should be switched ... but are not in this format contrary to STIF for example.
GCORE-4176 PAT-Man or Yield-Man GUI shows that the scheduler is "Stopped", but it is not stopped If a user stops the Yield-Man (YM) scheduler using a "stop" command, the YM GUI shows that the scheduler is "Stopped." But the YM scheduler process is not stopped and continues to run and insert files.
GCORE-3884 gs-emaild daemon does not send email While gex-email -gui is able to send email, the gs-emaild daemon is not able to send email.
GCORE-3858 Unable to run the gs-emaild daemon script because of errors in the script The gs-emaild script has invisible ^M characters at the end of every line, which produces an interpreter error when run.

Enhancements
Case Number Subject Description
GCORE-5005 Support for SECSII map format: file updater function PAT-Man includes a file updater function for the SECSII map format.
GCORE-4930 Allow Yield-Man to insert data files with binning mismatch to avoid TDR-1210 errors To reduce the number of data files rejected by Yield-Man, the user has the option to allow Yield-man to insert data files without a perfect match between soft bins and hard bins. The following case is allowed: Invalid PRR.Soft_Bin and PRR.Hard_Bin with no PRR.Part_Flg

Data inconsistencies for Test# are avoided by merging the TestName while retaining the Test# of the TSR or the first PTR according to the TestName.
GCORE-4840 Allow Yield-Man to insert data files with binning mismatch to avoid TDR-1210 errors To reduce the number of data files rejected by Yield-Man, the user has the option to allow Yield-man to insert data files without a perfect match between soft bins and hard bins. The following case is allowed: Invalid PRR.Soft_Bin but with valid PRR.Hard_Bin.
GCORE-4423 New "DBH Code" field in the Dongbu parser The Dongbu parser should put the contents of this field into the oper_nam field and parse the file whether the field exists or not. If the field does not exist, oper_name is NULL.

GCORE-5081 A custom parser for final test is broken The bugs in this CSV final test parser are fixed.

If 'ok' is not returned by the script, Yield-Man does not log an error message. An error message is required to alert the administrator that the .js trigger file is not 'ok'.

GCORE-4436 Allow for the override of Sbin/Hbin mis-match checks Because it can be valid for a set of data to have Sbin/Hbin combinations that are not consistently mapping one to one, the check is now optional.
GCORE-4344 IFF FT custom parser: Examinator DB Query limits are 0 instead of n/a The Yield-Man database shows 'null' or empty for the spec limits, but in the Examinator interactive table, the spec limits are '0', 0 UA, or 0V depending on the units. Null or empty spec limits should appear as 'n/a' in the Examinator interactive table.
