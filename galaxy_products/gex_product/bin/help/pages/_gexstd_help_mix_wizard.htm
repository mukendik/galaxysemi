Compare groups of files

To compare groups of files, all you have to do is:

  • 1: Launch the Compare groups of files assistant and create the groups to compare.

  • 2: Select the files to process. Note that files are added to the group selected prior to clicking the 'add files' icon. All data files from the same group are merged into one big data cluster, and then each data cluster from each group is compared to the other groups. For example, this assistant allows you to compare many lots tested under different conditions and lets you easily compare the results.

  • 3: Edit the filter by double clicking a file name (or clicking the properties button) to edit which test site to process, and/or which parts to analyze (e.g. Bin1 only, Failures only, specific list of parts, etc.). Repeat this for each file as necessary.

  • 4: Go to next assistant page to define what to include in your report (settings), then have it created. Default report includes test statistics tables, Soft bin wafermap, and histogram of all tests.

For more details, see the 'Data' tab description

Smart Import: Query over Time Period

You may find it very convenient to import all files that match a date and filters criteria. This very powerful function lets you only import test data coming from files generated between specific starting and ending dates. Furthermore, you can also define filters such as tester name, lot name, device type etc... For full details about this function, check the Smart Import page.

: Click this icon to import files based on creation dates and filters.
 

Load/Save your File List

Examinator allows you to save your list of files and filters to disk so you can later reload it with just one mouse click! This is a very convenient feature that boosts your productivity on repetitive tasks involving the same data files.

: Click this icon to save your current list of files and filters to disk (saves a script file)
: Click this icon to reload a list of files and filters from disk (reads a script file)

Map Test Numbers

From the Properties page, you can select a text file (comma separated format) that includes the list of tests to map to new test indexes. This is quite useful when you need to compare data files created on different test environment, or need to merge files with test numbers no longer matching because of new tests added. Full details about the file format can be found in the relevant scripting section
