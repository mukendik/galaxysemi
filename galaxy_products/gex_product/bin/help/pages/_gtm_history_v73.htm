V7.3.00 (25 September 2014) (Top)
Bug fixes
Case Number Subject Description
GCORE-1810 GTM/PAT engine attemps to compute limits even on tests with no test results GTM/PAT engine attemps to compute limits even on tests with no test results. When the recipe contains lot of tests but only few of them receive test results from the tester, this could potentially slow down the process on some systems.
GCORE-1617 The Quantix Tester Monitor (GTM) daemon is closing unexpectedly The GTM log indicates that it is closing gracefully when it should not be closing.
GCORE-1526 envision gtl libraries update the gtl_get() function in the enVision wrapper was not functioning properly.
GCORE-1502 Allow gtl_set sqlite file name after init but before close Test engineers working on LTX MX test platforms do not have the STDF file name at the time when gtl_init is called. As a result, engineers cannot set the sqlite file name to be the same as the STDF file name. Note that gtl is the Quantix Tester Library for performing PAT testing at final test.
GCORE-1356 During dynamic PAT, some tests do not have tuned PAT limits A case was reported where dynamic PAT baseline and tuning operations were not performed for all tests. Some tests did not have tuned PAT limits.
