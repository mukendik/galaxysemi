GCORE-1085 Bin failure is missing in the PAT G85 ASCII map Software validation requires that all output files from PAT must contain the same results and must match actual results. An exception was discovered when comparing the test results in a PAT G85 ASCII map with a PAT STDF file. One bin failure was missing from the PAT G85 ASCII map, and the PAT STDF file was correct.
GCORE-979 JST Output Wafer Map is different from GTF Output Wafer Map The outlined die are different in the JavaScript Trigger (JST) output report compared to the Quantix Trigger File (GTF) output report. The outlined die indicate that the particular die has been retested. The input files to the JST and GTF are identical so the result should be identical.
GCORE-959 Change default file policy for GEX-FTP from "rename-file" to "Leave File" Our integration class is set up to help users walk through a Yield-Man integration. The current default file policy causes the following problem: When someone uses gex-ftp to download files with the current default, it automatically appends .gex to the end of the filename. When the next person tries to walk through the integration class, their gex-ftp will not grab the files because the downloaded files have the filename suffix .std.gex instead of .std .
GCORE-892 JavaScript Trigger does not allow custom output file name when using BuildPATReport method A new method has been created to generate a PAT report only when the number of outliers exceeds a PAT threshold. The name of this automatically-generated PAT report was generic: 'ym_report_<timestamp_date>.pdf'. Customers wanted a more useful name for the generated PAT report. Therefore, the JavaScript Trigger now allows users to make the name of the PAT report the same as the Quantix Trigger File. The JavaScript statement for assigning a name to a PAT report is: pp.Set('ReportName', 'TriggerName').
GCORE-862 The GexPATEngine.BuildPATReport method is generating a false error message When a user uses a JavaScript trigger file to create a PAT Report, the GexPATEngine.BuildPATReport method generates an error message that the output STDF file does not exist even when the STDF file does exist.
GCORE-811 The resulting post PAT wafer map in G85 ASCII Format contains the # character for AVI fails This bug appears when the following steps are performed: Create a new GTF (Quantix Trigger File) key to force any AVI fail bins to a specific bin number. After the auto alignment and merge process, and the PAT process, the resulting post PAT wafer map in G85 ASCII format will contain the # character for the AVI fails.
GCORE-797 Crash at start up of PAT-Man and Yield-Man Crashed in Linux rhel 6.3 environment.
GCORE-772 Older gex-lm and gex-ls license managers do not run in V7.2 and V7.3 on Linux computers The older gex-lm and gex-ls license managers do not run because they are missing software libraries. This bug affects Quantix license manager packages for Linux 32 bit and 64 bit computers.

GCORE-528 H8140: Compute Exclusion Zone can't be unchecked Once the compute exclusion zones from the 3D PAT rule is activated, we can't enable it anymore unless we set the bad bin threshold to 0. It is expected to deactivate this rule while keeping the bad bin threshold to a value different from 0.

GCORE-503 8129: NNR incrorrectly computes median if number of elements is even when the number of samples is even (like 2N), the NNR algo takes either the N or N+1 element while it should compute the arithmetical mean between N and N+1.
GCORE-468 H8112: ZPAT does not work as expected ZPAT does not work as expected. In this use case we use a ZPAT rule with a low yield threshold set to 30%, which corresponds to 8 wafers or more with bad bins on a given location out of 25 wafers of the lot, or 17 wafers or less with a pass bin. With such a rule on our Lot there should be NO ZPAT at all! Indeed the stacked wafer map, stacked on bin 1, shows that the number of stacked pass bins in each locations is in worst case 21 wafers which is far greater than the 17 wafers limit. This demonstrates that we should not get any ZPAT fail in our composite map. As of today the composite map actually shows there are failing ZPAT at any locations where there is at least 2 fail dies. This would correspond to a bad bin threshold of 8% instead of 30% which is a big problem since we're now issuing much more ZPAT fails than expected.
GCORE-120 Unable to run GTM V7.2 as a daemon on Linux Unable to run GTM V7.2 (using Qt5) as a daemon on Linux. This is due to Qt5 not supporting anymore a QApplication with a GUIenabled flag set to false.

Customer believes it is better to control the yield alarm levels directly in the PAT recipe.
