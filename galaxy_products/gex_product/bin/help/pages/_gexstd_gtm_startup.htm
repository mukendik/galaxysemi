Quantix Tester Monitor
License Activation

Client/Server connection

From the current connection page. You have to enter the details to allow GTM to connect to the gex-lm license manager process:

  1. Enter the license server name or IP address

  2. Enter the communication port # (Socket port#)

  3. Click the 'Next' button

Arguments at launch time

The GTM application supports launch time arguments that can be very convenient:
Line Description
gex -WGTM Displays GTM Welcome page. Lets you reset the running mode and/or server name+port number.

eg: gex -WGTM will let you display GTM welcome page and allows to select any running mode available.


GEX-LM License manager

The gex-lm application handles all the floating license management and authorization mechanisms. It is installed with it's own Quantix License Manager package. It must be running on the server prior to running any client application (GTM or GEX-LS). This application communicates with the other software using the TCP/IP protocol. The only setting you may want/need to change is the socket port number used for the communication channel. If GEX-LM is launched without any argument, the default socket port 4242 is used. To force the application to use a different port, simply launch the application as follows:

gex-lm -port <PortNumber>     where <PortNumber> is the decimal Socket port number to use.

eg: gex-lm -port 4243

GEX-LS License status

You can see the current license usage at any time launching the client application gex-ls. When executed, simply specify the server name (or IP address) that is running the gex-lm application, and the socket port number used to establish the communication (default is 4242 unless you have specified a custom number when launching the gex-lm application on the server).
