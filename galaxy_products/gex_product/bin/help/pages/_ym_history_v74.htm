V7.4.0 (1 July 2015) (Top)
Enhancements

GCORE-4178 Develop IFF data parser for PCME data files IFF data parser is needed to parse data from Process Control Monitoring (PCM) data files for insertion into the Yield-Man database. In this case, the data files are .pcme data files.

Inserted data should include retest_index, retest_phase, retest_hbins, and any other information required for lot and sublot data consoldation at final test.

Update the IFF data parser to read the information in the PAT_REPORT and insert it into the database so that users can query the database for this information.

V7.4.RC1 (2 June 2015) (Top)
Bug fixes
Case Number Subject Description
GCORE-3845 IFF data parser is extracting the wrong wafer ID The IFF parser assigns WAFER_NUMBER to the wafer_id field in the Test Data Repository (TDR), which is wrong. The IFF parser should assign WAFER_ID to the wafer_id field in the TDR, and WAFER_NUMBER can be ignored by the IFF parser.
GCORE-3721 Insertion of EDF file causes Examinator and Yield-Man to crash This EDF file causes many crashes. For instance, making a default Examinator report causes Examinator to crash.
GCORE-3687 Final Test LVM FETTEST data is incorrectly processed as wafer sort data In Examinator, Final Test (FT) LVM FETTEST data is processed as final test data, as expected. But when Examinator parses FT LVM FETTEST data in a PROMIS data file, it rejects this data with the error message "wafer not found in PROMIS info file."
GCORE-3044 Keycontent update for prod_data results in incorrect field value When the user updates xx_splitlot.prod_data using the keyscontent table via a stored procedure, Yield-Man tries to write 'TRUE' to the prod_data field. The prod_data field is defined as a single character and expects a value of 'Y' or 'N'. Consequently, data insertion fails.
GCORE-1239 Incorrect bin names appear in Report Center bin reports when splitting products with same bin number but different bin names When doing a Stacked Bin report at final test and splitting by Product ID, the soft bin names are all the same while they differ for each product. The report is expected to show the appropriate bin names.

Enhancements
Case Number Subject Description

GCORE-3035 Allow data insertion from STDF files with invalid HBin and SBin data in PRR Give Yield-Man users the option to allow data insertion from STDF files with invalid hard bin and soft bin data in the PRR when it can be inserted without major loss of data integrity. Some customers are willing to tolerate known defects in STDF data to be able to analyze the rest of the STDF data.
GCORE-2668 Do not ignore the PROCESS and the SPEC VER fields inTSMC WAT files Currently the Yield-Man parser ignores the PROCESS and SPEC VER fields in TSMC WAT files. But some customers need the data from these two fields inserted into their database.
GCORE-205 Grey out the GUI area for the js hooks when ym_admin_db is not in use If a user is not using ym_admin_db, the js hooks entered in the GUI will not be saved in the .gexmo_tasks.xml for use/reload later. In this case, the hooks will be lost when Yield-Man is restarted. To prevent users from losing their work, grey out the GUI area for the js hooks when ym_admin_db is not in use.
Bug fixes
Case Number Subject Description
GCORE-3544 Examinator query results reveal defects in the Sky IFF parser Query of probe summary extracted from an IFF data file does not include test program, part counts, and other critical parameters. The test names associated with test numbers do not appear to be valid.
GCORE-3543 Sky IFF parser is incorrectly inserting data into database as "NA" Data in various IFF fields (Product, Program Name, ...) is being incorrectly inserted into the database as "NA."
GCORE-3501 Sky IFF parser does not correctly handle "NA" as a lower limit When reading an FTPI file, the parser converts a lower limit with an "NA" tag to numeric zero.
GCORE-3453 Reduce the data insertion errors for e-test data files E-test data files are delayed during date insertion into the Test Data Repository. Error code is TDR-1308: Error executing SQL query with "ERROR = Unknown column 'HBIN_COUNT' in 'field list'."
GCORE-3418 Customer's custom CSV parser does not extract test names correctly This CSV parser extracts a partial test name instead of the full test name. For example: Given the test name VCC1_B13_POS, the parser extracts the test name as VCC1 instead of VCC1_B13_POS.
GCORE-3396 Yield-Man rejects data files from test programs executing a fail-flow Enhance Yield-Man to accept and process data files from test programs executing a fail-flow. Fail-flow is a group of additional tests executed by the test program if a part fails a trigger test for the fail-flow. For example,digital levels in a single functional test are used to test continuity across many pins in a single pattern burst; on failure, the test program executes additional parametric tests (fail-flow) to collect parametric data for the continuity failure.
GCORE-1988 Remove SPC/ATAG items in the Monitor menu in the Parameter Specs Monitoring Task window In the Parameter Specs Monitoring Task window, remove the SPC/ATAG items in the Monitor menu: "Run of 7 points..." and "Trend of 7 points..." These two menu items are not supported and do nothing when selected.
GCORE-1784 An expired Yield-Man task is still able to run from Quantix Trigger File (GTF) A Quantix Trigger File can trigger a task to run and generate an alarm even though the task is expired. Expected action is no alarms for expired tasks.
GCORE-75 SYA Task GUI hover message says to Enter ProductIDs (plural) when it can only monitor one In the Statistical Yield Alarms (SYA) GUI, the field labeled 'Products' is plural, and the hover tool tip says 'Enter the ProductIDs (plural) to monitor.' When the user clicks on the 'pick-list' icon to the right of the 'Products' field, it informs the user to select only one product. This is inconsistent and confusing.

GCORE-2754 Allow Yield-Man to point to different ym_admin_db servers If all Yield-Man instances access the ym_admin_db database via a single ym_admin_db server, this will create a single point of failure that would bring down all Yield-Man instances. To make the system more robust, allow Yield-Man instances to point to different ym_admin_db servers and not just a single, common ym_admin_db server.
GCORE-2138 Provide an option to overwrite and correct data in the database by reloading data from a modified STDF file Currently, Yield-Man rejects a data file if the lot_id, product_id, tester_id, and start_t are the same as a previous data file. V7.4 includes an option to disable this restriction. Using this option, an engineer can correct data inserted into the database from a faulty data file by correcting the data in the file and then reloading the corrected data file.
GCORE-2046 Custom parser for IFF data files This is a custom parser purchased by a customer.
GCORE-1395 Perform enhanced shift analysis of sample units compared to control units Produce a measurement shift report that compares known control units (Controls) to unknown production units (Samples). This comparison report identifies data sets (Samples verses Controls) with the greatest normalized measurement shifts.
