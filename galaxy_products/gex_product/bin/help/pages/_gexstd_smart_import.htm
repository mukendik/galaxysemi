Smart Import: Query over Time Period

The Merge files and Compare groups of files assistants include the Smart Import icon   that makes an automatic selection of all files that match a given time period (i.e.: data tested during this period). This function also lets you filter files based on information such as: tester name, program name, lot name, etc...

Following is a screenshot of the Smart Import assistant:

 

  • From: Lets you define the starting date (data tested prior to this date are ignored). You can enter the date using the date editor or click the calendar icon to browse and pick your date.
  • To: Same as above, but for the ending date (data tested after this date are ignored).
  • File location path: The software will look for data file starting from this directory. Sub directories will only be analyzed if the 'Look in subfolders' option is checked. Enter your folder path, or click the browse button '...' to browse your disk and select a specific directory.
  • Look in subfolders: Instructs the software to look for data files in all subfolders too.

Filters are used to make a final decision if data matching the time period must be imported. To simplify entries and reduce typical case problems, filter strings entered are not case sensitive (the software makes no difference between uppercase and lowercase). Multiple filters conditions can be activated at once and include:

  • Tester name: lets you define a filter on the tester name. Keep it to * if you don't want to activate this filter, otherwise any string is valid, including wild characters such as * and ?. E.g.: test??_* will instruct the software to import test data matching the date period selected AND if data have been tested on a tester which name starts with the string test then includes 2 characters to ignore, then followed by a underscore character, then followed by any type of string.
  • Device name: Same as above except filter checks the device name (STDF part_type field)
  • Program name: Same as above except filter checks the test program name.
  • Lot name: Same as above except filter checks the lot name string.
