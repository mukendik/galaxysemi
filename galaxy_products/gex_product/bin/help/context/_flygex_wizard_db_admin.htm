More details:

: Create a new database collection.

Database location:

  • local: all data remain on your machine

  • server: remote data repository (assuming you have the write permission to the server).

Data storage:

Select Copy if you want the database to make a complete copy of all the data files imported (the database repository folders keep their own copy of the files imported). Otherwise, select or Link if you want the database to only maintain a structure of links.
If Link is selected, make sure not to move or delete your original test data files as long as the database exists! The benefit of the Link mode is a significant disk space saving as test data are not duplicated in an internal data structure!
Note: If using Yield-Man to import files automatically into the database, then the Copy mode is mandatory and Link should not be used as it wouldn't work properly. To avoid problems, this field is hidden when creating a database from the Yield-Man interface.

From the New database dialog box, click the More button to review advanced fields:

  • Description: enter any string to detail the purpose if the database

  • Import password: password protects the database so importing file can't be done without first entering the relevant password.

  • Delete password: password protects the database for delete.

: Lets you import files into a selected database. Import all types of files to be stored into the same data collection group (database). If the database was created with its data storage mode set to Link, then data files are not imported ; only the reference to the files are stored in the database structure, saving significant disk space!

: Lets you delete a database. The database and all its data will be permanently erased from your disk.
