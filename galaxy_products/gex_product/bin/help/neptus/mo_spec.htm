Parameter Specs. Monitoring

This task lets you monitor a parameter value (mean, Cp, Cpk, etc). Create as many tasks of this type as parameters values you want to monitor. This task is triggered each time a DataPump task is executed. As such, each time a data file is inserted into the database, the specified parameters are checked and an email notification is triggered in case of an alarm condition. Multiple emails and mailing list are supported (see below).

This task allows to monitor one or multiple parameters at once for one or multiple products. All you need is to enter the regular expression for the Product's name that matches your needs.

Product name string Comment
RFIC03 Monitor specs for one specific product (RFIC03 in this example)
RFIC03,RFIC04
or
RFIC03 | RFIC04		 Using the | (pipe character) or comma ','  you can list multiple products. In this case, Monitoring will check this monitoring rule on all the products listed (RFIC03 and RFIC04 in this example)
RFIC?? Use the ? character for wildcard selection. In this example, the current monitoring rule will apply to any product name starting with the string RFIC and followed by two characters.
RFIC* Use the * character for wildcard selection. In this example, the current monitroing rule will apply to any product name starting with the string RFIC.


The same apply to the Parameter number: you can monitor specifications from one or more parameters at once. All you need is to enter is the parameter / test# list:

Parameter/Test number string Comment
1000 Monitor specs for one specific parameter / test number (test# 1000 in this example)
1000,1010,1020
or
1000 | 1010 | 1020		 Using the | (pipe character) or comma ',' you can list multiple parameters. In this case, Monitoring will check this monitoring rule on all the tests listed (1000, 1010 and 1020 in this example)
* Use the * character for selecting ALL parameters/tests.

 

Mailing list

The email notification is sent to the email addresses or mailing list file you define. If you specify email addresses, use the semi-colon separator (e.g: user1@galaxysemi.com ; user2@galaxysemi.com)

If you click the Mailing list icon to specify a mailing list file, make sure the file selected complies with the following format:

  • File is in ASCII format

  • Each line includes one email address in format name@domain

  • If multiple addresses appear on one line, use the semi-colon separator ;

Example of mailing list ASCII file:

user1@galaxysemi.com
user2@galaxysemi.com
user3@galaxysemi.com ; user4@galaxysemi.com
user5@galaxysemi.com
