MASA MDE Models: Introduction to MDE models

 

What is an MDE model?

An MDE model is a representation that captures the statistical characteristics of a reference dataset of sample test vectors and allows for the analysis of all newly encountered test vectors to decide whether they are normal or abnormal with respect to the reference dataset.

Hence, an MDE analysis is a sort of novel multivariate analysis that has the following advantages over classical approaches such as T2/Hotelling:

  • the user has only one parameter to set
  • equally good performance is achieved both on highly non-gaussian data distributions and on gaussian data distributions

MDE analysis is applied to vectors of Inline Tests, possibly measured at a number of different locations on a wafer, thus making it possible to detect abnormal wafer patterns that could not be detected by a univariate analysis without complicated parameter setting procedures.

For each test vector, an MDE model outputs a MDE abnormality score. If a test vector has an MDE abnormality score of x, this means that, according to the MDE model,  (100.x)% of the reference test vectors are more normal than this vector. For example, an abnormality score of 0.99 indicates that only 1% of the reference vectors are more abnormal than the analyzed vector. In this case, if the user has set the MDE model's rejection rate parameter to 1% (or more) the tested data will be considered abnormal, that is rejected.

Note: MDE stands for Multivariate Density Estimation. MDE analysis is a technology invented and developed by MASA. Examinator's MDE functionalities are provided by the MASA MDE plug-in component for off-line analysis.

 

How does an MDE model work?

An MDE model is tuned during training over a dataset of reference test vectors. The reference dataset must be provided in one of the supported inputs formats.

Although the reference set must be representative of the normal behavior of the process, all reference test vectors do not necessarily need be normal. This means that it is not necessary to analyze and label every single test vector as normal or abnormal before training an MDE model.

To train an MDE model only one user parameter needs to be set, namely the desired rejection rate, namely the rate of wafers that the user wish to mark as abnormal in the reference set.

Once trained, an MDE model is stored in the MDE model library. MDE models are then used to estimate the abnormality scores of a dataset of test vectors, with respect to the reference set. The to-be-analyzed data must be provided in one of the supported inputs formats.

To perform an MDE analysis, the user selects an MDE model among the available ones and the result of this multivariate analysis is displayed in different MDE Reports that show the abnormality score distribution and the wafer result summaries.

The wafer result summary reports suggest lists of tests that are farthest from reference and provide links to their corresponding standard univariate reports provided by Examinator.
