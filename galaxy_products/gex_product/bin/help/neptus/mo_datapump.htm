DataPump: Automatic Data Insertion

Yield-Man should always have at least one task of this type. The DataPump task allows to automate the data collection of test data files and import them into a given database. From this task, you can automate the following:

Topics Comments
Import data files Define spooling folder that must be scanned periodically (if data come from remote places, favor FTP data transfers to the server using a script independent of the Examinator solution: this allows to maximize CPU and network load).

Data file in compressed format are supported for the following type:
  • *.zip : PC compression format from WinZip, PkZip or other tools
  • *.gz : Unix compression format from gzip tool
  • *.Z : Unix compression format from compress tool

Note: Have only one file per compressed archive (no multiple data files in a .zip or .gz or .Z file).

Valid files extensions (in compressed or uncompressed formats) include:

  • *.std and *.stdf for STDF V3 and STDF V4 files from all Test equipment vendors and E-Test.
  • *.atd and *.atdf for ATDF V2
  • *.gdf for Agilent's GDF format
  • *.pcm or *.csv for UMC foundry data files
  • *.wat or *.csv for TSMC foundry data files
  • *.spd for Credence / TMT
  • etc...
Database destination Must be a Server database type, so other users can have access to it from their computer).
Data import frequency Lets you define the period of execution; typically 30 minutes if you deal with very large amount of data, or less if you server has a configuration that can accommodate the volume of data to crunch.
Time window Time window: lets you disable the DataPump for certain hours...can be convenient to allow easy & save backup policy.
Source files after insertion... Lets you define what to do with the source files once inserted into the database. You can decide to discard them, or have them renamed. Default is to rename them erased from the polling folder

Note: Corrupted will be renamed and will have the .bad extension instead.
Yield Check Check yield: lets you check that any incoming file (with a minimum of parts tested) is over a yield limit. For a per-product yield monitoring, use the Yield Monitoring task instead as it gives more options. This task is rather a basic filter to ensure all data (no matter the product line) all path a minimum catastrophic yield level.

You can create as many DataPump tasks as you want. This offers you full flexibility to import data from multiple sources and/or into multiple databases.

 

Add/Change data header info at insertion

Yield-Man supports configuration files to let you add/change data header info on the flight at the time of the data insertion. The benefit is that you can be 100% sure on which data will be retrieved when you run queries and filters. As custom keys can be created, you can even create your own sorting/filtering keys (eg: Temperature, frequency, other...) [more]

 

Low Yield email notification: Mailing list

The email notification is sent to the email addresses or mailing list file you define. If you specify email addresses, use the semi-colon separator (e.g: user1@galaxysemi.com ; user2@galaxysemi.com)

If you click the Mailing list icon to specify a mailing list file, make sure the file selected complies with the following format:

  • File is in ASCII format

  • Each line includes one email address in format name@domain

  • If multiple addresses appear on one line, use the semi-colon separator ;

Example of mailing list ASCII file:

user1@galaxysemi.com
user2@galaxysemi.com
user3@galaxysemi.com ; user4@galaxysemi.com
user5@galaxysemi.com
 

Note: Have your system administrator check that the Monitoring email service is correctly setup otherwise emails notification won't go!
