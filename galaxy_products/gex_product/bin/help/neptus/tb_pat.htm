PAT-Man: Complete Outlier Identification for DPM (Defect-Per-Million) Reduction

PAT-Man - "Eat outliers before they eat you!"

Semiconductor parts that have abnormal characteristics (outliers) tend to be high contributors to quality and reliability issues. They directly affect the DPM level (Defects-Per-Million). To identify and eliminate such parts, Quantix has created PAT-Man, a  turnkey solution to easily address this challenging mission. This solution includes two modules:

  • Engineering module: used by test engineers at their desk for creating and editing the configuration files (relies on the Examinator client application, with the PAT-Man support.)

  • Production module: application running on the server (PC, Sun/Solaris or Linux) and automatically processes production data file to identify outliers and mark failing parts accordingly and email report on alarm condition (eg: yield loss to high)

There are many outlier identification algorithms available (Grubbs' test, Chauvenet's criterion, Tukey, recursive N*sigma, etc) but none is perfect to cover all situations. Distributions often suffer from skew ness (heavy tails) or asymmetry and kurtosis ; and therefore choosing the right outlier identification algorithms for a given parameter is key. Quantix PAT-Man's rule editor allows easy selection and customization of the algorithms applied to each individual parameter.

PAT-Man offers a unique user-friendly environment for quick and easily setup of a manual or automated outlier identification & removal process. Changing any rule and see the impact on the yield loss only takes seconds: the integrated rule editor allows quick and easy changes on any parameter.
