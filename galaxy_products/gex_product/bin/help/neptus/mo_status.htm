Web Site: Manufacturing/Test Status Web site

This task is MANDATORY when activating Yield-Man. It lets you enable the creation and automatic update of a Web site on your intranet. This web site shows:

  • Current products processed

  • Yield level per product

  • Testers in operation

  • Lots tested: lot,sub-lot, program name, yield level, etc

  • and more!

Be sure to specify a Web site path on your network that is visible to end-users. All Monitoring Web pages and Monitoring related files will be created under this web root path.
E.g.: email notifications on low yield will be created in a spooling folder under this path :
<StatusWebRootPath>/examinator_monitoring/.emails

Examinator-Monitoring lets you define the Web site structure to be created. It can be one of the following:

Web site organization Description
One site for ALL databases All status information are merged into one single web site organized per product and per tester.
One child-site per database One child web-site is created per database: avoids to merge all status information under one single Web site and eases analysis when focusing on one database only.

 

Also, an history of the past days is maintained. Click the Options tab, to can customize the history size from 1 week to a year and even more!
