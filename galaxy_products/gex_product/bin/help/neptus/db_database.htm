Databases: What is it, How does it work?

The Quantix Examinator Pro solution was designed with the following goals in mind:

  • Engine: Relies Quantix Examinator's very fast and easy to use engine

  • Data selection: Uses intuitive queries

  • Data collection: Provides a simple interface to easily create data containers (database) where any types of data (test data, simulation, design, ...) can be imported.

What is a Database?

A database is a collection of files in a proprietary format that converts the data you import so to be added to the existing data collection. You may dump all your data into a single database (e.g.: production database), or create different smaller databases with a specific purpose (e.g.: QA database, or Wafer test database). In any case, a simple way to think of a database is to visualize a collection of data into a folder...the folder being the database name!

Examinator Pro's approach with Databases was so as not to rely on Oracle or any generic database provider andf thereby eliminate the significant overhead of generic SQL queries. Examinator Pro  is using a much more straightforward and efficient way to deal with your data: all the data files you import in a Examinator Pro database are copied to the database folder and kept unchanged! This guarantees full availability of the information stored in your data files, and eliminates the unpleasant surprise of parsers that silently ignore key information when importing your file!
 

How does it work?

Examinator Pro allows you to create and work with databases that are either local (your account) or on the server. Each database can hold as much data as you want...the only limit being your disk size!

The purpose of local databases is when you are conducting data analysis on your own, with no need to share with others. The Examinator Pro interface will list your local databases, but other users won't have access to it. This mode also allows faster data analysis as data and queries are all executed locally [more].

The purpose of server databases is when data need to be accessed by different people and teams, and also when you want to have real-time information reporting (e.g: production line on-line Yield HTML report) [more].
 

Databases Names?

Databases names can include any character, there is no restriction in the characters to be used or the string length specified! The associated folder Examinator Pro creates is the same name except that all invalid characters for a folder name are replaced with the underscore '_' character. As such, the database My Data (1) will be saved under the folder name My_Data_1_
 

Databases Parameters / Tests

Because Quantix's focus remains the semiconductor industry, parameters manipulated are called tests. This is simply a terminological choice, and doesn't bring any limitation in what you can accomplish.  The solutions from Quantix support the following data types where any parameter is assigned a test number :

  • WIP Lot and Wafer History

  • Inline Lot and Wafer Metrology Measurements

  • Electrical Test  (E-Test)

  • Wafer Sort

  • Assembly

  • Final Test

  • And Non-Lot Equipment Measurements (simulation data, CSV files, etc.)
