Yield Monitoring

This task lets you define the yield level you expect on a per-product basis. Create as many tasks of this type as products you want to monitor. This task is executed each time a DataPump task is executed. As such, each time a data file is imported in the database, its yield level is checked (you can define which software bins to monitor). You can setup the task to look for a low yield (e.g: for good bins) or a high yield (e.g: for continuity bins) and trigger an email notification in case of an alarm condition. Multiple emails can be entered if separated with the semicolon character.

This task allows to monitor multiple products at once. All you need is to enter the regular expression for the Product's name that matches your needs.

Product name string Comment
RFIC03 Monitor yield for one specific product (RFIC03 in this example)
RFIC03,RFIC04
or
RFIC03 | RFIC04		 Using the | (pipe character) or comma ',' you can list multiple products. In this case, Monitoring will check this yield rule on all the products listed (RFIC03 and RFIC04 in this example)
RFIC?? Use the ? character for wildcard selection. In this example, the current yield rule will apply to any product name starting with the string RFIC and followed by two characters.
RFIC* Use the * character for wildcard selection. In this example, the current yield rule will apply to any product name starting with the string RFIC.


Bin list, Bin type

The bin list lets you define which software bins to monitor. Can be any list you want.
The Bin type is very important and should always be set to the correct value:

  • Good bins: This means the bin list is the list of good binning (this information is used in the Web pages created when showing the yield level in percents)

  • Failing bins: This means the bin list includes failing bins.


Mailing list

The email notification is sent to the email addresses or mailing list file you define. If you specify email addresses, use the semi-colon separator (e.g: user1@galaxysemi.com ; user2@galaxysemi.com)

If you click the Mailing list icon to specify a mailing list file, make sure the file selected complies with the following format:

  • File is in ASCII format

  • Each line includes one email address in format name@domain

  • If multiple addresses appear on one line, use the semi-colon separator ;

Example of mailing list ASCII file:

user1@galaxysemi.com
user2@galaxysemi.com
user3@galaxysemi.com ; user4@galaxysemi.com
user5@galaxysemi.com
 

Automatic Report Generation & Email

Using the 'Messages Contents' combo box, you can select the type of message to be send when a yield alarm condition is triggered. Your choices include:

  • Summary ASCII Report: you'll receive a text email with a summary (Yield level per bin, etc)

  • Word report: you'll receive the ASCII summary in the body of the email as well as a full Word report as an attachment. The report document is similar to the report created using Examinator with sections for test statistics, histograms, wafermap, binning and pareto. This report format is only available if Monitoring is running under Windows; this format is not supported under Unix.

  • Excel/CSV report: same as above except file format is CSV

  • PowerPoint report: same as above except file format is PowerPoint (not available under Unix)

  • PDF report: same as above except file format is PDF

Hint: To have a report generated for each data file processed, simply create a Yield monitoring task triggered if Yield level is greater than 0%!

Note: Have your system administrator check that the Monitoring email service is correctly setup otherwise emails notification won't go!
