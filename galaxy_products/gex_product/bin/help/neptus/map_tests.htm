Map / Change Test Numbers

Quantix Examinator makes it very easy to map any test number to a new index! It alsp allows to map test names to new names! This is very convenient when you have to compare data files created on different tester platforms with different test names and/or indexes. The process is the following:

  • Create a text file with your mapping table (see format below)

  • Click the Examinator 'Data' tab and double click any test file that must use your mapping table (multiple selections supported) to display the Properties page.

  • By the Map Tests field, click the file browse icon to select your mapping file

Mapping file format: Test number

Your tests numbers mapping table consists of two columns of numbers with a comma separator. The first column includes a keyword (action to take), then parameters:

Each line mapping a test number is as following:

Test#, <Old_Test_Number>  =   <New_Test_Number>

E.g.:

# Map Tests 1000 to 1002 to higher indexes!
Test#, 1000 = 1200
Test#, 1001 = 2200
Test#, 1002 = 3000

Few hints:

  • Any line starting with the # character is ignored

  • Tests not listed in the first column (old test number) are not mapped, and therefore remain unchanged

  • Tests mapped to a same new test (defined in the second  column) are merged. It is better to do this only on tests with same test limits!

  • Test numbers must be integers only (no pin map accepted. e.g.: 12.4 is NOT a valid test number)

  • The mapping file must be saved in ASCII format, file extension should be .txt or .csv so to be visible under the Examinator environment.

Mapping Test Name

Each line mapping a test name is as following:

TestName, <Old_Test_Name>  =   <New_Test_Name>

E.g.:

# Map Tests names
TestName, continuity_old1 = New_Continuity_Name_1
TestName, continuity_old2 = New_Continuity_Name_2
 

Note: mapping test names is very convenient for test programs that include typos and need test names to be fixed.

 

Typical pitfall

It may happen that while you've used the mapping feature then compare files, you don't end-up with the expected result and Examinator doesn't compare your tests. You'll wonder why Examinator failed to compare the tests while they now have the same test number? The answer is that Examinator has an option you can toggle that either compares tests based on the test number only...or also checks if test names are matching! So this is a typical pitfall to avoid: if you know tests to compare may not have the same test name, then force Examinator to compare tests based on the test number only. The option is available from the Examinator 'Options' tab, section 'Data processing options / Handling duplicated test numbers'.

 

Example of Mapping file:

#################################################################
# Mapping Test Numbers
#
# o Any line starting with the '#' is a comment line and is ignored.
# o File format consists of one line per test to map.
# o If multiple tests are mapped to the same index
# ...then data are merged under a single Test number!
#################################################################
# List of few tests to map...with 3 tests to merge into Test 5000.
# Test#, <Original_Test_Index> = <New_Test_Index>
#
Test#, 1000 = 5000
Test#, 1001 = 5000
Test#, 1002 =  5000
Test#, 1003 = 5001
Test#, 1004 = 5007
