Processing WAT Parameters files

The Examinator product line supports native WAT (Wafer Acceptance Test) files from TSMC. Files must have the extension .WAT and NOT .txt ... otherwise Examinator will invoke the CSV Import Wizard instead of automatically reading and parsing the WAT file!
Examinator supports many other test data formats so you are never let down!

Because Examinator supports the native WAT data format, you do not need any parser or third-party software to process it! Simply select the files you want to process, it is that simple!

To comply with the Examinator wafermap coordinates system, each WAT site is assigned a wafer map die area using X,Y coordinates as shown below:

WAT site name WAT site # Examinator X,Y die location assigned
Center 1  X= 1, Y= 1
Bottom 2  X= 1, Y= 2
Left 3  X= 0, Y= 1
Top 4  X= 1, Y= 0
Right 5  X= 2, Y= 1
Lower-Right corner 6  X= 2, Y= 2
Lower-Left corner 7  X= 0, Y= 2
Upper-Left corner 8  X= 0, Y= 0
Upper-Right corner 9  X= 2, Y= 0

For site# higher than 9, Examinator forces X,Y die location to 0.
